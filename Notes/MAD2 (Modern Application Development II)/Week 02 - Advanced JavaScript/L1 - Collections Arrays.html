JavaScript Collections

— arrays, and the tools built to walk through them —

Continuing from L06 Same functions and arrow syntax from DOM work — now applied to processing whole collections at once.

Arrays: Not Like C

A C array is an egg carton — fixed slots, one kind of item. A JS array is a junk drawer — grows as needed, holds anything.

Arrays Are Objects

Real Life

That junk drawer is really a labeled storage box — each item's "location" (0, 1, 2...) is just a label, the same way a box lid might say "left compartment" or "right compartment."

Analogy

Array indices are property names in disguise — arr[0] works a lot like obj.zero, because underneath, an array is an object.

Iteration & Functional Helpers

Instead of manually walking through a collection, hand the whole thing to a helper that already knows how to walk.

.map()

Transforms every item, returning a new array of the same length.

Real Life

A photo booth that stamps every ID photo onto a badge — one badge per photo, same order, same count.

Analogy

Every input gets processed into an output — nothing is skipped, nothing is added.

.filter()

Keeps only the items that pass a test.

Real Life

A bouncer at a club entrance, letting in only guests who meet the dress code.

Analogy

The line goes in; a smaller line comes out — everyone who didn't pass the test is simply turned away.

.find()

Returns the first item that matches, then stops looking.

Real Life

A search dog sniffing down a row of suitcases and sitting at the very first one that matches the scent.

Analogy

No need to check every remaining suitcase once a match is found — find stops the instant it succeeds.

Advanced Concepts

Destructuring

Extracts multiple values from an array into named variables in one line.

Real Life

Unpacking a grocery bag directly into labeled jars on the counter — milk into the milk jug, eggs into the egg tray — instead of leaving everything jumbled in the bag and reaching in each time.

Analogy

const [a, b] = arr is that unpacking step — one motion, everything lands where it's named.

Generators (yield)

Functions that pause mid-execution and resume later, remembering exactly where they left off.

Real Life

A chef who hands you one course at a time — pausing after each dish, remembering exactly where they left off, ready to continue the moment you ask for the next course.

Analogy

Nobody's stuck waiting for the whole seven-course meal to finish cooking before the first plate arrives — that's cooperative concurrency in a kitchen.

Emptying & Resizing Arrays

arr = null

Like closing down the store entirely — the shelf, the sign, the whole location is gone. Nothing left to reference.

arr.length = 0

Like keeping the store open but clearing every shelf — the array itself still exists, just empty and ready to be refilled.

Manually Setting Length (Holes)

Real Life

A parking garage sign advertising "10 spots," but only 3 cars are actually parked — the other 7 signs are lit up as empty, even though no one ever assigned them.

Analogy

If you set arr.length = 10 on a 3-item array, slots 3–9 become "holes" — accessing them returns undefined, exactly like peeking into an empty, never-filled parking spot.

Modern Collections: Map & Set

Map

Stores key–value pairs, cleanly.

Real Life

A coat check counter — you hand over your coat, get a numbered ticket, and that exact ticket retrieves your exact coat later.

Analogy

Any value can be a "ticket" (key) in a Map — not just plain strings, the way older object-based workarounds required.

Set

Stores only unique values — duplicates are automatically ignored.

Real Life

A guest list at the door — writing the same name down twice doesn't get anyone in twice; the list just has one entry per person.

Analogy

Adding a duplicate to a Set is a no-op — the guest list simply doesn't grow.

Arrays are junk drawers, not egg cartons — and map, filter, and find are just different-shaped hands reaching into that drawer for you.
