Getters & Setters

Properties that don't just store a value — they compute or react to one.

get — Derived Property

Looks like a plain property, but is computed fresh every time it's accessed.

Real Life

A car's dashboard "average speed" readout — it's not stored anywhere permanent, it's recalculated from distance and time the instant you glance at it.

Analogy

A get fullName() reads exactly like person.fullName — no parentheses needed — but under the hood it's combining firstName and lastName live.

set — Reactive Assignment

Runs custom logic whenever a value is assigned, often updating several fields at once.

Real Life

A thermostat dial — you set one number, "72°," but internally that single input adjusts the compressor, fan speed, and vents all at once.

Analogy

set fullName(name) can take one string and quietly split it back into firstName and lastName — one assignment, several updates.

call, apply, bind

All three control what this means inside a function — they just differ in when it runs and how arguments arrive.

call()

Runs immediately, with a specific this and arguments listed one by one.

Real life: a guest chef using another restaurant's kitchen right now, explicitly told "for tonight, treat me as head chef here."

apply()

Same as call(), but arguments arrive as one pre-packed array.

Real life: handing that same guest chef a grocery bag of ingredients already packed, instead of listing each item out loud.

bind()

Doesn't run yet — returns a new function with this (and maybe some args) locked in for later.

Real life: pre-programming a thermostat schedule once, so it can trigger the exact same settings automatically, anytime, without you re-entering them.

Prototypal Inheritance & __proto__

If a property isn't found on the object itself, JS checks __proto__ next — like checking the spare key at the neighbor's house before giving up.

Constructors & new

Initializes a fresh object's properties the moment it's created.

Real Life

A car factory's assembly template — every time the "start" button is pressed, a new car rolls out with the standard features already installed.

Analogy

new Car() is pressing that start button — the constructor is the assembly step that runs automatically for every unit.

extends & super

Lets a subclass inherit a parent's behavior, then add to or override it.

Real Life

A spinoff recipe that says: "first do everything my parent recipe does (that's super()), then add my own twist on top."

Analogy

A Manager class extending Employee still gets every base employee behavior for free — it only needs to describe what's different about managing people.

Getters and setters make properties act smart; class makes prototype chains look tidy — but underneath, it's still one object quietly asking another for what it doesn't have.
