Iteration & Destructuring

Master the four iteration patterns in JavaScript and the destructuring syntax that makes working with arrays and objects clean and concise.

Why This Topic Exists
Every application loops over data — rendering lists, processing API responses, transforming state. Choosing the wrong iteration method leads to bugs, poor performance, and unreadable code. Destructuring eliminates repetitive obj.property chains. Together, they're the difference between spaghetti code and declarative data pipelines.
The Problem This Solves
You receive a JSON API response with 10,000 user objects. You need to:
1. Extract just the names — but for loops require 5+ lines of boilerplate.
2. Filter active users — but for...in skips array holes and iterates prototype properties.
3. Unpack nested response objects — but accessing response.data.user.name everywhere is verbose.
4. Sort users by score — but sort() mutates the original array, breaking React state.
How do you iterate safely, unpack cleanly, and transform immutably?
Mental Model — The Assembly Line
Think of iteration as a factory assembly line:
Items enter the conveyor belt one by one. map repaints each item. filter removes defective ones. reduce melts them into a single ingot. sort rearranges the line. You chain these stations together — raw data goes in, finished product comes out.
for → Manual worker — controls every step, but slow and error-prone
for...of → Automated conveyor — processes values in order, clean
for...in → Label reader — reads names (keys), not the items themselves
Destructuring → Unboxing station — extracts contents into labeled bins
Concept Map — Iteration & Destructuring
1 Four Ways to Iterate

for, for...in, for...of, and forEach

Each iteration method has different behavior with arrays, objects, and iterables. Choosing the right one prevents subtle bugs.
// 1. Classic for loop — full control
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]);
}

// 2. for...in — iterates KEYS (indices for arrays)
for (const key in arr) {
  console.log(key, arr[key]); // key is "0", "1", "2" (strings!)
}

// 3. for...of — iterates VALUES (preferred for arrays)
for (const val of arr) {
  console.log(val);  // val is the actual element
}

// 4. forEach — functional, no return value
arr.forEach((val, idx) => {
  console.log(idx, val);
});
for...in (Avoid for Arrays)
Iterates string keys, skips holes, includes prototype properties. Use only for plain objects.
for...of (Use for Arrays)
Iterates actual values, works with any iterable (Array, Map, Set, String). Clean and predictable.

Object Iteration — Not a Built-in Iterable

Plain objects are not iterable with for...of. You must convert them first.
const user = { name: "Alice", age: 30, role: "admin" };

// Object.entries — [key, value] pairs
for (const [k, v] of Object.entries(user)) {
  console.log(`${k}: ${v}`);
}

// Object.keys — array of keys
Object.keys(user);   // ["name", "age", "role"]

// Object.values — array of values
Object.values(user); // ["Alice", 30, "admin"]
Common Mistake
for...in on an array iterates string indices, not values. "0" + "1" = "01" instead of 0 + 1 = 1. Always use for...of or forEach for arrays.
2 Array Methods for Data Processing

The Big Five: find, filter, map, reduce, sort

const users = [
  { name: "Alice", age: 28, active: true },
  { name: "Bob",   age: 35, active: false },
  { name: "Carol", age: 22, active: true },
];

// find — first match
users.find(u => u.age > 30);     // { name: "Bob", ... }

// filter — all matches
users.filter(u => u.active);     // [Alice, Carol]

// map — transform each
users.map(u => u.name);          // ["Alice", "Bob", "Carol"]

// reduce — accumulate
users.reduce((sum, u) => sum + u.age, 0); // 85

// sort — mutates! Use copy first
[...users].sort((a, b) => a.age - b.age); // sorted copy
Best Practice
Chain methods for readability: users.filter(u => u.active).map(u => u.name) is clearer than nested callbacks. Keep each method to one responsibility.
3 Execution Timeline — Method Chaining

STEP 2 — map(u => u.name)

4 Destructuring — Unpacking Syntax

Array Destructuring

Extract values from arrays into variables by position.
const rgb = [255, 128, 64];

// Basic
const [red, green, blue] = rgb;
console.log(red); // 255

// Skip elements
const [first, , third] = rgb;
console.log(third); // 64

// Rest pattern
const [head, ...tail] = rgb;
// head = 255, tail = [128, 64]

// Default values
const [a = 0, b = 0] = [];
// a = 0, b = 0

// Swap variables (no temp needed!)
let x = 1, y = 2;
[x, y] = [y, x]; // x = 2, y = 1

Object Destructuring

Extract properties by name, not position. Supports renaming and rest.
const user = { name: "Alice", age: 30, role: "admin" };

// Basic
const { name, age } = user;

// Rename
const { name: userName, role: userRole } = user;

// Default values
const { name, score = 0 } = user;
// score = 0 (not in object)

// Rest — everything else
const { name, ...rest } = user;
// rest = { age: 30, role: "admin" }

// Nested destructuring
const { address: { city } } = user;
// city = user.address.city
Real-World Analogy
Array destructuring is like unpacking a suitcase by order — first item is the shirt, second is the pants. Object destructuring is like opening labeled drawers — you grab the "name" drawer regardless of where it sits.
5 Limitations & Gotchas
Limitations
  • sort() mutates in place: Always spread first: [...arr].sort(). This is the #1 source of bugs in React/Vue.
  • for...in iterates prototypes: Use hasOwnProperty() check or prefer Object.keys()/Object.entries().
  • for...of doesn't work on plain objects: Objects aren't iterable. Use Object.entries() first.
  • Deep destructuring fails on undefined: const { a: { b } } = {} throws. Use defaults or optional chaining.
  • new Array(n) creates holes: [...new Array(3)] gives [undefined, undefined, undefined], not filled values.
  • forEach can't break: No return, break, or continue. Use for...of or .find() instead.
6 Debugging Checklist
When Iteration/Destructuring Goes Wrong
1
Check if sort() mutated your original array — verify with console before/after
2
If for...in gives unexpected keys, check if prototype properties are leaking — use Object.hasOwn(obj, key)
3
If destructuring gives undefined, verify the source object has that key — log the object first
4
If map returns unexpected results, check if you're chaining on a non-array (e.g., NodeList from querySelectorAll)
5
If reduce throws "cannot read property of undefined", verify you provided an initialValue
6
If for...of throws "is not iterable", check if the variable is actually an array — wrap with Array.from() if needed
7 Code Patterns — Do This, Not That
Anti-pattern Mutating sort in React 
// BAD — mutates state, won't trigger re-render
const sorted = items.sort((a, b) => a.name.localeCompare(b.name));
setItems(sorted); // same reference!
Correct Immutable sort with spread 
// GOOD — new array reference triggers re-render
const sorted = [...items].sort((a, b) => a.name.localeCompare(b.name));
setItems(sorted);
Pattern Clean API response processing 
// Chain: filter → map → sort
const activeNames = users
  .filter(u => u.active)
  .map(u => u.name)
  .sort();
8 Previous Year Questions (PYQs)
PYQs — Iteration & Destructuring
Q: What is the difference between for...in and for...of?
A: for...in iterates over keys (property names / indices as strings). for...of iterates over values of any iterable. for...in includes prototype properties; for...of does not.
20243 marks
Q: Write a program to extract names of active users using array methods.
A: users.filter(u => u.active).map(u => u.name). Filter keeps active users, map extracts names. Both are non-mutating and chainable.
20233 marks
Q: What is the output of: const [a, , b] = [1, 2, 3]; console.log(a + b);
A: 4. The comma skip ignores index 1, so a = 1, b = 3. 1 + 3 = 4.
20242 marks
Q: How do you iterate over an object's properties?
A: Use Object.entries(obj) to get [key, value] pairs, then iterate with for...of. Alternatively, use Object.keys(obj) or Object.values(obj) for just keys or values.

10 Memory Tricks
Mnemonics to Remember
for...in = "Key In"
Key In — Key In. for (key in obj). Keys go IN your variable. Think "I have the key to the door."
for...of = "Value Of"
Value Of — you get the value OF each item. for (val of arr). Think "the value OF this thing."
sort mutates = "SOrt Mutates"
S-O-M = Sort Only Mutates. sort() is the ONLY common array method that mutates in place. All others (map, filter, find) return new arrays.
Destructuring direction
Into variables = Destructure. Out of a container = Spread. Same ... syntax, opposite directions. D = Distribute into parts. S = Scatter outward.

12 One-Page Revision Summary
Iteration & Destructuring — Complete Cheat Sheet
for loop
C-style, full control. for (let i=0; i. Use when you need index or break/continue.
for...in
Iterates keys. Use for objects only. For arrays, it gives string indices — avoid.
for...of
Iterates values. Use for arrays, Maps, Sets, Strings. Supports break/continue.
forEach
Functional style. No return value, no break. Use for side effects only.
Array Destructuring
const [a, b] = arr. Skip: [a, , c]. Rest: [h, ...t]. Swap: [x,y] = [y,x].
Object Destructuring
const { name, age } = obj. Rename: { name: n }. Rest: { name, ...r }.
sort() mutates!
Always use [...arr].sort() for immutable sorting. Default sort is lexical (string-based).
Object iteration
Object.entries(obj) → pairs. Object.keys(obj) → keys. Object.values(obj) → values.
