JavaScript Collections

Iterations and Destructuring

1

Iteration techniques

For loop

Classic C-style. Explicit index control via let i. Full control over increment and range.

For...in

Iterates over keys (indices for arrays, property names for objects). Skips empty/undefined slots.

Caution: skips holes

For...of

Iterates over values of any iterable. Does not skip undefined entries.

Preferred for arrays

Object iteration

Objects are not iterable directly. Use Object.entries(obj) to get [key, value] pairs, then destructure.

2

Array methods for data processing

find

Returns the first element that satisfies the test function. Returns undefined if none match.

filter

Returns a new array of all elements passing the test. Non-destructive.

map

Creates a new array by transforming every element. 1-to-1 mapping.

reduce

Collapses array into a single value. Takes a callback and an initial accumulator.

sort

Sorts in place. Default is lexical (string). For numbers use (a, b) => a - b.

Mutates original
3

Advanced syntax and destructuring

Array construction

new Array(n) creates an array of size n with empty slots (holes), not undefined. Fill individually or use .fill().

Spread operator ...

Expands an iterable inline — useful for merging arrays, copying data, or passing elements as arguments. E.g. [...a, ...b] merges two arrays.

Array destructuring

Unpack array values into variables by position: const [a, b] = x. Clean, concise alternative to index access.

Object destructuring

Extract properties into variables. Rename: const { first_name: fn } = person  ·  Rest: const { first_name, ...rem } = person captures all remaining properties.
