JSON

— the flat-pack format that lets any two machines ship data to each other —

Continuing from L10 Same objects and classes from earlier — now flattened into text so they can travel across a network.

JSON is a shipping format, not a language.

It doesn't care whether the sender was written in JavaScript, Python, or Java — as long as both sides agree on the same flat, text-based packing format, the data survives the trip.

A "Frozen" Specification

Real Life

A passport's page layout — every country's passport office agrees to the exact same format, forever, so any border checkpoint anywhere can read it without guessing.

Analogy

JSON being "frozen" means no country (or company) can quietly redesign the format later — every system that reads JSON today will still read it correctly years from now.

stringify() & parse()

stringify() flattens the assembled chair into a flat-pack box for shipping. parse() unpacks that box back into something usable at the other end.

JSON.stringify()

Turns a JS object into a plain text string.

Real Life

Flat-packing a fully built chair back into its box before shipping — smaller, standardized, and easy for a courier to carry.

JSON.parse()

Turns a JSON string back into a usable JS object.

Real Life

Opening the flat-pack box at the destination and reassembling the parts back into a chair you can actually sit in.

Reconstructing Classes

The Problem

Real Life

The flat-pack box only ships the wood panels and screws — not the brand's assembly manual or any of the chair's folding hinges' built-in mechanisms.

Analogy

JSON ships the data (the wood and screws) but not the class's methods (the folding mechanism) — parsing alone gives you loose parts, not a working Cat instance.

The Fix: fromJSON

Real Life

Calling in the original manufacturer's assembly manual to turn those loose parts back into a fully working, foldable chair — hinges and all.

Analogy

A static Cat.fromJSON(data) method takes the plain parsed object and feeds it into the real constructor — restoring every method the plain data alone couldn't carry.

Why JSON, Specifically

Readable & Language-Agnostic

Real Life

A packing slip written in plain, universal terms ("2 chairs, 4 screws") that any warehouse worker can read, regardless of which country or company packed the box.

Analogy

A Python server and a JavaScript browser don't speak the same "language" internally — but both can read the same JSON packing slip without translation.

JSON is the universal flat-pack format for data — stringify boxes it up, parse opens it back up, and fromJSON is the manual that restores what got left out of the box.
