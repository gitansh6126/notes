⚠️ Sparse Arrays & Array-like Objects

Sparse arrays have holes: const arr = [1, , 3]. Holes are treated as undefined by most iteration methods. Array-like objects have a length property and indexed elements (e.g., arguments, DOM NodeLists). Convert with Array.from() or spread.

📝 PYQ — Array Method Output Difficulty: ⭐ 
const arr = [1, 2, 3];
arr.splice(1, 1, 99);
console.log(arr);

Answer: [1, 99, 3]

Destructuring

📐 Array Destructuring

const [a, b] = [1, 2];            // a=1, b=2
const [x, , z] = [10, 20, 30];    // x=10, z=30
const [p, q = 99] = [1];          // p=1, q=99 (default)
const [first, ...rest] = [1,2,3];  // first=1, rest=[2,3]

📐 Object Destructuring

const { name, age } = { name: 'Alice', age: 25 };
const { name: n, age: a } = { name: 'Bob', age: 30 };  // rename
const { city = 'Chennai' } = {};                    // default
const { address: { pin } } = { address: { pin: 600 } };  // nested

🔄 Swapping Variables

let a = 1, b = 2;
[b, a] = [a, b];  // a=2, b=1
📝 PYQ — Destructuring Output Difficulty: ⭐⭐ 
const obj = { x: 10, y: 20, z: 30 };
const { x, ...rest } = obj;
console.log(x, rest);

Answer: 10 { y: 20, z: 30 }

Spread Operator

// Array spread
const arr1 = [1, 2];
const arr2 = [3, 4];
const merged = [...arr1, ...arr2];  // [1, 2, 3, 4]

// Object spread
const obj1 = { a: 1, b: 2 };
const obj2 = { b: 3, c: 4 };
const mergedObj = { ...obj1, ...obj2 };  // { a: 1, b: 3, c: 4 }

// Function arguments
Math.max(...[3, 7, 1]);  // 7

⚠️ Shallow Copy Trap

const original = { a: 1, nested: { b: 2 } };
const clone = { ...original };
clone.nested.b = 99;
console.log(original.nested.b);  // 99 — shared reference!
📝 PYQ — Spread Operator Difficulty: ⭐⭐ 
const a = [1, 2];
const b = [3, ...a, 4];
console.log(b);

Answer: [3, 1, 2, 4]

map()

arr.map(callback(currentValue, index, array)) — Returns a new array of the same length. Chainable with filter() and reduce().

const nums = [1, 2, 3];
const doubled = nums.map(n => n * 2);  // [2, 4, 6]
const parsed = ['1', '2'].map(Number);  // [1, 2]
📝 PYQ — parseInt with map (🔥 TRICKY) Difficulty: ⭐⭐⭐ 
console.log(['1', '2', '3'].map(parseInt));

Answer: [1, NaN, NaN]
parseInt receives (element, index, array). So parseInt('1',0) → 1, parseInt('2',1) → NaN, parseInt('3',2) → NaN (base 2 has no digit 3).

filter()

arr.filter(callback) — Returns a new array with elements for which callback returns truthy. Skips empty slots.

const nums = [1, 2, 3, 4, 5];
const evens = nums.filter(n => n % 2 === 0);  // [2, 4]

// Chaining
nums.filter(n => n > 2).map(n => n * n);  // [9, 16, 25]
📝 PYQ — Filter Output Difficulty: ⭐ 
console.log([1, , 2, , 3].filter(x => true));

Answer: [1, 2, 3] — empty slots are skipped by filter.

reduce()

arr.reduce(callback(acc, current, index, array), initialValue)

// Sum
[1, 2, 3].reduce((a, b) => a + b, 0);           // 6

// Flatten
[[1,2], [3], [4,5]].reduce((a, b) => [...a, ...b], []);  // [1,2,3,4,5]

// Group by
['apple', 'banana', 'apricot'].reduce((acc, s) => {
  const key = s[0];
  (acc[key] = acc[key] || []).push(s);
  return acc;
}, {});

// Count occurrences
['a','b','a'].reduce((acc, v) => (acc[v] = (acc[v]||0) + 1, acc), {});
📝 PYQ — Reduce Output Difficulty: ⭐⭐ 
console.log([1, 2, 3].reduce((a, b) => a + b));

Answer: 6 — no initialValue, so a=1, b=2 (first iteration), then a=3, b=3.

find()

arr.find(callback) — Returns the first element that satisfies the condition, or undefined.

  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' },

];
const user = users.find(u => u.id === 2);  // { id: 2, name: 'Bob' }

forEach()

arr.forEach(callback) — No return value (returns undefined). Used for side effects. Cannot break or continue (except via thrown exception).

const arr = [1, 2, 3];
arr.forEach(n => console.log(n * 2));  // 2, 4, 6 (returns undefined)

const mapped = arr.map(n => n * 2);     // [2, 4, 6] (returns array)

Promises

📌 States

Pending → Fulfilled or Rejected (settled, immutable thereafter)

🔧 Static Methods

MethodResolves whenRejects when
Promise.all(iterable)All settle fulfilledAny rejects (fail-fast)
Promise.allSettled(iterable)All settle (any result)Never rejects
Promise.race(iterable)First settles (any)First settles (any)
Promise.any(iterable)First fulfillsAll reject
 
// Promise constructor
const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve('done'), 1000);
});

// Chaining
Promise.resolve(1)
  .then(x => x + 1)
  .then(console.log);  // 2
📝 PYQ 1 — Error Propagation Difficulty: ⭐⭐ 
Promise.resolve(1)
  .then(x => x + 1)
  .then(x => { throw x })
  .then(x => console.log('A'))
  .catch(x => console.log(x))

Answer: 2 — throws the value 2, caught by catch.

📝 PYQ 2 — Execution Order Difficulty: ⭐ 
Promise.resolve()
  .then(() => console.log(1))
  .then(() => console.log(2))

Answer: 1 (then) 2 (then) — chained thens execute sequentially.

📝 PYQ 3 — Control Flow Difficulty: ⭐⭐⭐ 
Promise.resolve()
  .then(() => console.log(1))
  .catch(() => console.log(2))
  .finally(() => console.log(3))

Answer: 1 (then) 3 (finally) — catch skipped because no rejection.

📝 PYQ 4 — Promise.all with Rejection Difficulty: ⭐⭐⭐⭐ 
Promise.all([
  Promise.resolve(1),
  Promise.reject('error'),
  Promise.resolve(3)
]).catch(e => console.log(e));

Answer: 'error' — Promise.all rejects fast on first rejection.

Async / Await

An async function always returns a Promise. await pauses execution until the promise settles.

async function fetchData() {
  try {
    const res = await fetch(url);
    return await res.json();
  } catch (err) {
    console.error(err);
  }
}

// Parallel execution
const [a, b] = await Promise.all([p1(), p2()]);
📝 PYQ 1 — Basic Execution Order Difficulty: ⭐⭐ 
async function foo() {
  console.log(1);
  await Promise.resolve();
  console.log(2);
}
foo();
console.log(3);

Answer: 1 (sync), 3 (sync), 2 (microtask after await).

📝 PYQ 2 — Async/Await with Promise.all Difficulty: ⭐⭐⭐ 
async function bar() {
  const r = await Promise.all([
    Promise.resolve(10),
    Promise.resolve(20)
  ]);
  console.log(r);
}
bar();

Answer: [10, 20]

📝 PYQ 3 — Error Handling Difficulty: ⭐⭐ 
async function fail() {
  throw new Error('oops');
}
fail().catch(e => console.log(e.message));

Answer: 'oops'

Event Loop

🔁 Execution Order

Sync code → Microtasks (Promise.then/catch/finally, queueMicrotask, MutationObserver) → Macrotasks (setTimeout, setInterval, I/O)

console.log('sync');

setTimeout(() => console.log('macrotask'), 0);

Promise.resolve().then(() => console.log('microtask'));

console.log('sync end');

// Output: sync → sync end → microtask → macrotask
📝 PYQ 1 — Classic setTimeout vs Promise Difficulty: ⭐⭐ 
console.log(1);
setTimeout(() => console.log(2), 0);
Promise.resolve().then(() => console.log(3));
console.log(4);

Answer: 1, 4, 3, 2

📝 PYQ 2 — Nested Microtasks Difficulty: ⭐⭐⭐ 
Promise.resolve().then(() => {
  console.log(1);
  Promise.resolve().then(() => console.log(2));
}).then(() => console.log(3));

Answer: 1, 2, 3 — microtasks are queued and processed before next macrotask.

📝 PYQ 3 — setInterval with Promise Difficulty: ⭐⭐⭐⭐ 
let i = 0;
const id = setInterval(() => {
  console.log('A');
  if (++i === 2) clearInterval(id);
}, 0);
Promise.resolve().then(() => console.log('B'));

Answer: B, A, A (microtask B runs before each macrotask A).

Tricky Questions 🔥 Watch Out

  1. parseInt with map: ["1","2","3"].map(parseInt) → [1, NaN, NaN]
    parseInt receives index as radix!
  2. reduce without initialValue with empty array: [].reduce((a,b) => a+b) → TypeError
  3. Promise inside forEach vs map: forEach ignores returned promises (parallel), map can collect them
  4. async/await in forEach: doesn't wait! Use for...of instead
  5. Event loop: setTimeout(0) runs after Promise.resolve()
  6. Chained filter().map(): executes fully sequentially — filter completes before map starts
  7. Object.is() vs ===: Object.is(NaN, NaN) → true, Object.is(-0, +0) → false
  8. Spread with nested objects: shallow copy trap — nested objects are shared references
  9. Destructuring with rename and default: const { x: y = 5 } = {} → y = 5
  10. Missing return in .then(): next then receives undefined

Common Traps

❌ Wrong Belief

"reduce works without return"

✅ Correct

Without return, accumulator becomes undefined

❌ Wrong Belief

"map passes one argument to callback"

✅ Correct

map passes (element, index, array) — this is why parseInt breaks

❌ Wrong Belief

"await works in forEach"

✅ Correct

forEach ignores async; use for...of for sequential awaits

❌ Wrong Belief

"setTimeout(0) runs immediately"

✅ Correct

Microtasks (Promise callbacks) always run before macrotasks

❌ Wrong Belief

"Spread creates a deep copy"

✅ Correct

Shallow copy only — nested objects are shared references
