Evolution of Modules

Three generations of "how do I borrow code from another file," each solving the last one's problem.

CommonJS

Server-side (Node.js). Loads synchronously — you wait right there until it's ready.

Like ordering at a local counter: you stand and wait until the order is handed to you before doing anything else.

AMD

Client-side, asynchronous — built for browsers that can't afford to freeze and wait.

Like ordering delivery: you place the order and keep doing your day, no standing around required.

ES6 Modules

The modern standard — asynchronous, and works in both browser and server.

Like a universal courier service that delivers whether you're at home or the office — one system, everywhere.

Exporting & Importing

export / import

Expose specific values from a file; pull them into another.

Real Life

An author publishing one chapter publicly (export), while another book's footnotes reference and borrow that exact chapter (import).

Analogy

A default export is like the book's title chapter — there's only one, and it's what you get if you don't ask for anything specific by name.

Renaming Imports

Real Life

Giving a nickname to a borrowed tool so it doesn't get confused with a tool you already own with the same name.

Analogy

import { add as sum } is exactly that — same borrowed tool, relabeled so it fits cleanly into your own toolbox.

Read-Only Imports

Real Life

A library book — you can read every page, but you can't scribble edits directly into the original copy.

Analogy

To actually change something, you have to ask the library itself (call a function the module exports) to make the edit on your behalf.

NPM + Bundlers

A package registry, plus tools (Webpack, Rollup) that package modules for the browser.

Real Life

NPM is a giant warehouse of parts; a bundler is the moving crew that picks only the parts your project actually needs and packs them into one box a delivery truck (the browser) can carry.

Analogy

The warehouse (npm) was built for backend use; bundlers are what make its contents portable enough for the front end.

Objects & this

Objects as Dictionaries

Real Life

A filing folder with labeled tabs — "Name," "Age," "Address" — each tab leading straight to its own piece of paper.

Analogy

An object literal { name: "Ana" } is that folder, built instantly by writing the tabs and their contents in one go.

this in Methods

Real Life

A Swiss Army knife's blade, when unfolded, "knows" it belongs to the same tool as the corkscrew and scissors right next to it.

Analogy

this is how a method reaches sideways into the other properties of the exact object it's attached to.

Prototype-Based Inheritance

If the apprentice doesn't know something, they ask their mentor. If the mentor doesn't know either, they ask their own mentor — the prototype chain.

class Keyword

Real Life

A clean instruction sheet describing the apprentice-mentor relationship — but underneath, the actual process is still one person asking another up the chain.

Analogy

class is syntax sugar over prototypes, not a different mechanism — same chain, tidier paperwork.

Single-Track Inheritance

Real Life

Unlike a family tree with two parents, each apprentice here has exactly one mentor to fall back on — not two mentors merged together.

Analogy

Every object's prototype chain is a single line going up, never a tree branching from two sources at once.

Copying Objects

y = x (Reference)

Two remote controls paired to the exact same TV. Press a button on either one, and the same TV changes — there's only ever been one TV.

{...x} (Shallow Copy)

Photocopying a document. You now hold a second, independent sheet of paper — editing your copy doesn't touch the original.

The Nested-Object Caveat

Real Life

Your photocopy is independent — but if the original page had a sticky note saying "see the folder in the drawer," your photocopy's sticky note points to that same drawer, not a new one.

Analogy

A shallow copy only duplicates the top page — nested objects inside still point back to the original, shared drawer.

Modules are how files lend each other tools without tangling; prototypes are how objects lend each other behavior without a rigid class hierarchy.
