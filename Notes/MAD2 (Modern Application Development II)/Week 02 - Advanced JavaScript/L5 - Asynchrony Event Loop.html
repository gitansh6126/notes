JavaScript Asynchrony

— how one thread fakes doing several things at once —

Continuing from L09 Same function calls as before — now watching exactly how they queue up and return.

The Call Stack

A stack of plates — each function call adds a plate on top. The top plate (most recent call) has to be finished and removed before the one beneath it can continue.

Stack Frames

Real Life

A pile of sticky notes on a desk — each one records exactly what task you were doing and where to pick up once the note above it is done.

Analogy

Finishing a function "pops" its sticky note off, revealing exactly where the function below it left off.

Single-Threaded

Real Life

A single cashier at a checkout counter — there's only one register, one person serving, one customer at a time.

Analogy

The cashier can't split into two people to serve two customers simultaneously — JS can't run two stack frames in parallel either.

Blocking the Thread

A Blocked Call Stack

Real Life

That same cashier stuck doing a slow, manual price check for one customer — the entire line behind them freezes, no matter how many people are waiting.

Analogy

A heavy synchronous loop or file read does the same thing to the call stack — nothing else gets touched, including the UI, until that one task steps aside.

Event Loop & Task Queue

The waiter sends the order to the kitchen and keeps serving other tables. Only once a dish hits the pass window — and the waiter is free — does it get delivered.

Offloading & Rejoining

Real Life

The kitchen (network, disk, timers) works independently of the waiter — the waiter never stands at the stove waiting.

Analogy

A finished async operation waits at the "pass window" (task queue) until the call stack is completely clear — the waiter only grabs it once every other table's been handled.

Callbacks

Callback Function

A function passed in to be run only after a long task completes.

Real Life

Leaving your phone number at a hotel front desk and being told "we'll call you the moment your room is ready" — instead of standing at the desk the whole time.

Analogy

You go do other things (the call stack stays free); the desk "calls back" exactly when the room (the async result) is ready.

Why Anonymous?

Real Life

A quick verbal instruction left with the desk — "call this number when ready" — rather than filing a formal, permanently-named memo for a one-time request.

Analogy

Since the callback is usually only ever used in this one spot, there's little value in giving it a name that gets filed away for reuse elsewhere.

JavaScript never actually multitasks — it just gets very good at handing slow work to someone else, and picking it back up the moment it's free.
