The Fetch API

— the modern courier service for talking to a server —

Continuing from L13 Same tracking number and package — fetch() is what actually places the order.

Fetch vs. XMLHttpRequest

Real Life

A modern courier app with live tracking, versus the old system of calling a dispatch office on a rotary phone and waiting on hold.

Analogy

Both eventually get the package to you — fetch is just the cleaner, Promise-based interface built for how the web actually works today.

The Delivered-But-Wrong Problem

The Promise only tracks whether the truck reached your door — it says nothing about whether the box actually contains what you ordered.

fetch() Only Rejects on Network Errors

Real Life

The truck breaking down or getting lost is the only thing that counts as a failed delivery. If it reaches your door but hands you an empty box, or the wrong item, the delivery still technically "succeeded."

Analogy

A 404 or 500 status is a wrong or empty box, not a lost truck — the Promise resolves fine either way. You must check response.ok or response.status yourself, like reading the packing slip inside.

Unpacking the Response

Different boxes need different tools to open.

.json()

A structured assembly manual — clean, labeled, ready to use as an object.

.text()

A plain handwritten note — no structure, just raw content.

.blob()

A sealed package of physical goods — photos, files, anything binary.

.formData()

A filled-out paper form tucked inside the box.

Configuring the Shipment

Method, Headers, Body

Real Life

Filling out your own shipping label: delivery type (GET pickup vs. POST/PUT drop-off), a customs declaration of what's inside (headers), and the actual contents being shipped (body).

Analogy

Just like flat-packing a chair for JSON transport, sending JSON in the body still means running it through JSON.stringify() first — the customs form has to describe it in the universal format.

Forms & File Uploads via FormData

Real Life

Mailing a package with a completed customs form and the physical items enclosed together in one box — exactly what happens when you upload a file alongside form fields.

Reusable Request Objects

Real Life

A pre-filled shipping label template you keep on hand for repeat orders to the same vendor — reused across shipments instead of filled out from scratch every time.

Best Practices

async/await for Readability

Real Life

Writing "wait for the delivery, then open it" in plain, top-to-bottom order — instead of a tangle of "when it arrives, do this, and if that fails, do that" callbacks.
