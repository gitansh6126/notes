Vue + Fetch Integration

— the house from Lecture 12 finally starts receiving deliveries —

Two Threads Merge The Vue house (L12) meets the delivery truck (L13–14) — this is where they connect.

Fetching in mounted

Why mounted, Specifically

Real Life

The house is finally standing at its address (mounted to the DOM) — only now does it make sense to schedule the welcome-basket delivery.

Analogy

Calling for a delivery before the house exists (in created) risks the truck showing up to an empty lot — mounted guarantees somewhere real to receive it.

Dynamic Visual Feedback

A porch light wired to the delivery status — glowing yellow while the truck is en route, switching the moment the package status changes.

🟡 yellow — truck en route (saving) → 🟢 green — delivered safely (success)
🟡 yellow — truck en route (saving) → 🔴 red — delivery failed (error)

Class Binding = The Porch Light

Real Life

A porch light rigged to change color automatically based on the delivery app's live status.

Analogy

Vue's class binding swaps the icon's CSS class the instant the request's state changes — no one has to manually flip a switch.

try-catch: A Household Safety Net

Why Wrap the Fetch Call

Real Life

If the delivery truck crashes en route, the house shouldn't collapse — the resident should calmly note "delivery failed today" and carry on with the rest of their day.

Analogy

Without try-catch, one failed fetch could crash the whole component — with it, the failure is contained and the app stays usable.

State Management & Reactivity

Data → View, Automatically

Real Life

A smart mailbox that raises its little flag the instant mail lands inside — no one has to walk out and check manually.

Analogy

Assigning fetched data into a reactive messages array is dropping mail into that smart mailbox — Vue raises the "flag" (re-renders the view) automatically.

Watchers: A Different Kind of Trigger

Watching a Specific Value

Real Life

A motion sensor pointed only at the mailbox — it doesn't care about anything else happening on the property, only whether that one mailbox's contents changed.

Analogy

A watcher on a search input fires a new fetch every time that one value changes — like the sensor re-triggering a fresh delivery check the moment the mailbox's contents shift, without waiting for the whole house's lifecycle to run again.

The house doesn't just get built once — it keeps a porch light wired to every incoming delivery, a mailbox that flags itself, and a sensor that reacts the moment something specific changes.
