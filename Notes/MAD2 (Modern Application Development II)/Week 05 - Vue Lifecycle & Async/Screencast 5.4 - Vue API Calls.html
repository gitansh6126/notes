Core Intuition

Think of your Vue component as a single counter at a coffee shop franchise. The customer (user) never walks into the kitchen (backend) directly — they place an order at the counter, the counter calls the kitchen over an intercom (fetch), and while the kitchen is preparing it, the counter shows a status light so the customer isn't left guessing. fetch is that intercom line; it's asynchronous because the kitchen takes time to respond, and the counter (the UI) stays responsive while it waits.

M5

Deep Fundamentals

1. Why was Fetch created?

Before fetch, browsers used XMLHttpRequest — verbose, callback-heavy, and awkward to chain. fetch is a modern, Promise-based replacement built into the browser: no library needed to talk to a server.

2. What problem does it solve in a Vue app?

A Vue component's data() only knows what you put in it. Real apps need data that lives on a server (a database of messages, users, orders). Fetch is the bridge that pulls that server data into Vue's reactive data, or pushes local data out to be saved.

3. What happens internally?

Calling fetch(url) immediately returns a Promise — a placeholder for "a response, eventually." The browser opens a network connection in the background while your JavaScript keeps running. When the server replies, the Promise resolves with a Response object, which you then convert to usable data via .json() (itself another Promise, since parsing the body also takes a tick).

4. How does Vue execute it?

Vue doesn't do anything special to fetch — it's plain JavaScript. Vue's job starts after the data arrives: the moment you assign the result to this.messages, Vue's reactivity system detects the change and re-renders any part of the template that reads messages.

Syntax Anatomy

Breaking down async mounted() { try { ... } catch (e) { ... } }

  • async Marks this function as one that can use await inside it. Without it, await is a syntax error.
  • mounted() The lifecycle hook name Vue looks for — no parameters, called automatically once the component's template is attached to the real DOM.
  • try { } Wraps the risky async code — anything that might throw if the network fails or the server errors out.
  • await fetch(url) Pauses this function (not the whole app) until the network request settles, then unwraps the Promise into a plain Response object.
  • catch (e) Catches a rejected Promise or thrown error from anywhere in the try block — this is where you'd flip a status icon to red.
  • this Inside mounted() (a regular method, not an arrow function), this is bound to the component instance — required to reach this.messages.

Annotated component

export default {
  data() {
    return {
      name: '',          // bound to an <input>
      messages: [],       // filled by the GET request
      statusIcon: 'text-warning' // starts yellow: "idle"
    }
  },
  async mounted() {
    // runs ONCE, right when the shop opens
    try {
      const res = await fetch('/messages');
      const data = await res.json();
      this.messages = data;   // Vue re-renders automatically
    } catch (e) {
      this.statusIcon = 'text-danger'; // red: fetch failed
    }
  },
  methods: {
    async sayHi() {
      // runs every time the customer places an order
      this.statusIcon = 'text-warning'; // yellow: sending
      try {
        await fetch('/messages', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({ name: this.name })
        });
        this.statusIcon = 'text-success'; // green: kitchen confirmed
      } catch (e) {
        this.statusIcon = 'text-danger';   // red: kitchen unreachable
      }
    }
  }
}

Status icon binding in the template

<i class="bi bi-circle-fill" v-bind:class="statusIcon"></i>

v-bind:class merges the base classes with whatever string statusIcon currently holds — Vue swaps the Bootstrap color class the instant the data changes, no manual DOM manipulation needed.

Behind the Scenes

this.messages = data
→
Dependency Tracking
→
Virtual DOM Diff
→
Patch Real DOM

When you assign to a reactive data property, Vue doesn't repaint the whole page. It checks which parts of the template read that property (dependency tracking, set up when the component was created), rebuilds only the affected virtual DOM nodes, diffs them against what's currently rendered, and patches just the changed pixels. This is why swapping statusIcon from yellow to green only touches that one icon, not the entire component.

Common Mistakes

❌ Fetching data inside created() when the template needs a DOM element (e.g. a chart library) that isn't attached yet
✅ Fetch in created() is fine for plain data; use mounted() only when you also need the real DOM to exist
❌ Forgetting await before res.json() — you end up storing a Promise object instead of the actual data
✅ const data = await res.json(); — both the fetch and the body-parsing are async
❌ Using an arrow function for mounted in the Options API — this won't point to the component
✅ Always declare lifecycle hooks and methods as regular functions in the Options API
❌ No try/catch around await fetch — a dropped network connection crashes the async function silently
✅ Wrap every awaited fetch in try/catch and use the catch branch to flip the status icon red

IITM Exam Pattern

Trigger words

"on page load", "as soon as the component appears", "automatically fetch", "status indicator", "changes color", "watch a variable"

Trap words

created() vs mounted() swapped in options; .then() chain mixed incorrectly with async/await in the same snippet; missing await shown as a "trick" answer that looks plausible.

MCQ tricks

Watch for options where this is used inside an arrow function passed as a method — a classic wrong-answer distractor. Also watch for a POST request missing method: 'POST' (defaults silently to GET).

PYQ pattern

"Which lifecycle hook should be used to fetch initial data?" → mounted() (sometimes created() accepted if no DOM dependency is implied). "What does v-bind:class with a data variable achieve?" → dynamic class toggling based on state.

Elimination technique

Cross out any option using beforeCreate/beforeMount for data fetching (data/DOM not ready). Cross out any option calling .json() without an await or .then() before it.

Trap: an option may show mounted: () => { this.messages = ... } — the arrow function breaks this binding. This is wrong even though the fetch logic "looks" correct.
