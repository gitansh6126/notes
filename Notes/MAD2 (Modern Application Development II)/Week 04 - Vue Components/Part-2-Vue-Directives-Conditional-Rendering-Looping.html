Vue Directives, Conditional Rendering & Looping

Directives are special reactive attributes prefixed with v- that apply special behaviors to the rendered DOM.

1 DIRECTIVES OVERVIEW
🎯 Definition

A directive is a special token in the markup that tells Vue to do something to a DOM element. Directives are prefixed with v- to indicate they are Vue-specific attributes. They apply reactive behavior to the rendered DOM — binding data, listening to events, conditionally rendering, or looping.

2 v-bind — ONE-WAY DATA BINDING
🎯 Definition

v-bind synchronizes data from the model to the view. Updating the JavaScript variable automatically updates the display, but changes in the display do not modify the variable. It's a one-way data flow: data → DOM.

💻 Syntax
<!-- Full syntax -->
<img v-bind:src="imageUrl">

<!-- Shorthand (:) -->
<img :src="imageUrl">
<a :href="link.url">{{ link.text }}</a>
<button :disabled="isSubmitting">Submit</button>
🏠 Real-Life Analogy
Analogy — TV Screen
Your TV displays whatever channel the cable box sends. Changing the channel on the cable box updates the TV. But pressing buttons on the TV screen itself doesn't change the cable box. The TV is bound one-way from the source.
💼 Industry Example
Industry Example
LinkedIn profile page: The profile photo <img :src="user.avatarUrl"> is bound to the user data. When a user uploads a new photo, the server updates the avatar URL, and Vue re-renders the image automatically. The image tag never needs manual reassignment.
⚠ Common Mistakes
Common Mistake
Forgetting the quotes or using wrong syntax. :src="imageUrl" (correct) vs :src="imageUrl" without quotes (correct) vs :src={{ imageUrl }} (wrong — don't use {{ }} with directives).
💡 When to Use
When to Use
Use v-bind (or :) for any HTML attribute that needs to be dynamic: src, href, class, style, disabled, title, alt, custom component props, etc.
3 v-model — TWO-WAY DATA BINDING
🎯 Definition

v-model creates a bidirectional relationship between the data model and a form element. Changing the UI input updates the underlying state, and updating the state updates the UI input. It's syntactic sugar for :value + @input.

💻 Syntax
<!-- Text input — syncs both ways -->
<input v-model="username" placeholder="Enter name">
<p>Hello, {{ username }}</p>

<!-- Checkbox — binds to boolean -->
<input type="checkbox" v-model="isAgreed">
<button :disabled="!isAgreed">Continue</button>

<!-- Select dropdown -->
<select v-model="selectedCity">
  <option v-for="city in cities" :value="city.code">{{ city.name }}</option>
</select>
🏠 Real-Life Analogy
Analogy — Two-Way Mirror
A two-way mirror lets both sides see each other. When you change expression (data), the mirror (view) reflects it. When you press on the mirror (view), the other side sees it (data). v-model is that two-way connection between form inputs and your data.
💼 Industry Example
Industry Example
Google Docs title field: The document title is an editable input. When you type, the title updates in the browser tab and the auto-save draft. Conversely, if the auto-save restores a previous title, the input field updates to match — classic two-way binding.
⚠ Common Mistakes
Common Mistake
Using v-model on non-form elements. v-model is designed for <input>, <textarea>, <select>, and custom components that emit input events. Don't use it on <div> or <p> — use v-bind instead.
💡 When to Use
When to Use
Use v-model for form inputs where the user can modify data and you need the state to reflect those changes immediately — text fields, checkboxes, radio buttons, selects, textareas.
4 v-on — EVENT BINDING
🎯 Definition

v-on listens to DOM events (clicks, keypresses, form submissions) and triggers specified JavaScript methods or expressions when the event occurs. It's how Vue handles user interaction.

💻 Syntax
<!-- Full syntax -->
<button v-on:click="handleSubmit">Submit</button>

<!-- Shorthand (@) -->
<button @click="handleSubmit">Submit</button>
<input @keyup.enter="search">
<form @submit.prevent="onSubmit">...</form>
🏠 Real-Life Analogy
Analogy — Doorbell
A doorbell has a button (DOM event) and a chime (event handler). You wire the button to the chime once. Whenever someone presses the button (click event), the chime rings (handler executes). @click="ringBell" is exactly that wiring.
💼 Industry Example
Industry Example
Twitter's "Like" button: @click="toggleLike" fires an API call, updates the heart icon, and increments the counter. The click handler may also debounce to prevent double-clicks and handle optimistic UI updates before the server confirms.
⚠ Common Mistakes
Common Mistake
Calling the function instead of referencing it. @click="handleSubmit()" (calls immediately on render) vs @click="handleSubmit" (calls on click). Only use parentheses if you need to pass arguments: @click="deleteItem(item.id)".
💡 When to Use
When to Use
Use v-on (or @) for all user interactions — clicks, keypresses, mouse events, form submissions, scroll events, custom component events.
5 CONDITIONAL RENDERING — v-if vs v-show
🎯 Definition

v-if and v-show both conditionally toggle elements based on a boolean expression, but their underlying mechanisms are fundamentally different.

💻 Syntax
<!-- v-if with v-else-if and v-else -->
<div v-if="status === 'loading'">Loading...</div>
<div v-else-if="status === 'error'">Error: {{ errorMsg }}</div>
<div v-else>Data loaded: {{ data }}</div>

<!-- v-show — always rendered, just hidden -->
<div v-show="isVisible">Quick toggle content</div>

🏠 Real-Life Analogy
Analogy
v-if = Renting an apartment: If you don't need it, you leave it empty (destroy). When you need it, you furnish it (create). Costly to set up each time, but no overhead when not in use.
v-show = Closing the blinds: The furniture stays in the room, you just hide it from view. Cheap to toggle, but the room always exists.
💼 Industry Example
Industry Example
YouTube's comments section: The "Show more replies" button toggles a thread with v-if because it's a heavy section that loads on demand. The "Like" animation uses v-show because it toggles rapidly and the element is lightweight.
⚠ Common Mistakes
Common Mistake
Using v-if on frequently toggled elements. If an element toggles many times per second (dropdown menus, tooltips, modals), v-show is far more performant. v-if's DOM destruction/creation will cause jank.
6 LOOP RENDERING — v-for
🎯 Definition

v-for iterates through any standard JavaScript iterable — Arrays, Objects, or Strings — rendering a template block for each item.

💻 Syntax — Arrays
<!-- Basic array -->
<li v-for="item in items" :key="item.id">{{ item }}</li>

<!-- With index -->
<li v-for="(item, index) in items" :key="index">
  {{ index }}: {{ item }}
</li>
💻 Syntax — Objects
<!-- Object values only -->
<li v-for="value in obj">{{ value }}</li>

<!-- Value + key -->
<li v-for="(value, key) in obj">{{ key }}: {{ value }}</li>

<!-- Value + key + index -->
<li v-for="(value, key, index) in obj">
  {{ index }}. {{ key }} = {{ value }}
</li>
🏠 Real-Life Analogy — Shopping Cart
Analogy — Shopping Cart
Your shopping cart has multiple items. Each item has the same layout: product image, name, price, quantity selector. Instead of writing HTML for each item individually, you use v-for to stamp out one row template per cart item. If a new item is added, a new row appears automatically.
💼 Industry Example
Industry Example
Amazon product listing: The search results page uses v-for="product in products" :key="product.asin" to render hundreds of product cards. Each card is identical in structure, only the data differs. The :key (ASIN — Amazon's unique product ID) lets Vue efficiently update only the cards whose data changed.
🔑 Why Keys Matter — Virtual DOM Deep Dive
Why :key is Critical
Every time a list updates, Vue generates new Virtual DOM representations. The diffing algorithm compares old and new Virtual DOM trees to determine minimal real DOM changes. Without :key, Vue uses a heuristic (comparing by index) which fails when items reorder. With :key, Vue can identify each element uniquely and move/reuse DOM nodes instead of destroying and recreating them. This preserves component state (e.g., input values in forms) and boosts performance.
⚠ Common Mistakes
Common Mistake
Using index as key for sortable lists. When the array reorders, index-based keys cause Vue to reuse DOM nodes incorrectly — input values, focus states, and scroll positions get mapped to the wrong items. Always use a unique, stable identifier like item.id when the list can be reordered.
Common Mistake
Omitting :key entirely. Vue warns you in development mode. Without keys, Vue falls back to an inefficient in-place patch strategy that can cause subtle bugs and poor performance.
7 v-text & v-html
🎯 Definition

v-text updates the element's textContent (same as {{ }} but applied as an attribute). v-html updates the element's innerHTML — it renders raw HTML, which requires caution due to XSS risks.

💻 Syntax
<!-- v-text — same result as {{ }} -->
<p v-text="message"></p>
<p>{{ message }}</p>  <!-- equivalent -->

<!-- v-html — renders raw HTML (CAUTION!) -->
<div v-html="rawHtml"></div>
⚠ Common Mistakes
Common Mistake
Using v-html with user-generated content. v-html bypasses Vue's XSS sanitization. If the content comes from user input, comments, or any untrusted source, you risk cross-site scripting attacks. Use v-text or {{ }} whenever possible.
💡 Best Practice
Best Practice
Prefer {{ }} over v-text for readability ({{ }} is more familiar to developers). Only use v-html when you explicitly trust the HTML source (e.g., rich text from an internal CMS with sanitized output).
8 REVISION CHEAT SHEET
📋 Key Takeaways
  • v-bind (:) — One-way data → DOM. Use for dynamic HTML attributes.
  • v-model — Two-way binding for form inputs. Sugar for :value + @input.
  • v-on (@) — Event listeners. Use .prevent, .stop, .enter modifiers.
  • v-if destroys/creates DOM; v-show toggles display. Choose based on toggle frequency.
  • v-for iterates arrays and objects. Always provide :key with a unique identifier.
  • v-text = {{ }}. v-html renders raw HTML — avoid with untrusted content.
  • Shorthands: : for v-bind, @ for v-on.
❓ Interview Question
Q: What is the difference between v-if and v-show, and when would you use each?
A: v-if conditionally creates or destroys DOM elements — it has higher toggle cost but lower initial cost if condition is false. v-show always renders the element and toggles CSS display — lower toggle cost but higher initial render cost. Use v-if for rarely-changed, heavy sections (e.g., tab panels, modals). Use v-show for frequently-toggled, lightweight elements (e.g., dropdowns, tooltips).
Q: Why is the :key attribute important in v-for?
A: :key gives Vue a unique identity for each element in a list. During Virtual DOM diffing, Vue uses keys to track which elements were added, removed, or moved. Without keys, Vue uses an index-based heuristic that fails during reordering — causing incorrect state mapping, ghost input values, and performance degradation.
