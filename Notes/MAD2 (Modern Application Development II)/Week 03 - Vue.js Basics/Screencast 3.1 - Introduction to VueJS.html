2Core Intuition

Why Vue?

In traditional JavaScript development, you write manual DOM update statements every time your state changes. Vue flips this process: you bind your data properties to the HTML template, and Vue watches for changes and handles updates automatically.

Analogy Vanilla JS is like drawing a flipbook by hand: if you change one frame, you must redraw all affected frames manually. Vue is like a 3D rendering engine: you change the data variables (coordinates, colors), and the engine draws the frames for you automatically.

5Deep Fundamentals

Why Declarative Rendering Works

1. Why was Vue created? Vue was built to resolve complex, error-prone manual DOM updating in large-scale single-page apps.

2. What problem does it solve? The decoupling of application logic (JS) from visual representations (DOM). State consistency issues are fully avoided.

3. What happens internally? Vue replaces traditional properties with Getters and Setters (using reactive trackers) to track precisely which components depend on which pieces of state.

- new Vue: Instantiate the core Vue framework constructor.
- el: Specifies the target HTML mounting root selector.
- data(): A function returning reactive state elements bound to the template.
8Code Walkthrough

A Basic Reactive Counter

index.html
<!-- Target Mounting Node -->
<div id="app">
  <p>Count: {{ count }}</p>
  <button @click="increment">Add 1</button>
</div>

<script src="https://cdn.jsdelivr.net/npm/vue@2.6.14/dist/vue.js"></script>
<script>
  new Vue({
    el: '#app',
    data: { count: 0 },
    methods: {
      increment() { this.count++ }
    }
  })
</script>
9Behind the Scenes

The Reactivity Cycle

Under the hood, Vue loops through the properties returned in your data block and wraps them using JavaScript's Object.defineProperty (in Vue 2) or Proxies (in Vue 3). When properties are accessed during template render, Vue records them as "dependencies". When they change, a re-render is triggered automatically.

11Common Mistakes

Anti-Patterns to Avoid

❌
Wrong: Adding new properties directly without declaring them in the data object. 
this.newProp = 'Won\'t be reactive';
✅
Correct: Pre-initialize all data variables inside data, even if they start as null/empty.
12Debugging Mindset

Vue Warnings & Solutions

Error MessageRoot CauseImmediate Fix
"Property or method is not defined on the instance..."Misspelled data property or forgot to add it under data/methods.Check capitalization and confirm its existence in the instantiation script.
"Cannot find element: #app"Vue loaded and executed before the target HTML node was parsed by the browser.Place the script tag at the bottom of the body, or load it inside a DOMContentLoaded event.
13IITM Exam Pattern

Exam Survival Guide

  • Trap Words: "Imperative Vue" (Vue is strictly declarative), "dynamic reactivity injection" (Vue 2 cannot track property addition directly without Vue.set).
  • MCQ Tricks: Watch for arrow functions used in methods where this resolves to window or undefined instead of the Vue instance!
14Interview Questions
Easy
What does declarative rendering mean?
Reveal Answer
It means describing *what* the UI should look like based on current state, leaving the step-by-step DOM updates (*how*) to the framework.
Medium
Why can't Vue automatically track properties added to data after initialization?
Reveal Answer
Because Vue 2 uses `Object.defineProperty` under the hood during the startup phase. It cannot hook into properties that did not exist during that initial conversion pass.
