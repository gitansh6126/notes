Vue uses a declarative approach: you describe what the UI should look like as a function of state, and Vue figures out how to update the DOM when state changes. In Vanilla JS, you write imperative code describing how to update the DOM step by step.

03

Reactive Programming

Reactivity is the core superpower of Vue. When your data changes, the DOM updates automatically. You never need to write textContent = ... or classList.add(...) again.

⚡ How Vue's Reactivity Works

🧠 Reactive vs Non-Reactive

Vanilla JS (non-reactive)JavaScript
let board = ['', '', ''];
board[1] = 'X';
// DOM is still empty! You must write:
cells[1].textContent = 'X';
Vue JS (reactive)Vue
// Template: {{ board[1] }}  // auto-updates!

data() {
  return { board: ['', '', ''] }
},
methods: {
  makeMove(i) {
    this.board[i] = 'X';
    // DOM updates automatically! No manual sync needed.
  }
}
💡
Real-Life Analogy: A Spreadsheet

In Excel, when you change a cell value (A1 = 10), every formula that references A1 recalculates automatically. Vue's reactivity works the same way: change your data, and every part of the template that depends on that data updates automatically. You never recalculate by hand.

One-Way vs Two-Way Binding

v-bind (:) is one-way: data flows from state to the template. v-model is two-way: changes in the input update the state and vice versa. For Tic Tac Toe, we only need one-way binding because the game board is display-only (users click buttons, not type into inputs).

Complete Vue template for Tic Tac ToeVue template
<div id="app">
  <h1>Tic Tac Toe</h1>

  <!-- Status -->
  <p class="status">{{ statusMessage }}</p>

  <!-- Board -->
  <div class="board">
    <button
      v-for="(cell, i) in board"
      :key="i"

Shorthand Reminder

: is shorthand for v-bind: (attribute binding). @ is shorthand for v-on: (event binding). So :class="..." = v-bind:class="..." and @click="..." = v-on:click="...".

07

Automatic UI Updates

The biggest difference between Vanilla JS and Vue: Vue handles DOM updates for you. You change state; Vue recalculates what changed and patches the DOM.

⚙ How Automatic Updates Work

📝 Before & After: The Same Move

Vanilla JSJS
function handleClick(i) {
  if (gameOver || board[i]) return;
  board[i] = turn;
  cells[i].textContent = turn;
  cells[i].classList.add('taken');

  if (checkWin(turn)) {
    status.textContent = 'Player ' + turn + ' wins!';
    gameOver = true;
  } else {
    turn = turn === 'X' ? 'O' : 'X';
    status.textContent = 'Player ' + turn + "'s turn";
  }
}
Vue.jsVue
methods: {
  makeMove(i) {
    if (this.gameOver || this.board[i]) return;
    this.board[i] = this.turn;

    if (this.checkWin(this.turn)) {
      this.winner = this.turn;
      this.gameOver = true;
    } else {
      this.turn = this.turn === 'X' ? 'O' : 'X';
    }
    // No DOM code! Template handles everything.
  }
}
⚠
But You Still Need Game Logic!

Vue only handles the DOM updates. You still write the game logic (win check, turn toggle, draw detection) yourself. Vue is not magic — it just eliminates the tedious, error-prone textContent and classList boilerplate.

Complete Vue Tic Tac Toe

The entire game in one HTML file using Vue 3 CDN. Notice: no DOM manipulation anywhere.

Complete file (runnable)HTML
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<div id="app">
  <h2>Tic Tac Toe (Vue)</h2>
  <p class="status">{{ status }}</p>

  <div class="board">
    <button
      v-for="(cell, i) in board"
      :key="i"
      class="cell"
      :class="{ taken: cell, x: cell==='X', o: cell==='O' }"
      @click="makeMove(i)"
    >{{ cell }}</button>
  </div>

  <button v-if="gameOver" @click="resetGame">
    Play Again
  </button>
</div>

<script>
const { createApp } = Vue;

createApp({
  data() {
    return {
      board: ['','','','','','','','',''],
      turn: 'X',
      winner: null,
      gameOver: false
    }
  },
  computed: {
    status() {
      if (this.winner) return `Player ${this.winner} wins!`;
      if (this.gameOver) return "It's a draw!";
      return `Player ${this.turn}'s turn`;
    }
  },
  methods: {
    makeMove(i) {
      if (this.gameOver || this.board[i]) return;
      this.board[i] = this.turn;
      if (this.checkWin(this.turn)) {
        this.winner = this.turn;
        this.gameOver = true;
      } else if (this.board.every(c => c)) {
        this.gameOver = true;
      } else {
        this.turn = this.turn === 'X' ? 'O' : 'X';
      }
    },
    checkWin(player) {
      const wins = [[0,1,2],[3,4,5],[6,7,8],
                    [0,3,6],[1,4,7],[2,5,8],
                    [0,4,8],[2,4,6]];
      return wins.some(p => p.every(i => this.board[i] === player));
    },
    resetGame() {
      this.board = ['','','','','','','','',''];
      this.turn = 'X';
      this.winner = null;
      this.gameOver = false;
    }
  }
}).mount('#app');
</script>

<style>
.board { display: grid; grid-template-columns: repeat(3, 80px); gap: 4px; }
.cell { width: 80px; height: 80px; font-size: 2rem; }
.taken { cursor: default; background: #eef; }
.x { color: #a8482a; } .o { color: #2c6b5d; }
</style>
💡
Notice: No DOM Code

Compare this with the Vanilla JS version. There are zero document.querySelector, zero .textContent, zero .classList calls. Vue's template bindings ({{ }}, :class, @click) replace all of them. The computed property status even updates automatically when winner, gameOver, or turn change.

Advantages of Using Vue

✅ Advantages

  • No manual DOM manipulation — Vue handles all DOM updates through reactivity.
  • Declarative templates — you describe the UI as a function of state, not as a sequence of DOM commands.
  • Reactivity built in — change this.board[i] and the cell updates automatically.
  • Directives (v-for, v-if, :class) make templates expressive and concise.
  • Computed properties derive values reactively (our status message).
  • Scoped CSS (in SFCs) prevents style conflicts.
  • Progressive — use a little (CDN) or a lot (full SFC build).

❌ Limitations

  • Learning curve — new concepts: directives, reactivity, virtual DOM, lifecycle.
  • Extra dependency — the Vue library must be loaded (~40 KB min+gzip).
  • Abstraction overhead — harder to debug when you do not understand the underlying DOM.
  • Not magic — you still write all the game logic (win check, turn management).
  • SEO challenges — vanilla Vue renders in the browser, so search engines see an empty page (Nuxt fixes this).
  • State lost on refresh — same as Vanilla JS (need localStorage or a backend).
ℹ
Why Use Vue for Tic Tac Toe?

Vue is overkill for a 9-cell game, but it is a perfect learning tool. The same patterns scale to real applications: reactive state, declarative templates, event handling, computed properties. Once you understand these, you can build anything from a game to a dashboard to an e-commerce site.

Common Mistakes

✖ Mistake 1: Forgetting this. in methods

WrongVue
methods: {
  makeMove(i) {
    if (gameOver || board[i]) return;  // ✖ ReferenceError!
  }
}

Fix: Always prefix data and methods with this. inside Vue's options: this.gameOver, this.board[i], this.turn.

✖ Mistake 2: Mutating Arrays in Ways Vue Cannot Track

WrongVue
this.board[3] = 'X';          // ✖ Vue 3 CAN track this (Proxy)

// But in Vue 2, this would NOT be reactive:
// this.board[3] = 'X';  // Vue 2 misses array index assignment!
// Use Vue.set(this.board, 3, 'X') in Vue 2

Note: Vue 3 uses Proxy and detects all mutations. Vue 2 uses Object.defineProperty and cannot detect array index assignments. Know which version you are using.

✖ Mistake 3: Not Using :key in v-for

Without :key, Vue may reuse DOM elements incorrectly when the list changes. Always provide a unique :key with v-for. For our static 9-cell board it matters less, but it is a good habit.

✖ Mistake 4: Async Code in Computed Properties

Computed properties must be synchronous and pure (no side effects). Putting an API call or setTimeout inside a computed property will break Vue's reactivity system. Use methods or watch for async operations.

✖ Mistake 5: Overcomplicating with Vue when Vanilla JS Would Suffice

Not every interactive element needs a framework. A simple toggle button, a tooltip, or a small form might be simpler with plain JS. Vue shines when you have multiple pieces of state that affect multiple DOM elements — exactly the Tic Tac Toe scenario with 9 cells + status + game-over state.

Interview & Exam Questions

Q1: What is Vue's reactivity system and how does it differ from Vanilla JS state management?+

Vue's reactivity system wraps your data() object in a Proxy (Vue 3) that intercepts get/set operations. When you read a property in a template, Vue records that the template depends on that property. When you set the property, Vue notifies all dependents and re-renders only the affected parts of the DOM. In Vanilla JS, you must manually update the DOM after every state change with .textContent and .classList.

Q2: Explain the difference between v-if and v-show.+

v-if conditionally renders the element (adds/removes from DOM). v-show always renders but toggles display: none. Use v-if for rare toggles (our "Play Again" button) since it avoids creating the DOM node until needed. Use v-show for frequent toggling (like a dropdown) since the element is always in the DOM.

Q3: What is the purpose of :key in v-for?+

:key gives Vue a unique identity for each element in a list. When the list changes (items reorder, added, removed), Vue uses the key to match old and new elements efficiently instead of re-rendering everything. Without keys, Vue's default "in-place patch" strategy can cause bugs with stateful elements like <input> fields.

Q4: How does v-bind (:) differ from interpolation {{ }}?+

{{ }} inserts text content into an element. v-bind: (shorthand :) binds a JavaScript expression to an HTML attribute (like class, src, href, or custom attributes). Example: :class="{ active: isActive }" binds the class attribute; {{ isActive }} would display the text "true" or "false".

Q5: What is a computed property and when would you use one?+

A computed property derives a value from other reactive state. It caches its result and only re-evaluates when its dependencies change. In Tic Tac Toe, status is computed from winner, gameOver, and turn. Computed properties are read-only by convention, synchronous, and pure (no side effects).

Q6: Compare the developer experience of building Tic Tac Toe in Vanilla JS vs Vue.+

Vanilla JS: You have complete control but must manually select elements, update their content, add/remove classes, and manage events. Every DOM operation is explicit. Good for learning how the browser works. Vue: You write declarative templates and focus on state and logic. Vue handles all DOM synchronization. Less boilerplate, fewer bugs from forgotten updates, but you must learn Vue's concepts (directives, reactivity, computed properties).

Q7: How does Vue 3's Proxy-based reactivity differ from Vue 2's Object.defineProperty?+

Vue 2 (Object.defineProperty): Could not detect new properties added to objects or direct index assignment on arrays (arr[3] = val). Required Vue.set() for such cases. Vue 3 (Proxy): Can intercept any property access, addition, deletion, and array index assignment. No special workarounds needed. Proxy also allows Vue to handle Map, Set, and other built-in types natively.

Q8: What happens when you call createApp() and .mount()?+

createApp() creates a Vue application instance with the given options object (data, methods, computed, etc.). It initialises the reactivity system but does not touch the DOM yet. .mount('#app') tells Vue to take control of the #app DOM element, compile its template, create the virtual DOM, render the initial DOM, and attach event listeners. After mounting, Vue's reactivity system keeps the DOM in sync with state.

Q9: How would you persist the Tic Tac Toe game state across page refreshes in Vue?+

Use the mounted lifecycle hook to load state from localStorage, and use a watch or save manually in makeMove to persist state whenever it changes:

localStorage persistenceVue
mounted() {
  const saved = localStorage.getItem('tictactoe');
  if (saved) Object.assign(this, JSON.parse(saved));
},
methods: {
  saveState() {
    localStorage.setItem('tictactoe', JSON.stringify({
      board: this.board, turn: this.turn,
      winner: this.winner, gameOver: this.gameOver
    }));
  }
}
Q10: What is the virtual DOM and how does Vue use it?+

The virtual DOM is a lightweight JavaScript object representation of the real DOM. When state changes, Vue creates a new virtual DOM tree, compares (diffs) it with the previous one, calculates the minimum set of changes needed, and applies only those changes to the real DOM. This is far more efficient than rebuilding the entire DOM. Vue's reactivity system triggers this process automatically when data changes.

Key Takeaways

  • Vue's reactivity automatically syncs the DOM with your state. You change data; Vue updates the UI.
  • Declarative templates ({{ }}, directives) replace imperative DOM manipulation.
  • Directives (v-for, :class, @click) are the bridge between state and DOM.
  • Computed properties derive values reactively and cache results for performance.
  • The virtual DOM makes updates efficient by calculating only what changed before touching the real DOM.
  • Vue eliminates the manual document.querySelector + textContent + classList pattern.
  • But you still write game logic — Vue handles UI, not business logic.
  • Vue 3 uses Proxy for reactivity, catching all property changes (including array index assignment).
  • State is lost on refresh — the same limitation as Vanilla JS. Use localStorage or a backend for persistence.
  • Vue scales from CDN script to full build system. The concepts you learn here apply to all Vue applications.

Summary

📖 Lecture at a Glance

The Vue.js Tic Tac Toe demo introduces reactive programming for the first time. State lives in data() and is wrapped in Vue's Proxy-based reactivity system. You write declarative templates with directives like v-for, :class, and @click. When state changes, Vue automatically determines which parts of the DOM need updating and patches them efficiently through its virtual DOM.

  • Reactivity — state changes automatically trigger DOM updates via a Proxy-based system.
  • Data binding — {{ }}, :attr, @event connect state to the template.
  • Directives — v-for loops over arrays, :class binds CSS, @click handles events.
  • Computed properties — derive values reactively (status message based on game state).
  • Lifecycle — created (data ready), mounted (DOM ready), beforeUnmount (cleanup).
  • No manual DOM — zero textContent, zero classList, zero addEventListener.
  • Same game logic — win detection, turn toggling, draw checking are unchanged from Vanilla JS.
  • Vue is progressive — start with a CDN script, scale up to a full SFC build system.

Glossary

Computed PropertyA derived value that depends on reactive state and updates automatically. Cached until dependencies change.
Data BindingConnecting JavaScript state to HTML template elements using {{ }}, :attr, or v-model.
DirectiveA special v- prefixed attribute in Vue templates that applies reactive behaviour to the DOM (v-for, v-if, :class).
Lifecycle HookA function you can define (created, mounted, beforeUnmount) that Vue calls at specific points in a component's life.
ProxyA JavaScript object wrapper that can intercept fundamental operations (get, set, delete). Vue 3 uses Proxy for reactivity.
ReactivityThe ability of a system to automatically react to data changes. In Vue, changing state triggers DOM updates.
Reactive StateData properties returned from data() that Vue wraps in a reactive Proxy. Changes to these properties trigger re-renders.
TemplateThe HTML-like markup in a Vue component that declares how the UI should look as a function of state.
v-bind (:)A directive that binds a JavaScript expression to an HTML attribute. Shorthand is : (colon).
v-forA directive for rendering a list of items by iterating over an array. Always pair with :key.
v-if / v-showv-if conditionally renders an element; v-show toggles display: none. v-if is lazy; v-show is always rendered.
v-on (@)A directive that listens for DOM events. Shorthand is @. Example: @click="handler".
Virtual DOMA lightweight JavaScript representation of the DOM. Vue diffs virtual DOM trees to minimise real DOM operations.
