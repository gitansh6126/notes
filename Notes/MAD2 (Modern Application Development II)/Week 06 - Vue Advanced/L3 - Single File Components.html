2Core Intuition · WHY

Why this exists

🗂️
Imagine cooking from recipe cards scattered across the kitchen — one card for ingredients pinned to the fridge, the steps scrawled on a napkin, and a photo of the finished plate somewhere in your phone. That's a component built with global registration, string templates, and global CSS. An SFC is a single labeled recipe page: ingredients, steps, and photo, all on the same sheet, filed neatly in a binder — nothing gets misplaced, nothing bleeds into the next recipe.

As an app grows past a handful of components, that scattering becomes the actual bottleneck — not the logic itself, but finding and safely changing it without breaking something else.

5Deep Fundamentals

The four questions

1. Why was this feature created?

As Vue apps grew past a few components, global registration (Vue.component(...)), inline string templates, and global CSS became genuinely unmanageable — name collisions, unreadable templates, and styles bleeding across the whole app.

2. What problem does it solve?

Modularity and encapsulation: every component's HTML, logic, and CSS live together in one file, scoped to that component alone, so it can be reasoned about, edited, and deleted in isolation.

3. What happens internally?

A build tool parses the .vue file's three blocks separately: the template is compiled into a render function, the script is treated as an ES6 module, and scoped styles get rewritten with a unique attribute selector so they can never leak out.

4. How does Vue/JavaScript execute it?

The bundler resolves every import/export across all your .vue files into a dependency graph, then packages it all into plain JS bundles (plus HTML/CSS) that any browser can execute — no special Vue-aware runtime needed in the browser itself.

7Syntax Anatomy

Every block, explained

<template>
  <button @click="count++">{{ count }}</button>
</template>

<script>
export default {
  data() { return { count: 0 } }
}
</script>

<style scoped>
button { color: teal; }
</style>
<template> exactly one root-friendly HTML block — the component's markup
<script> an ES6 module — export default is how this component's options object is shared with whatever imports it
<style scoped> the scoped attribute is what tells the compiler to lock these rules to this component only
8Code Walkthrough

A tiny SFC, imported and used

// Counter.vue
<template>
  <button @click="increment">Count: {{ count }}</button>
</template>

<script>
export default {
  name: 'Counter',
  data() {
    return { count: 0 }
  },
  methods: {
    increment() { this.count++ }
  }
}
</script>

<style scoped>
button { background: #2DD4BF; }
</style>
// App.vue
<template>
  <Counter />
</template>

<script>
import Counter from './Counter.vue'

export default {
  components: { Counter }
}
</script>
✅
No Vue.component('Counter', ...) registered globally, no chance of another file accidentally defining a component with the same name — Counter only exists where it's explicitly imported.
9Behind the Scenes

What the build tool is really doing

How "scoped" actually works

The compiler adds a unique attribute — something like data-v-7ba5bd90 — to every element the template renders, and rewrites every selector in that <style> block to include the same attribute. The result: identical CSS class names in two different components simply cannot collide.

Bundling & the dependency graph

The bundler starts from your entry file, follows every import statement it finds, and builds a full dependency graph — then packages everything actually used into as few files as possible, often removing unused code entirely (tree-shaking).

⚠️
Supply chain risk: every npm install pulls in code you didn't write and likely haven't read — a compromised or malicious package deep in your dependency tree runs with the same trust as your own code.
10Real Project Usage

Where you've already used this

GitLab

Its web frontend is built from hundreds of Vue SFCs, bundled with Webpack, so teams can own individual components without stepping on each other's CSS.

Nintendo & large e-commerce sites

Large storefronts use SFCs so a "Product Card" component's styling never accidentally affects a "Cart" component elsewhere on the page.

The 2021 "ua-parser-js" incident

A widely-used npm package was compromised and briefly served malware to anyone who installed it — a textbook real-world supply chain attack through a dependency almost no one had audited directly.

Vite-powered dashboards

Modern internal admin tools favor Vite specifically for its near-instant dev server startup compared to older Webpack-based setups.

11Common Mistakes

Wrong vs. Right

❌ WrongForgetting scoped on <style>, letting rules apply globally
✅ Correct<style scoped> — styles stay locked to this component
❌ WrongDouble-clicking a .vue file expecting the browser to open it
✅ Correct.vue files only run through a build tool (Vite/Webpack) — never directly in the browser
❌ WrongInstalling any npm package without checking its popularity, maintenance, or audit results
✅ CorrectVet dependencies, keep a lockfile, run npm audit periodically
❌ WrongMixing global Vue.component() registration with SFC imports for no reason
✅ CorrectImport components explicitly where they're used — avoids name collisions entirely

Trap words

Questions may conflate Webpack, Vite, Rollup, and esbuild as if interchangeable — know that they differ in speed and approach, not just name.

MCQ tricks

Watch for options claiming SFCs "run directly in the browser" — false, they always require a build step.

PYQ pattern

Frequently asks to identify what's missing from a broken SFC snippet (usually the scoped keyword or a missing import).

🎯
Elimination technique: any option saying "no build tool is required for .vue files" is automatically wrong.
14Interview Questions

Three tiers

Easy
What are the three blocks inside a Single File Component?
<template> for HTML, <script> for logic, and <style> for CSS, all in one .vue file.
Medium
How does the "scoped" attribute prevent CSS from leaking?
The compiler tags every element the component renders with a unique data attribute, and rewrites the component's CSS selectors to require that same attribute — so identical class names in different components never collide.
Advanced
What is a supply chain attack, and how does it apply to an npm-based Vue project?
It's an attack where malicious code is smuggled in through a trusted third-party dependency rather than the project's own code. Since npm projects can pull in hundreds of nested dependencies, a single compromised package deep in that tree runs with the same privileges as first-party code — making dependency auditing and lockfiles a real security practice, not just hygiene.
