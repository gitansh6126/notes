1What is JavaScript?

The Language of the Web

JavaScript (JS) is a high-level, dynamic, interpreted programming language that conforms to the ECMAScript specification. It is one of the three core technologies of the World Wide Web — alongside HTML and CSS — and is supported by every modern browser.

Key Characteristics

  • Dynamic typing — variables can hold any type; type is determined at runtime
  • First-class functions — functions are treated like any other value (passed as arguments, returned, assigned)
  • Prototype-based inheritance — objects inherit directly from other objects (not classical classes)
  • Event-driven — built around an event loop for asynchronous operations
  • Single-threaded — one call stack, one heap, async via the event loop

2Runtime Environments

Where JavaScript Runs

JavaScript was originally created to run only in browsers. Today it runs almost everywhere.

Browser Runtime

  • Google Chrome — V8 engine
  • Mozilla Firefox — SpiderMonkey engine
  • Apple Safari — JavaScriptCore (Nitro)
  • Microsoft Edge — V8 (Chromium-based)

The browser provides Web APIs: document, console, fetch, setTimeout, DOM APIs, and more.

Server-Side Runtime

  • Node.js — V8 engine + libuv for I/O; file system, networking, process APIs
  • Deno — V8 + Rust-built; secure by default, supports TypeScript natively
  • Bun — JavaScriptCore-based; built-in bundler, test runner, package manager
3Development Setup

Tools of the Trade

Browser Console

Every browser includes DevTools (F12 / Ctrl+Shift+I). The Console tab is a REPL — Read-Eval-Print Loop — where you can type and run JS instantly.

// Open console → paste this → press Enter
console.log("Hello from the browser console!");
console.table({ name: "JS", year: 1995 });

Script Tags in HTML

There are two ways to include JS in an HTML page:

<!-- Internal script (inside HTML) -->
<script>
  console.log("Internal JS");
</script>

<!-- External script (separate .js file) -->
<script src="app.js"></script>
Best Practice
Place <script> tags just before </body> or use the defer attribute to avoid blocking HTML parsing. Always prefer external scripts for production.
4JavaScript vs Other Languages

How JS Compares

Java / C++
Statically typed, class-based OOP, compiled. Verbose. Threads via OS threads.
JavaScript
Dynamically typed, prototype-based, interpreted. Concise. Async via event loop (single-threaded).
Python
Dynamically typed, indent-based, synchronous by default. Used for data/backend/ML.
JavaScript
Curly-brace syntax, async-native, prototype-based. Only language in the browser.
Real-World Analogy
Think of JavaScript as the universal translator of the web. HTML is the skeleton, CSS is the clothes, and JavaScript is the muscles that make everything move and respond. Without JS, web pages would be static documents — with JS, they become applications.

Key Takeaways

  • JS is dynamically typed — flexible but requires discipline
  • Functions are first-class citizens — a core paradigm
  • Prototype-based inheritance is unique to JS
  • JS is single-threaded but non-blocking via the event loop
  • The V8 engine (Chrome/Node/Deno) uses JIT compilation for speed

6Common Pitfalls
Common Mistake
Confusing JavaScript with Java. Despite the similar name, they are entirely different languages. Java is compiled, statically typed, and runs on the JVM. JavaScript is interpreted, dynamically typed, and runs in browsers/Node. The name "JavaScript" was a marketing ploy during the browser wars.
Common Mistake
Blocking the main thread. JS is single-threaded. If you run a long synchronous loop, the entire page freezes. Always use async patterns (callbacks, promises, async/await) for I/O and heavy computation.
Common Mistake
Not using semicolons in ambiguous cases. While JS has automatic semicolon insertion (ASI), it can break on lines starting with (, [, or `. Always use semicolons or learn the few places where ASI fails.

8Interview Q&A
Q: Is JavaScript compiled or interpreted?
A: Traditionally interpreted, but modern engines use JIT (Just-In-Time) compilation. V8 compiles JS to native machine code at runtime. So it's best described as a JIT-compiled language.
Q: What is the difference between JavaScript and ECMAScript?
A: ECMAScript (ES) is the specification / standard. JavaScript is the most popular implementation of that standard. Other implementations include JScript (Microsoft), ActionScript (Adobe). When people say "ES6" they refer to the 6th edition of the ECMAScript specification.
Q: What is the event loop in JavaScript?
A: The event loop is the mechanism that allows JS to perform non-blocking I/O despite being single-threaded. The call stack executes code, the event loop checks if the call stack is empty, and then pushes pending callbacks (from the callback queue) onto the stack. This is the foundation of JavaScript's concurrency model.
Q: What are the different ways to include JavaScript in an HTML page?
A: (1) Inline — directly in HTML attributes (onclick), bad practice. (2) Internal — <script> tag in the HTML body. (3) External — <script src="file.js">, best for production. Use defer or async attributes for non-blocking loading.
