JavaScript Basics

Understanding primitive types, operators, type coercion, and the foundational building blocks of JavaScript. Master the quirks that trip up every beginner.

1Primitive Types

The 7 Primitive Types + BigInt

JavaScript has 7 primitive data types (immutable, not objects) plus BigInt for arbitrary-precision integers. Everything else is an Object.
Type Example typeof Result
string "hello", 'world', `hi` "string"
number 42, 3.14, Infinity, NaN "number"
boolean true, false "boolean"
undefined undefined "undefined"
null null "object" ← historical bug
symbol Symbol("id") "symbol"
bigint 9007199254740991n "bigint" 
// typeof operator
typeof "hello";      // "string"
typeof 42;            // "number"
typeof true;         // "boolean"
typeof undefined;    // "undefined"
typeof null;         // "object"  ← famous bug!
typeof Symbol("id"); // "symbol"
typeof 1n;            // "bigint"
Common Mistake
typeof null === "object" — This is a well-known bug from the first version of JavaScript that cannot be fixed because it would break existing code. Always use value === null to check for null.
Real-World Analogy
Primitive types are like atoms — the smallest indivisible units. Objects are like molecules — composed of atoms and other molecules. You can't change an atom (immutable), but you can replace it. You can change parts of a molecule (mutable).
2Type Coercion

Implicit vs Explicit Conversion

JavaScript is a dynamically typed language, meaning types are converted automatically (implicit coercion) or manually (explicit coercion).

Explicit Conversion (Intentional)

String(123);       // "123"
Number("456");     // 456
Boolean(0);         // false
parseInt("42px");   // 42
parseFloat("3.14"); // 3.14

Implicit Coercion (Surprising)

"5" - 3;     // 2  (string coerced to number)
"5" + 3;     // "53"  (number coerced to string!)
"5" * "2";   // 10  (both coerced to number)
0 == false;  // true  (loose equality coerces)
0 === false; // false  (strict equality, no coercion)
== (Loose Equality)
Coerces types before comparing. Avoid this operator. 0 == "" → true, false == "0" → true.
=== (Strict Equality)
No coercion. Returns false if types differ. Always use === and !== in modern code.
3Truthy & Falsy Values

What Evaluates to true/false?

In a boolean context (if, while, &&, ||), values are coerced to boolean. Only 6 falsy values exist.
Falsy (all 6)
false, 0, "" (empty string), null, undefined, NaN
Truthy (everything else)
"0", "false", [], {}, Infinity, -1, all functions
if ("0") { console.log("truthy"); }  // logs! (non-empty string)
if ([])   { console.log("truthy"); }  // logs! (empty array is truthy)
if (0)    { console.log("truthy"); }  // does NOT log
Common Mistake
Checking array length incorrectly. if (arr) is truthy even for empty arrays. Always check if (arr.length > 0) or if (arr.length).
4Operators

Arithmetic, Comparison, Logical, Assignment

Arithmetic

10 + 3;  // 13  (addition)
10 - 3;  // 7   (subtraction)
10 * 3;  // 30  (multiplication)
10 / 3;  // 3.333... (division → always float)
10 % 3;  // 1   (modulus)
2 ** 3;  // 8   (exponentiation, ES2016)

Logical Operators

// Short-circuit evaluation
true  || anything;  // short-circuits (returns true)
false && anything;  // short-circuits (returns false)

// Common patterns
const name = user.name || "Guest";  // default value (older)
const age  = user.age  ?? 18;       // nullish coalescing (ES2020)
const street = user?.address?.street; // optional chaining (ES2020)

Special Values: NaN

NaN (Not-a-Number) is a peculiar value. It is the only value that is not equal to itself.

typeof NaN;       // "number"  (ironic!)
NaN === NaN;      // false  (the only exception)
isNaN("abc");    // true
Number.isNaN("abc"); // false  (stricter check, doesn't coerce)
5Template Literals & String Methods

Modern String Handling

Template literals (backtick strings from ES6) allow interpolation and multi-line strings. Combined with built-in string methods, they make text manipulation clean and readable. 
const name = "Alice";
const age  = 30;

// Template literal with interpolation
console.log(`My name is ${name} and I am ${age} years old.`);

// Multi-line without \n
const poem = `Roses are red,
Violets are blue,
JavaScript is fun,
And so are you.`;

// Useful string methods
"hello".toUpperCase();          // "HELLO"
"  trim  ".trim();             // "trim"
"abc".repeat(3);               // "abcabcabc"
"hello world".includes("world"); // true
"a,b,c".split(",");             // ["a","b","c"]
"hello".startsWith("he");       // true
"hello".endsWith("lo");         // true
6Industry Context
Industry Insight
Understanding type coercion is not just an academic exercise — it's the source of countless production bugs. A 2022 study of 1,500+ JavaScript repositories found that loose equality (==) was involved in ~12% of all bugs. Linters like ESLint enforce the use of === and !== to eliminate entire categories of bugs. Most tech companies (Google, Meta, Netflix) require strict equality in their style guides.
7Interview Q&A
Q: What is the difference between null and undefined?
A: undefined means a variable has been declared but not assigned a value. null is an intentional assignment meaning "no value" or "empty". Use ?? (nullish coalescing) to handle both cases.
Q: Why is typeof null === "object"?
A: It's a bug from the original 1995 implementation. The type tag for objects was 0, and null's representation (0x00) was incorrectly classified. It can't be fixed due to backward compatibility — thousands of sites rely on this behavior.
Q: What is NaN and how do you check for it?
A: NaN (Not-a-Number) is a numeric value representing an invalid number. Use Number.isNaN(x) for a reliable check (doesn't coerce), or x !== x (NaN is the only value not equal to itself).
Q: What is the difference between == and ===?
A: == performs type coercion before comparison (e.g., 5 == "5" → true). === requires both value and type to be identical (strict equality). Always use ===.
