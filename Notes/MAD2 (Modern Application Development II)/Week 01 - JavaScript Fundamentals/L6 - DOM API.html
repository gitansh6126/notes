DOM API

The Document Object Model (DOM) is the bridge between JavaScript and HTML. Learn how to select, modify, create, and remove elements, and how to handle events that make pages interactive.

1What is the DOM?

The DOM Tree

When the browser loads an HTML page, it creates a tree-like representation of the document called the DOM. Each HTML element becomes a node in this tree. JavaScript can read, modify, add, or delete any node.

The document Object

The entry point to the DOM. Every browser exposes the global document object in JavaScript.

console.log(document);            // the entire HTML document
console.log(document.head);       // <head> element
console.log(document.body);       // <body> element
console.log(document.title);      // the page title (string)
Real-World Analogy
The DOM is like an architectural blueprint of a building. HTML is the raw building material, but the DOM is the detailed plan that shows every wall, door, and window. JavaScript is the construction worker who reads the blueprint, walks through the building, and modifies walls, adds windows, or demolishes rooms.
2Selecting Elements

Finding Nodes in the DOM

Traditional (Legacy)
getElementById("id") — single element
getElementsByClassName("cls") — live HTMLCollection
getElementsByTagName("div") — live HTMLCollection
Modern (Recommended)
querySelector("css-selector") — first match
querySelectorAll("css-selector") — static NodeList
// Modern selectors (pass any CSS selector)
const header = document.querySelector("header");
const nav = document.querySelector(".nav-bar");
const btn = document.querySelector("#submit-btn");
const items = document.querySelectorAll(".list-item");
const firstLi = document.querySelector("ul li:first-child");

// querySelectorAll returns a NodeList (not an array!)
console.log(items.length);          // 3
items.forEach(item => {             // NodeList has forEach
  console.log(item.textContent);
});
const arr = [...items];             // convert to Array if needed
Best Practice
Always prefer querySelector and querySelectorAll. They accept any CSS selector, return static (not live) collections, and are consistent across browsers. The legacy methods (getElementById excepted) have quirks that cause bugs.
3Modifying Content & Attributes

Changing What the User Sees

Text Content

const el = document.querySelector("h1");

el.textContent = "New Title";       // sets text (safe, no HTML parsing)
el.innerHTML = "<em>New</em> Title"; // parses HTML (XSS risk!)
el.innerText = "New Title";         // similar to textContent but slower

Attributes & Styles

// Attributes
el.setAttribute("data-id", "123");
el.getAttribute("data-id");   // "123"
el.hasAttribute("disabled"); // true/false
el.removeAttribute("disabled");

// Direct property access (for standard attributes)
el.id = "main-header";
el.href = "https://example.com";   // for <a>
el.src = "image.jpg";               // for <img>

Class List

const div = document.querySelector(".box");
div.classList.add("active");           // add class
div.classList.remove("hidden");        // remove class
div.classList.toggle("dark-mode");      // toggle on/off
div.classList.contains("active");       // true/false
div.classList.replace("old", "new");   // replace class

Inline Styles

el.style.color = "red";
el.style.backgroundColor = "#333";    // camelCase, not background-color
el.style.fontSize = "16px";
Common Mistake
Using innerHTML with user input. innerHTML parses HTML strings. If you insert user-provided content (e.g., comments, search terms), this creates an XSS vulnerability. Always use textContent for text, and innerHTML only with trusted content or after sanitization.
4Creating & Removing Elements

Dynamic DOM Manipulation

// Create element
const li = document.createElement("li");
li.textContent = "New Item";
li.classList.add("list-item");

// Append to parent
const ul = document.querySelector("ul");
ul.appendChild(li);                 // adds at end
ul.insertBefore(li, ul.firstChild); // adds at beginning

// Modern append/prepend
ul.append(li);                       // at end
ul.prepend(li);                      // at start

// Remove element
li.remove();                          // modern way (IE no)
ul.removeChild(li);                   // legacy way
5Events

Making Pages Interactive

Events are signals fired when something happens — a click, keypress, form submit, mouse move, page load. We listen for events and handle them with functions.

addEventListener

const btn = document.querySelector("button");

btn.addEventListener("click", function(event) {
  console.log("Button clicked!");
  console.log(event.target);    // the clicked element
  console.log(event.type);      // "click"
});

// Arrow function
btn.addEventListener("click", e => {
  console.log("Clicked!");
});

Common Events

EventTrigger
clickMouse click on element
submitForm submitted
keydownKey pressed
DOMContentLoadedHTML parsed, DOM ready
inputValue of input/textarea changed
Real-World Analogy
The event system is like a doorbell. You install a listener (the doorbell button). When someone presses it (event fires), your handler runs (you answer the door). The event object is like the camera feed — it tells you who is at the door (target) and what happened (type). You can also stop the event from propagating further (stopPropagation), like telling your dog to stop barking at the doorbell.
6Event Propagation

Bubbling & Capturing

When an event fires on an element, it travels through the DOM in three phases:
  1. Capturing phase — event travels from document down to the target
  2. Target phase — event reaches the target element
  3. Bubbling phase — event travels from target back up to document
// By default, listeners fire during bubbling phase
parent.addEventListener("click", () => {
  console.log("Parent (bubbling)");
});
child.addEventListener("click", () => {
  console.log("Child");
});
// Clicking child logs: "Child" → "Parent (bubbling)"

// Capture phase: pass { capture: true } or true
parent.addEventListener("click", () => {
  console.log("Parent (capture)");
}, { capture: true });
// Now clicking child logs: "Parent (capture)" → "Child" → "Parent (bubbling)"

// Stop propagation
child.addEventListener("click", e => {
  e.stopPropagation();  // prevents bubbling
});

Event Delegation

A powerful pattern where a single listener on a parent handles events for all current and future children.

// Instead of adding listeners to every li...
ul.addEventListener("click", e => {
  const li = e.target.closest("li");
  if (li) {
    console.log("Clicked:", li.textContent);
  }
});
Best Practice
Use event delegation for lists, tables, and dynamic content. It uses less memory (one listener vs hundreds) and automatically handles elements added later. Use e.target.closest(selector) to find the right element.
7Industry Context
Industry Insight
Modern frameworks like React and Vue abstract away direct DOM manipulation. They use a Virtual DOM — a lightweight JS representation — and batch updates for performance. However, understanding the raw DOM API is still critical for debugging, working with Web Components, and scenarios where you need direct control (canvas, games, performance-critical code). Every React dev inevitably needs document.querySelector or .addEventListener at some point.
8Interview Q&A
Q: What is the difference between textContent and innerHTML?
A: textContent sets/gets text only — it escapes HTML tags (safe). innerHTML parses the string as HTML (XSS risk if content is user-provided). Use textContent unless you specifically need to insert HTML.
Q: What is event delegation and why is it useful?
A: Event delegation attaches a single event listener to a parent element to handle events from its children via bubbling. Benefits: (1) less memory (one listener instead of hundreds), (2) works for dynamically added elements, (3) cleaner code.
Q: What is event.stopPropagation() vs event.preventDefault()?
A: stopPropagation() prevents the event from traveling further through the DOM (bubbling/capturing). preventDefault() prevents the browser's default behavior (e.g., link navigation, form submission) but does not stop propagation.
Q: What is a NodeList vs an Array?
A: NodeList is the return type of querySelectorAll. It has forEach but not map, filter, or reduce. Convert with Array.from() or spread [...nodeList] to use full array methods.
