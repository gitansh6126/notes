Data Types & Variables

Strings & UTF-16

JS strings are UTF-16 — some characters (like emoji) take up two "code units," not one.

Real Life

Mailing a letter where most stamps cover one page, but a few thick, glossy pages secretly need two stamps to send.

Analogy

If you count "stamps" instead of "pages," a letter with a few glossy inserts looks longer than it visually is — exactly how "😀".length returns 2, not 1.

undefined vs null

undefined: a survey question the person simply skipped — never answered.

null: a survey question they answered with "Prefer not to say" — deliberately marked as empty.

let vs const

let: a whiteboard — write a number, erase it, write a new one. Perfect for a loop counter.

const: a labeled moving box. The label ("myBox") never changes hands — but you can still open the box and rearrange what's inside if it's an object.

=== Strict Equality

Compares both value and type — no silent conversions.

Real Life

A strict border officer who checks that your passport number and document type both match exactly — a driver's license with the right number doesn't count.

Analogy

Loose equality (==) is the lenient officer who just glances at the number and waves you through, even if the document type doesn't match — how "5" == 5 sneaks past but "5" === 5 doesn't.

Control Flow

if / else

Real Life

A fork in a hiking trail with a sign: "If raining, take the covered path. Otherwise, take the scenic route."

Analogy

One condition, one branch taken — exactly how a hiker (or the program) commits to a single path based on a check.

switch

A restaurant's fixed combo menu: "Combo 1," "Combo 2," "Combo 3" — pick a number, get a whole pre-set meal.

Analogy

Cleaner than a long chain of "if you want this, or this, or this" — one clear menu of named cases instead of nested forks.

An assembly line moving down a numbered row of stations, one at a time, until the last one.

Functions

Functions are just recipes — the differences are in how they're written down and handed out.

Standard Declarationfunction add(){}
A recipe published in a cookbook under its own name — anyone can look it up by title and use it.Real life: a named recipe everyone in the kitchen can reference by calling out its name.
Anonymous Functionlet add = function(){}
A recipe card with no title, handed directly to one specific person (a variable) to hold onto.Real life: an unlabeled index card passed privately, rather than filed in the public cookbook.
Arrow Function(x, y) => x + y
A quick sticky note instead of a full recipe card — best for short, simple steps.Real life: scribbling "add butter + sugar" on a Post-it instead of writing a formal recipe card for something that simple.
IIFE(function(){...})()
A recipe written and cooked in the very same breath — used once, immediately, then gone.Real life: a firework — built and set off in a single action, never reused. Handy once, but modern kitchens (modules) rarely need this trick anymore.
Function Objectsadd.helpText = "..."
Sticking extra Post-it notes directly onto a kitchen tool itself, rather than writing them in the recipe book.Real life: taping a note onto your whisk instead of writing it in the cookbook — technically works, but clutters the tool and confuses the next cook.

Every JS keyword here is really just a labeling choice — how strict, how permanent, how public you want each piece of data or logic to be.
