Kernel PCA — PCA
Without Ever Leaving Kernel Space

Last time, we mapped data into a high-dimensional space to make non-linear structure linear — then hit a wall of exploding dimensions. Kernel PCA is the way out: it gets every benefit of that high-dimensional space while never actually visiting it.

Builds on · Feature mapping φ, the kernel trick Core idea · Only pairwise dot products ever matter Output · Non-linear, low-dimensional projections
1

Motivation: Skipping the Trip Up

RECAP + THE LEAP

We wanted the high-dimensional view — not the high-dimensional bill

Standard PCA can only find straight-line structure. To handle curved, non-linear data, the previous lecture mapped every point X into a much bigger space via a feature map φ, where the non-linear structure straightens out. The catch was cost: φ(X) can have millions of coordinates, or more.

Kernel PCA's insight: PCA never actually needs the coordinates of φ(X). Every step of PCA — variance, covariance, projections — can be rewritten using only dot products between mapped points, φ(Xᵢ)ᵀφ(Xⱼ). And a well-chosen kernel function K can compute that dot product directly from the original, low-dimensional Xᵢ and Xⱼ — no detour through the huge space required.

Real-life analogy

Suppose you want to know how "compatible" two people's music tastes are — a comparison that, in principle, would mean comparing their entire, enormous streaming histories side by side. A recommendation app doesn't do that. It hands you a single compatibility score for each pair, computed behind the scenes, without ever showing you (or itself) the full listening history in one giant table. The kernel function K is exactly that shortcut score — it tells you the answer to "how similar are these two in the big space?" without either of you having to visit the big space.

Inputs to the Algorithm

WHAT YOU NEED

Just data, and a way to score similarity

  • A dataset of n points X₁, ..., Xₙ ∈ ℝd — ordinary, low-dimensional data, exactly as collected.
  • A valid kernel function K that computes, for any two points, the dot product they would have had in the high-dimensional feature space — without ever forming that space:
K(Xi, Xj)  =  φ(Xi)T φ(Xj) a single number, computed directly from the original points — the high-dimensional detour is invisible

That's the entire input list. Notice what's absent: there's no need to choose, specify, or even know φ explicitly. The kernel function is the only place non-linearity enters the picture.

Building the Kernel Matrix, Then Centering It

STEP 1

Compute the kernel matrix — all pairwise similarities, at once

Build an n × n matrix K where entry Kij = K(Xᵢ, Xⱼ). Every row and column corresponds to one data point; every cell holds how similar that pair looks in the (never-visited) high-dimensional space.

Kernel matrix as a similarity heat-map (6 sample points)

Darker blue = higher similarity. The diagonal (Xᵢ compared to itself) is always the darkest — every point is maximally similar to itself.

Real-life analogy

Think of a round-robin tournament where every player faces every other player exactly once, and you record a "closeness of match" score for each pairing. Lay all those scores out in a grid — that grid, not any individual player's private stats, is all the kernel matrix is.

STEP 2

Center the kernel — because PCA insists on zero mean

Ordinary PCA subtracts the mean from every point before doing anything else. Here, the points technically live in φ-space, which we can't touch directly — so centering has to happen through algebra on K itself:

Kc  =  K  −  1nK  −  K1n  +  1nK1n 1ₙ is the n×n matrix where every entry equals 1/n — this expression is provably identical to centering φ(X) directly

Real-life analogy

Recentering a group photo usually means physically moving people or the camera. But if all you have is a table of "how far apart" every pair of people stood, you can still work out how to mathematically re-balance that table — shifting everyone's effective position to average out to the center — without ever seeing an actual photo. That's what the centering formula does to K.

Worth noting

Centering isn't optional bookkeeping — skip it, and the "directions of maximum variance" that PCA finds next will be distorted by wherever the data happens to sit, rather than reflecting its true spread.

Eigen-decomposition & Projections

STEPS 3 & 4

Solve the small matrix, then rescale

3

Eigen-decompose K꜀

Solve for eigenvectors βk and eigenvalues λk of the centered kernel matrix.

4

Normalize

Rescale: αk = βk / √(nλk), so each direction has unit length in φ-space.

5

Project

Combine αk with kernel values to get each point's coordinate on component k.

6

Keep top k

Retain the few components with the largest λk — your reduced representation.

12projection[i, k] = sum(alpha_k[j] * K(X_i, X_j) for j in range(n))
  • Same n×n trick from the previous lecture — the expensive D×D step never happens.
  • The normalization step matters: without it, projections would be on an arbitrary, inconsistent scale.
  • The projection formula is the payoff — a weighted sum of kernel values, no φ(x) in sight.
Projectionk(Xi)  =  Σj=1n αkj · K(Xi, Xj) the new "coordinate" of Xᵢ on principal component k — built entirely from kernel evaluations against every other point

Xᵢ

To find where Xᵢ (red center) lands on a principal component, we don't look at Xᵢ alone — we take a weighted sum of its kernel similarity to every point in the dataset (blue), scaled by that component's α weights.

Real-life analogy

It's the same logic GPS trilateration uses: a single receiver can't know its own position from just one signal, but combine distance readings (a kind of similarity score) from several satellites, each weighted appropriately, and a precise position falls out — without ever "seeing" the sky directly. Kernel PCA locates each point on a principal component the same way: by combining its similarity scores to everyone else, weighted just right.

Important Considerations

DESIGN CHOICE

Avoid reconstruction, on purpose

It might feel like something is "missing" — shouldn't we recover the actual high-dimensional eigenvectors? Usually we can't: φ(X) is often inaccessible or infinite-dimensional. Kernel PCA sidesteps this entirely by only ever computing projection values — and that's sufficient for almost every downstream task (classification, clustering, visualization).

GENERAL PRINCIPLE

Any algorithm built on dot products can be kernelized

Kernel PCA isn't a one-off hack — it's an instance of a general pattern. Any algorithm whose math depends solely on pairwise dot products between data points can swap those dot products for kernel evaluations and gain the same non-linear upgrade — no matter how that algorithm is normally described.

Watch out

Skipping the centering step (Step 2) is a common silent bug: the algorithm will still run and produce numbers, but the resulting "principal directions" will be distorted by an off-center implicit cloud — not an error you'll see, just wrong answers.

Key insight

Every quantity Kernel PCA ever touches — variance, covariance, projections — reduces to combinations of K(Xᵢ, Xⱼ). That single fact is the whole trick: non-linear power, linear-algebra cost.

✓

Key Takeaways

1

Kernel PCA performs non-linear PCA by working entirely with an n×n kernel matrix — never the (potentially huge) mapped features φ(X) themselves.

2

Centering must happen algebraically on K (K꜀ = K − 1ₙK − K1ₙ + 1ₙK1ₙ), since the implicit feature space can't be touched directly.

3

Eigen-decompose K꜀, normalize by √(nλₖ), and every quantity you need falls out — still without ever forming φ(X).

4

Final projections are weighted sums of kernel values against every training point — sufficient for most ML tasks, with no reconstruction needed.
