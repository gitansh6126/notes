Dimensionality Reduction Series · Part 2

Kernel PCA — PCA
Without Ever Leaving Kernel Space

Last time, we mapped data into a high-dimensional space to make non-linear structure linear — then hit a wall of exploding dimensions. Kernel PCA is the way out: it gets every benefit of that high-dimensional space while never actually visiting it.

§1 Motivation §2 Kernel Function §3 Kernel Matrix & Centering §4 Eigen-decomp & Projections §5 Watch-outs §6 Takeaways
01

Motivation: Skipping the Trip Up

Standard PCA can only find straight-line structure. To handle curved, non-linear data, the previous lecture mapped every point $\mathbf{x}$ into a much bigger space via a feature map $\varphi$, where the non-linear structure straightens out. The catch was cost: $\varphi(\mathbf{X})$ can have millions of coordinates, or more.

✔
Kernel PCA's insight

PCA never actually needs the coordinates of $\varphi(\mathbf{X})$. Every step of PCA — variance, covariance, projections — can be rewritten using only dot products between mapped points, $\varphi(\mathbf{x}_i)^\mathsf{T}\varphi(\mathbf{x}_j)$. And a well-chosen kernel function $k$ can compute that dot product directly from the original, low-dimensional $\mathbf{x}_i$ and $\mathbf{x}_j$ — no detour through the huge space required.

▶ Intuition
Suppose you want to know how "compatible" two people's music tastes are — a comparison that, in principle, would mean comparing their entire, enormous streaming histories side by side. A recommendation app doesn't do that. It hands you a single compatibility score for each pair, computed behind the scenes, without ever showing you (or itself) the full listening history in one giant table. The kernel function $k$ is exactly that shortcut score — it tells you the answer to "how similar are these two in the big space?" without either of you having to visit the big space.

02

The Kernel Function — the Only Non-Linearity

A kernel function $k(\mathbf{x}_i, \mathbf{x}_j)$ computes the dot product that $\mathbf{x}_i$ and $\mathbf{x}_j$ would have in the high-dimensional feature space — computed directly from the original low-dimensional inputs.

Equation (1)
Kernel Function Definition
$$k(\mathbf{x}_i, \mathbf{x}_j) = \varphi(\mathbf{x}_i)^\mathsf{T} \varphi(\mathbf{x}_j)$$
A single number computed directly from the original points — the high-dimensional detour is invisible
▶ Read the Equation in English
“The kernel value between two points $\mathbf{x}_i$ and $\mathbf{x}_j$ equals the dot product of their images under the feature map $\varphi$ — even though $\varphi$ is never explicitly evaluated. The kernel function is the only place non-linearity enters the picture.”
▶ Symbol Legend
SymbolMeaningExample
$k(\mathbf{x}_i, \mathbf{x}_j)$ Kernel function — similarity score in $\varphi$-space RBF, polynomial, linear
$\varphi(\mathbf{x})$ Feature map (implicit, never computed) Can be infinite-dimensional
$\mathbf{x}_i, \mathbf{x}_j$ Original low-dimensional data points $\mathbf{x}_i \in \mathbb{R}^d$
$k(\mathbf{x}, \mathbf{y}) = \mathbf{x}^\mathsf{T}\mathbf{y}$

Linear kernel

Just the standard dot product — recovers ordinary PCA.

$k(\mathbf{x}, \mathbf{y}) = (\mathbf{x}^\mathsf{T}\mathbf{y} + c)^p$

Polynomial kernel

Captures all feature interactions up to degree $p$. The implicit $\varphi$-space has $O(d^p)$ dimensions — but the kernel computes it in $O(d)$.

$k(\mathbf{x}, \mathbf{y}) = \exp\!\left(-\frac{\|\mathbf{x} - \mathbf{y}\|^2}{2\sigma^2}\right)$

RBF (Gaussian) kernel

Corresponds to an infinite-dimensional $\varphi$-space. The most expressive — and most commonly used — kernel.

i
The input list is minimal

Kernel PCA only needs: (1) A dataset of $n$ points $\mathbf{x}_1, \dots, \mathbf{x}_n \in \mathbb{R}^d$ — ordinary, low-dimensional data, exactly as collected. (2) A valid kernel function $k$. That's it. No need to choose or even know $\varphi$ explicitly.

03

Building the Kernel Matrix, Then Centering It

All pairwise similarities are collected into an $n \times n$ matrix $\mathbf{K}$. Then, because PCA requires centered data, we must center $\mathbf{K}$ algebraically.

Equation (2)
Kernel (Gram) Matrix
$$\mathbf{K}_{ij} = k(\mathbf{x}_i, \mathbf{x}_j), \qquad \mathbf{K} \in \mathbb{R}^{n \times n}$$
Every row and column is one data point; every cell is the similarity of that pair in $\varphi$-space
▶ Read the Equation in English
“The kernel matrix $\mathbf{K}$ has $n$ rows and $n$ columns. The entry at position $(i, j)$ is the kernel value $k(\mathbf{x}_i, \mathbf{x}_j)$ — the dot product of $\mathbf{x}_i$ and $\mathbf{x}_j$ in the implicit feature space. This matrix is the complete description of the data in kernel PCA.”
▶ Kernel Matrix as a Similarity Heat-Map (6 Sample Points)
Darker blue = higher similarity. The diagonal entries (self-similarity) are always the maximum.
▶ Intuition
Think of a round-robin tournament where every player faces every other player exactly once, and you record a "closeness of match" score for each pairing. Lay all those scores out in a grid — that grid, not any individual player's private stats, is all the kernel matrix is. Every piece of information Kernel PCA needs is in that grid.
Equation (3)
Centering the Kernel Matrix
$$\widetilde{\mathbf{K}} = \mathbf{K} - \mathbf{1}_n \mathbf{K} - \mathbf{K} \mathbf{1}_n + \mathbf{1}_n \mathbf{K} \mathbf{1}_n$$
$\mathbf{1}_n$ is the $n \times n$ matrix where every entry equals $1/n$ — this expression centers $\varphi(\mathbf{X})$ without ever touching it
▶ Read the Equation in English
“To center the kernel matrix, subtract the row mean ($\mathbf{1}_n\mathbf{K}$), subtract the column mean ($\mathbf{K}\mathbf{1}_n$), add back the overall mean ($\mathbf{1}_n\mathbf{K}\mathbf{1}_n$). This is provably identical to subtracting the mean of $\varphi(\mathbf{X})$ in the feature space — but done purely through algebra on $\mathbf{K}$.”
▶ Centering Symbol Legend
SymbolMeaningRole
$\mathbf{K}$ Uncentered kernel matrix Raw pairwise similarities
$\widetilde{\mathbf{K}}$ Centered kernel matrix Mean-subtracted in $\varphi$-space
$\mathbf{1}_n$ $n \times n$ matrix with all entries $= 1/n$ The "averaging" matrix
▶ Before and After Centering (Implicit $\varphi$-Space)
The centering formula re-centers the implicit $\varphi$-space cloud without ever accessing $\varphi$ directly.
!
Centering isn't optional

Skip it, and the "directions of maximum variance" that PCA finds next will be distorted by wherever the data happens to sit, rather than reflecting its true spread. This is a common silent bug — the algorithm will still run and produce numbers, but the results will be wrong.

04

Eigen-decomposition & Projections

The centered kernel matrix $\widetilde{\mathbf{K}}$ is eigen-decomposed, its eigenvectors are normalized, and then used to compute projection coordinates for any data point.

▶ Kernel PCA Algorithm — Steps 3–6
3
Eigen-decompose $\widetilde{\mathbf{K}}$ — an $n \times n$ problem
$\widetilde{\mathbf{K}} \boldsymbol{\beta}_k = \tilde{\lambda}_k \boldsymbol{\beta}_k$
↓
4
Normalize the eigenvectors so that directions have unit length in $\varphi$-space
$\displaystyle \boldsymbol{\alpha}_k = \frac{\boldsymbol{\beta}_k}{\sqrt{n \tilde{\lambda}_k}}$
↓
5
Project a point $\mathbf{x}$ onto component $k$ using only kernel evaluations
$\displaystyle y_k(\mathbf{x}) = \sum_{j=1}^{n} \alpha_{kj} \, k(\mathbf{x}, \mathbf{x}_j)$
↓
6
Retain top $m$ components with largest $\tilde{\lambda}_k$
$\mathbb{R}^n \to \mathbb{R}^m$, $m \ll n$
Equation (4)
Projection onto the $k$-th Principal Component
$$y_k(\mathbf{x}) = \sum_{j=1}^{n} \alpha_{kj} \; k(\mathbf{x}, \mathbf{x}_j)$$
The new "coordinate" of $\mathbf{x}$ on component $k$ — a weighted sum of kernel values against every training point
▶ Read the Equation in English
“The projection of a point $\mathbf{x}$ onto the $k$-th principal component equals the sum over all $n$ training points of the coefficient $\alpha_{kj}$ times the kernel similarity between $\mathbf{x}$ and $\mathbf{x}_j$. This is a weighted vote from every training point, mediated by the kernel.”
▶ Projection Symbol Legend
SymbolMeaningExample
$y_k(\mathbf{x})$ Coordinate of $\mathbf{x}$ on the $k$-th principal component A single scalar
$\alpha_{kj}$ $j$-th entry of the normalized eigenvector $\boldsymbol{\alpha}_k$ Weight for training point $j$
$k(\mathbf{x}, \mathbf{x}_j)$ Kernel similarity between test point and $j$-th training point RBF value
kernel_pca.py
# K_c: the centered n x n kernel matrix from Step 2
eigvals, eigvecs = eigh(K_c)         # solve an n x n problem, not D x D

# keep only the top k eigenpairs (largest eigenvalues)
beta_k  = eigvecs[:, :k]
lambda_k = eigvals[:k]

# normalize so directions have unit length in feature space
alpha_k = beta_k / sqrt(n * lambda_k)

# project point X_i onto component k using ONLY kernel values
projection[i, k] = sum(alpha_k[j] * K(X_i, X_j) for j in range(n))
▶ Intuition
It's the same logic GPS trilateration uses: a single receiver can't know its own position from just one signal, but combine distance readings (a kind of similarity score) from several satellites, each weighted appropriately, and a precise position falls out — without ever "seeing" the sky directly. Kernel PCA locates each point on a principal component the same way: by combining its similarity scores to everyone else, weighted just right.
▶ Projecting One Point: A Weighted Vote From Everyone Else
To find where $\mathbf{x}_i$ (red center) lands on a principal component, we take a weighted sum of its kernel similarity to every point in the dataset (blue), scaled by that component's $\alpha$ weights.

05

Important Considerations

Kernel PCA is powerful, but differs from standard PCA in several important ways that affect how it's used and interpreted.

No reconstruction

We can't recover the actual eigenvectors in $\varphi$-space because $\varphi(\mathbf{X})$ is often inaccessible or infinite-dimensional. But projection values are sufficient for almost every downstream task.

Kernel choice matters

The kernel function determines the implicit feature space. RBF is flexible but has a length-scale $\sigma$ hyperparameter. Polynomial kernels require choosing degree $p$ and offset $c$.

Any algorithm can be kernelized

Any algorithm whose math depends solely on pairwise dot products can swap those for kernel evaluations — this includes PCA, regression, and many classifiers.

!
Silent bug: skipping centering

The algorithm will still run and produce numbers, but the resulting "principal directions" will be distorted by an off-center implicit cloud — not an error you'll see, just wrong answers. Always center $\mathbf{K}$.

✔
Key insight

Every quantity Kernel PCA ever touches — variance, covariance, projections — reduces to combinations of $k(\mathbf{x}_i, \mathbf{x}_j)$. That single fact is the whole trick: non-linear power, linear-algebra cost.

▶ Where Kernel PCA Shines
Non-linear dimensionality reduction Visualizing high-dimensional data on a circle, sphere, or manifold — standard PCA would miss the structure entirely.
Preprocessing for classifiers Kernel PCA projections often improve downstream SVM or logistic regression accuracy by revealing the data's intrinsic geometry.
Noise filtering Reconstructing data from the top $m$ kernel PCs can denoise non-linear signals that linear PCA would corrupt.
Anomaly detection Points with high reconstruction error in kernel PCA space are anomalies — even if the normal data lies on a non-linear manifold.
06

Key Takeaways

▶ The Big Picture
Kernel PCA completes the journey from Lecture 9 to Lecture 12: Lecture 9 showed that PCA's $d \times d$ covariance can be replaced by an $n \times n$ Gram matrix (the linear kernel trick). Lecture 10 showed that non-linear data needs a feature map $\varphi$, but $D$ explodes. This lecture ties it all together: replace the Gram matrix's dot product with any valid kernel function, and you get the power of the high-dimensional $\varphi$-space at the cost of the $n \times n$ computation.

Kernel PCA works in $\varphi$-space

It performs non-linear PCA by working entirely with an $n \times n$ kernel matrix — never the (potentially huge) mapped features $\varphi(\mathbf{X})$ themselves.

Centering is algebraic

Since the feature space is implicit, centering must happen via $\widetilde{\mathbf{K}} = \mathbf{K} - \mathbf{1}_n\mathbf{K} - \mathbf{K}\mathbf{1}_n + \mathbf{1}_n\mathbf{K}\mathbf{1}_n$.

Projections are kernel sums

Final coordinates $y_k(\mathbf{x}) = \sum_j \alpha_{kj} \, k(\mathbf{x}, \mathbf{x}_j)$ are weighted kernel sums — sufficient for most ML tasks, with no reconstruction needed.

What comes next

With Kernel PCA in hand, the same "kernelize everything" philosophy extends to other algorithms — Kernel Ridge Regression, Support Vector Machines, and Gaussian Processes all build on the same fundamental trick of replacing dot products with kernel functions.
