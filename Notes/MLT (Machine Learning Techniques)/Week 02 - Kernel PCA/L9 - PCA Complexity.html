PCA's hidden cost:
when features outnumber data

PCA's textbook recipe quietly assumes you can afford to decompose a $d \times d$ matrix. The moment your features ($d$) vastly outnumber your data points ($n$) — genomics, images, text — that assumption becomes the whole bottleneck.

§1 The bottleneck §2 The key trick §3 The algorithm §4 Feel the speedup §5 Toward kernels
01

The bottleneck: a matrix too big to decompose

PCA finds principal components by eigen-decomposing the covariance matrix of the data. That matrix's size is dictated entirely by the number of features — not the number of data points.

Equation (1)
Covariance Matrix of the Data
$$\mathbf{C} = \frac{1}{n} \mathbf{X} \mathbf{X}^\mathsf{T}, \qquad \mathbf{X} \in \mathbb{R}^{d \times n}, \quad \mathbf{C} \in \mathbb{R}^{d \times d}$$
The covariance matrix is $d \times d$ — its size depends on features, not samples
▶ Read the Equation in English
“The covariance matrix $\mathbf{C}$ equals $\frac{1}{n}$ times the data matrix $\mathbf{X}$ multiplied by its transpose $\mathbf{X}^\mathsf{T}$. Since $\mathbf{X}$ has $d$ rows (one per feature) and $n$ columns (one per data point), the product $\mathbf{X}\mathbf{X}^\mathsf{T}$ is a $d \times d$ matrix.”
▶ Symbol Legend
SymbolMeaningExample
$\mathbf{C}$ Covariance matrix — measures how features vary together $d \times d$ matrix
$\mathbf{X}$ Data matrix — each column is one data point $d \times n$
$d$ Number of features (dimensions) per data point $d = 20{,}000$ genes
$n$ Number of data points (samples) $n = 200$ patients
$O(d^3)$ Time complexity of eigen-decomposing a $d \times d$ matrix $20{,}000^3 \approx 8 \times 10^{12}$
▶ Intuition
Imagine describing 5 people using 20,000 medical measurements each (blood markers, genetic variants, scan readings). You'd build a giant $20{,}000 \times 20{,}000$ "how do these measurements relate to each other" matrix — but there are only 5 people's worth of actual information in the room. Most of that giant matrix is structurally redundant before you even start solving it.
▶ Visual: $d \times d$ vs $n \times n$ matrix sizes
The covariance matrix $\mathbf{C}$ scales with $d$, the kernel-style matrix $\mathbf{K}$ scales with $n$.
!
Why this hurts

Eigenvalue decomposition of a $d \times d$ matrix costs $O(d^3)$. With $d$ in the tens of thousands — genes, pixels, vocabulary words — this becomes computationally impossible long before it becomes merely slow.

!
Common Mistakes

✖ Thinking $d$ and $n$ are interchangeable: The covariance matrix size depends on $d$ (features), not $n$ (samples). Even with $n = 10$ points and $d = 10{,}000$ features, the matrix is $10{,}000 \times 10{,}000$.

✖ Ignoring the $O(d^3)$ scaling: Doubling $d$ increases decomposition cost by $8\times$. With $d = 1000$, that's $10^9$ operations — feasible. With $d = 100{,}000$, that's $10^{15}$ — infeasible.

02

The key trick: recycle the data itself

The escape route relies on one elegant fact: every principal direction we're searching for is secretly already buildable from the data points we have.

▶ Intuition
Think of mixing paint. You don't need to invent a brand-new pigment to get a new principal direction — you just need to blend the paint samples (data points) you already own, in different proportions. With only $n$ samples, there are only $n$ independent ways to blend, no matter how many color channels (features) each sample has.
Equation (2)
Eigenvector as Linear Combination of Data Points
$$\mathbf{w}_k = \sum_{i=1}^{n} \alpha_{ki} \, \mathbf{x}_i = \mathbf{X} \boldsymbol{\alpha}_k, \qquad \boldsymbol{\alpha}_k \in \mathbb{R}^n$$
Every eigenvector $\mathbf{w}_k$ of the covariance matrix is just a weighted combination of the $n$ data points
▶ Read the Equation in English
“The $k$-th principal direction $\mathbf{w}_k$ equals the sum over $i$ from $1$ to $n$ of weight $\alpha_{ki}$ times the $i$-th data point $\mathbf{x}_i$. In matrix form, $\mathbf{w}_k = \mathbf{X} \boldsymbol{\alpha}_k$ where $\boldsymbol{\alpha}_k$ is a vector of $n$ scalar weights.”
▶ Symbol Legend
SymbolMeaningExample
$\mathbf{w}_k$ $k$-th principal direction (eigenvector of $\mathbf{C}$) Direction of max variance
$\mathbf{x}_i$ $i$-th data point (column of $\mathbf{X}$) A patient's gene expression vector
$\boldsymbol{\alpha}_k$ Coefficient vector — the blend weights for $\mathbf{w}_k$ $n$ scalars, one per data point
$\mathbf{X}$ Data matrix ($d \times n$) $20{,}000 \times 200$
▶ Step-by-Step: Trading $d \times d$ for $n \times n$
1
Start with the eigenvalue equation for the covariance matrix
$\mathbf{C} \mathbf{w}_k = \lambda_k \mathbf{w}_k$
↓
2
Substitute $\mathbf{C} = \frac{1}{n} \mathbf{X} \mathbf{X}^\mathsf{T}$ and $\mathbf{w}_k = \mathbf{X} \boldsymbol{\alpha}_k$
$\frac{1}{n} \mathbf{X} \mathbf{X}^\mathsf{T} \mathbf{X} \boldsymbol{\alpha}_k = \lambda_k \mathbf{X} \boldsymbol{\alpha}_k$
↓
3
Multiply both sides on the left by $\mathbf{X}^\mathsf{T}$
$\frac{1}{n} \mathbf{X}^\mathsf{T} \mathbf{X} \mathbf{X}^\mathsf{T} \mathbf{X} \boldsymbol{\alpha}_k = \lambda_k \mathbf{X}^\mathsf{T} \mathbf{X} \boldsymbol{\alpha}_k$
↓
4
Define $\mathbf{K} = \mathbf{X}^\mathsf{T} \mathbf{X}$ (the $n \times n$ Gram matrix) — it emerges naturally
$\frac{1}{n} \mathbf{K} \mathbf{K} \boldsymbol{\alpha}_k = \lambda_k \mathbf{K} \boldsymbol{\alpha}_k$
↓
5
Simplify: $\mathbf{K} \boldsymbol{\alpha}_k = n \lambda_k \boldsymbol{\alpha}_k$ — an $n \times n$ eigenvalue problem!
$\displaystyle \mathbf{K} \boldsymbol{\alpha}_k = \tilde{\lambda}_k \boldsymbol{\alpha}_k, \quad \tilde{\lambda}_k = n \lambda_k$
✔
The payoff

We've traded a $d \times d$ eigen-problem for an equivalent $n \times n$ one. When $d \gg n$, that's the difference between "impossible" and "instant."

i
What this means

The matrices $\mathbf{X}\mathbf{X}^\mathsf{T}$ (size $d \times d$) and $\mathbf{X}^\mathsf{T}\mathbf{X}$ (size $n \times n$) share the same non-zero eigenvalues. We never lose any real structure — we just choose to compute it in whichever matrix is smaller. This is a standard linear-algebra fact that underwrites the entire trick.

▶ Numerical Example: Size Comparison
Setup: A genomics dataset with $d = 20{,}000$ genes and $n = 200$ patients.

03

The algorithm

Four steps take you from raw data to the same principal components PCA would have found the expensive way.

▶ The Efficient PCA Algorithm
1
Compute $\mathbf{K} = \mathbf{X}^\mathsf{T} \mathbf{X}$ — the $n \times n$ Gram matrix, not the $d \times d$ covariance
$\mathbf{K}_{ij} = \mathbf{x}_i^\mathsf{T} \mathbf{x}_j$
↓
2
Eigen-decompose $\mathbf{K}$ to get eigenvectors $\boldsymbol{\beta}_k$ and eigenvalues $\tilde{\lambda}_k$
$\mathbf{K} \boldsymbol{\beta}_k = \tilde{\lambda}_k \boldsymbol{\beta}_k$
↓
3
Normalize — $\mathbf{K}$'s eigenvectors need rescaling to be the true PCA coefficients
$\displaystyle \boldsymbol{\alpha}_k = \frac{\boldsymbol{\beta}_k}{\sqrt{n \lambda_k}}$
↓
4
Reconstruct the principal components in the original feature space
$\mathbf{w}_k = \mathbf{X} \boldsymbol{\alpha}_k$
Equation (3)
Reconstruction of Principal Components
$$\mathbf{w}_k = \mathbf{X} \boldsymbol{\alpha}_k = \frac{1}{\sqrt{n \lambda_k}} \mathbf{X} \boldsymbol{\beta}_k$$
Map back from $n$-dimensional coefficient space to $d$-dimensional feature space
▶ Read the Equation in English
“To recover the $k$-th principal direction $\mathbf{w}_k$ in the original $d$-dimensional space, take the data matrix $\mathbf{X}$, multiply by the normalized eigenvector $\boldsymbol{\alpha}_k$ from the $n \times n$ problem. The result is a $d$-dimensional vector — the same one you'd get from decomposing the $d \times d$ covariance matrix directly.”
▶ Algorithm Symbol Legend
SymbolMeaningRole
$\mathbf{K}$ Gram matrix of pairwise dot products $\mathbf{K} = \mathbf{X}^\mathsf{T}\mathbf{X}$
$\boldsymbol{\beta}_k$ Eigenvector of $\mathbf{K}$ (unit length in $\mathbb{R}^n$) $\|\boldsymbol{\beta}_k\| = 1$
$\tilde{\lambda}_k$ Eigenvalue of $\mathbf{K}$ $\tilde{\lambda}_k = n \lambda_k$
$\boldsymbol{\alpha}_k$ Normalized coefficient vector $\boldsymbol{\alpha}_k = \boldsymbol{\beta}_k / \sqrt{\tilde{\lambda}_k}$
▶ Intuition
Think of $\boldsymbol{\beta}_k$ as a set of instructions in the "data-point space" (which points to emphasize), and $\mathbf{X}$ as the dictionary that translates those instructions back to the original feature space. Step 3 (normalization) ensures each translated direction has unit length in the feature space — without it, the scaling would be arbitrary.
▶ Algorithm Flow: From Data to Principal Components
The data flows through four stages: Gram matrix, eigen-decomposition, normalization, and reconstruction — all $O(n^3)$ instead of $O(d^3)$.
i
Why this is valid

A standard linear-algebra fact underwrites the whole trick: $\mathbf{X}\mathbf{X}^\mathsf{T}$ and $\mathbf{X}^\mathsf{T}\mathbf{X}$ share the same non-zero eigenvalues. We never lose any real structure — we just choose to compute it in whichever matrix is smaller.

!
Common Mistakes

✖ Forgetting normalization: The eigenvectors $\boldsymbol{\beta}_k$ of $\mathbf{K}$ are not directly the PCA coefficients. Without dividing by $\sqrt{n\lambda_k}$, the reconstructed $\mathbf{w}_k$ will not have unit length.

✖ Confusing $\mathbf{K}$ with the covariance: $\mathbf{K} = \mathbf{X}^\mathsf{T}\mathbf{X}$ is the Gram matrix (unnormalized, uncentered). The covariance is $\frac{1}{n}\mathbf{X}\mathbf{X}^\mathsf{T}$, which is $d \times d$ and different.

What this means in practice

A $d = 20{,}000$ eigen-decomposition takes roughly hours on a modern CPU (and may crash from memory). The $n = 200$ equivalent takes milliseconds. The speedup of $1{,}000{,}000\times$ isn't a metaphor — it's the literal ratio of $(20{,}000/200)^3$. This trick turns infeasible problems into routine computations.

05

One more observation — the door to Kernel PCA

Look closely at what $\mathbf{K} = \mathbf{X}^\mathsf{T}\mathbf{X}$ actually contains: entry $(i, j)$ is $\mathbf{x}_i^\mathsf{T} \mathbf{x}_j$ — the dot product between data points $i$ and $j$.

Equation (4)
Gram Matrix Entry
$$\mathbf{K}_{ij} = \mathbf{x}_i^\mathsf{T} \mathbf{x}_j$$
Only pairwise dot products — the raw data vectors themselves are never needed individually after $\mathbf{K}$ is built
▶ Read the Equation in English
“The entry in row $i$, column $j$ of the Gram matrix $\mathbf{K}$ is simply the dot product of the $i$-th and $j$-th data points. That's it — a single scalar that summarizes how aligned those two points are.”
▶ Intuition
It's like judging how similar two songs are just by how they make you feel side by side, rather than comparing every individual audio frequency in each track. $\mathbf{K}$ is nothing but a table of pairwise similarities between data points — PCA never actually needs the raw high-dimensional features themselves, only how each point relates to every other point.
why_this_matters.txt
// K only ever needs pairwise dot products:
K[i][j] = x_i . x_j

// swap the dot product for any valid "similarity" function,
// and the whole algorithm still works — even for data where
// no straight-line relationship exists at all
K[i][j] = kernel(x_i, x_j)
→
This is the seed of Kernel PCA

Once PCA is rewritten purely in terms of pairwise similarities, replacing the dot product with a kernel function lets PCA uncover non-linear structure — without ever explicitly building the giant feature vectors.

▶ From Gram Matrix to Kernel Matrix
The dot product is just one specific kernel. Replace it with any valid kernel function, and the algorithm discovers non-linear structure.
▶ Real-World Applications
Genomics ~20,000 genes, ~200 patients — $d/n$ ratio of 100:1, the $d \times d$ matrix is never feasible
Text mining ~50,000 vocabulary terms, ~1,000 documents — sparse, huge feature space per document
Small-sample imaging ~1,000,000 pixels, ~50 images — extreme $d \gg n$; only the $n \times n$ route is tractable
Collaborative filtering Millions of items, thousands of users — pairwise similarity matrix avoids the item-item covariance
!
Common Mistakes

✖ Thinking the trick only works for linear PCA: Every PCA step — variance, covariance, projections — can be rewritten using only dot products. Replacing the dot product with a kernel is a seamless upgrade, not a new algorithm.

✖ Forgetting to center in kernel space: When using a kernel, centering must be done algebraically on $\mathbf{K}$ (via $\mathbf{K}_c = \mathbf{K} - \mathbf{1}_n\mathbf{K} - \mathbf{K}\mathbf{1}_n + \mathbf{1}_n\mathbf{K}\mathbf{1}_n$), since the feature space may be implicit.
