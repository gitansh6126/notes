Choosing $K$ —
Balancing Fit Against Complexity

Every earlier lecture quietly assumed $K$ was handed to us. It never is. This lecture confronts the last open question in K-Means head-on: left unchecked, the objective function wants $K$ as large as possible — so something else has to want the opposite.

§1 The Trap §2 Penalty Framework §3 AIC & BIC §4 Elbow Method §5 Takeaways
01

The Problem with Minimizing the Objective Alone

If $K$ is free to grow, the minimizer is trivial: set $K = n$, give every point its own cluster, and every distance term becomes exactly zero.

Equation (1)
Within-Cluster Sum of Squares
$$\text{WCSS}(K) = \sum_{i=1}^{n} \|\mathbf{x}_i - \boldsymbol{\mu}_{z_i}\|^2$$
WCSS is monotonically non-increasing in $K$ — it always prefers more clusters
▶ Read the Problem
“The WCSS objective $\sum_i \|\mathbf{x}_i - \boldsymbol{\mu}_{z_i}\|^2$ is minimized at $K = n$, where each point is its own cluster and every distance is zero. This is a useless model: no grouping, no compression, no insight.”
▶ WCSS vs. $K$ — The monotonic decrease
$K$WCSSWhat it means
$1$ High All points in one huge cluster — maximum compression, terrible fit
$2$ Lower Two groups — better fit, but still coarse
$\vdots$ $\searrow$ Each added cluster reduces WCSS
$n$ $0$ Perfect fit, zero compression — each point is its own cluster
▶ Real-Life Analogy
It's like summarizing a book by quoting every single sentence verbatim — technically a perfect, zero-loss "summary," but it has compressed nothing and taught you nothing you didn't already have. A useful summary has to lose some detail to gain any value.
!
Watch out

✖ Never select $K$ by comparing raw WCSS values across different $K$. WCSS is monotonically non-increasing in $K$ by construction — it will always vote for more clusters, right up to the useless extreme of $K = n$.

02

Fixing It: Add a Penalty for Complexity

If the objective always prefers larger $K$, the selection criterion needs a second term that always prefers smaller $K$.

Equation (2)
Regularized Objective
$$\text{Total Cost}(K) = \underbrace{\text{WCSS}(K)}_{\text{decreases with } K} \;+\; \underbrace{\lambda \cdot \text{Penalty}(K)}_{\text{increases with } K}$$
The sum of two opposing trends produces a genuine interior minimum $K^*$
▶ Read the Equation in English
“The total cost of using $K$ clusters equals the WCSS fit term (which always falls as $K$ grows) plus a complexity penalty (which always rises as $K$ grows). Their sum has a unique minimum at $K^*$.”
▶ Two Opposing Trends Meet at $K^*$
$K^*$ sits where the total-cost curve bottoms out — not where either individual curve is smallest.
▶ Real-Life Analogy
It's the same logic behind deciding how many suitcases to pack for a trip: more suitcases let you bring more things (better "fit"), but each extra suitcase costs you real hassle at the airport. The right number balances both, rather than maximizing either alone.
03

Two Concrete Penalties: AIC and BIC

Both criteria formalize "Total Cost" using a probabilistic lens: score how well the parameters explain the data via likelihood, and subtract a penalty that grows with the number of parameters.

Equation (3)
AIC and BIC Criteria
$$\text{AIC}(K) = -2\log\mathcal{L}(K) + 2p(K)$$ $$\text{BIC}(K) = -2\log\mathcal{L}(K) + p(K)\log n$$
Both balance log-likelihood against a complexity penalty; BIC's $p(K)\log n$ grows harsher with dataset size
▶ Read the Criteria in English
“Both AIC and BIC score a model by taking the negative log-likelihood (how well the model fits the data — lower is better) and adding a penalty for the number of parameters $p(K)$. In K-Means with $d$-dimensional data, $p(K) = Kd$: each of $K$ clusters has a $d$-dimensional mean. AIC uses penalty $2p(K)$; BIC uses $p(K)\log n$, which grows with the dataset size.”
▶ AIC vs. BIC
CriterionPenalty termComplexity stanceGrows with $n$?
AIC $2p(K)$ More permissive — favors larger $K$ No
BIC $p(K)\log n$ More stringent — favors smaller $K$ Yes
$\mathcal{L}(K)$

Likelihood

Probability of data under the $K$-cluster model. Higher is better.

$p(K) = Kd$

▶ Numerical Example: AIC/BIC for K-Means
Setup: 2D data ($d=2$), $n=100$ points. Comparing $K=3$ vs $K=5$. Assume variance $\sigma^2=1$.
$K$$p(K)=Kd$WCSS$-2\log\mathcal{L} \approx \frac{\text{WCSS}}{\sigma^2}$AICBIC

→
Worth noting

Both criteria lean on "likelihood," which presumes some probabilistic model (commonly, that points scatter around their cluster mean according to a Gaussian). The penalty's job either way is the same: make additional clusters earn their keep, rather than being free.

04

The Elbow Method & Silhouette Score

Two practical heuristics that don't require a probabilistic model: the elbow method (visual) and the silhouette score (numeric).

Equation (4)
Elbow — Rate of WCSS decrease
$$\Delta \text{WCSS}(K) = \text{WCSS}(K-1) - \text{WCSS}(K)$$
Choose $K$ where adding another cluster gives diminishing returns — the "elbow" in the plot
▶ How the Elbow Method Works
Plot WCSS as a function of $K$. At first, each added cluster dramatically reduces WCSS (each new centroid captures a real mode of the data). Eventually, adding clusters only splits existing modes, giving smaller and smaller reductions. The "elbow" — the point where the plot bends — is a good candidate for $K^*$.
Equation (5)
Silhouette Score
$$s(i) = \frac{b(i) - a(i)}{\max\{a(i), b(i)\}} \quad \in [-1, 1]$$
$a(i)$ = mean distance to points in same cluster; $b(i)$ = mean distance to points in nearest other cluster
▶ Read the Equation in English
“For each point $i$, let $a(i)$ be the average distance to other points in its own cluster (cohesion), and $b(i)$ be the average distance to points in the nearest other cluster (separation). The silhouette score $s(i) = (b - a) / \max(a, b)$ ranges from $-1$ (wrong cluster) through $0$ (on boundary) to $+1$ (well-clustered). The mean silhouette over all points measures overall clustering quality.”
▶ Silhouette Score Interpretation
ScoreMeaning
$+1$ Point is far from its neighbors in other clusters — excellent assignment
$0$ Point sits on the boundary between two clusters — ambiguous assignment
$-1$ Point is closer to another cluster — likely misassigned

Elbow Method

Visual heuristic: plot WCSS vs. $K$. Pick the bend. Subjective but fast.

Silhouette Score

Numerical: $s(i) = (b-a)/\max(a,b)$. Average over all points. Pick $K$ maximizing mean silhouette.

AIC / BIC

Principled: penalized likelihood. BIC is more conservative; AIC is more permissive.

▶ Intuitive Summary
There is no single "correct" $K$ — the right number depends on the granularity you need. The elbow method is the cheapest visual check; silhouette gives a numeric score; AIC/BIC provide a principled model-selection framework. Use them together: if multiple methods agree on $K^*$, you can be confident.
05

Key Takeaways

Raw WCSS Fails

Minimizing the raw K-Means objective alone always drives $K$ toward $n$, where every point is its own (useless) cluster.

Penalty Rescues It

A workable selection rule needs a penalty term that increases with $K$, so the sum $\text{WCSS}(K) + \text{Penalty}(K)$ has a genuine interior minimum $K^*$.

Multiple Methods

AIC, BIC, elbow method, and silhouette score each offer a different tradeoff between principle and simplicity. Use them together.
