K-means clustering: Voronoi geometry, limitations, and kernel extension study notes.

K-means / Lloyd's algorithm — lecture W_L

Core geometric claim

When K-means converges, every point assigned to cluster k is closer (squared Euclidean) to μk than to any other mean. This forces the partition to be a Voronoi diagram.

assign(x) = argmin_k ‖x − μ_k‖²

K = 2

Boundary = perpendicular bisector of segment joining μ₁ and μ₂. One hyperplane.

General K

Space split into K Voronoi cells. Each cell = intersection of K−1 half-spaces.

Key property

Every Voronoi cell is convex. The algorithm cannot produce non-convex clusters.

Voronoi partition — visual

Limitation: convexity constraint

Voronoi regions are always convex → K-means fails on non-convex data structures like concentric rings or crescents. It will cut with a straight boundary regardless.

Solution: kernel K-means

The idea

Map data into a higher-dimensional feature space using a kernel function φ(x). In that space, non-convex clusters can become linearly separable → standard Voronoi logic applies there.

k(x, x') = ⟨φ(x), φ(x')⟩

The kernel trick avoids computing φ(x) explicitly — only inner products are needed.

Conceptually related to spectral clustering — both exploit similarity in a transformed space to find non-convex structure.
