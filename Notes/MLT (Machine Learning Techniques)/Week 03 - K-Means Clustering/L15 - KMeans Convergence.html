Clustering Series · Lecture 15

Convergence of K-Means —
Why Lloyd's Algorithm Always Stops

K-Means alternates between two simple steps — assign points, then recompute means — and just... stops. Nobody tells it when. This lecture proves that it has to, explains why the resting point it finds isn't always the best one, and unpacks what that means for how you actually run the algorithm.

01

The Mean Minimizes Total Squared Distance

Everything in this lecture rests on one geometric fact: given any fixed set of points, the mean is the unique point minimizing the sum of squared distances to all of them.

Equation (1)
Mean Minimizer Property
$$\boldsymbol{\mu}^* = \arg\min_{\mathbf{v}} \sum_{i=1}^{L} \|\mathbf{x}_i - \mathbf{v}\|^2 = \frac{1}{L}\sum_{i=1}^{L} \mathbf{x}_i$$
The mean of a set is the single point that minimizes total squared Euclidean distance to all members
▶ Read the Equation in English
“The point $\boldsymbol{\mu}^*$ that minimizes the sum of squared distances to points $\mathbf{x}_1, \dots, \mathbf{x}_L$ is their arithmetic mean. No other candidate — median, mode, random point — does better on this specific criterion.”
▶ Symbol Legend
SymbolMeaningExample
$\boldsymbol{\mu}^*$ Optimal center (minimizer) $\boldsymbol{\mu}^* = [1.5, 2.0]^T$
$\displaystyle\sum_{i=1}^{L}$ Sum over all $L$ points in the set $L = 10$ points
$\|\mathbf{x}_i - \mathbf{v}\|^2$ Squared Euclidean distance $\|x - v\|^2 = (x_1 - v_1)^2 + (x_2 - v_2)^2$
$\arg\min_{\mathbf{v}}$ The $\mathbf{v}$ achieving the minimum Not the minimum value, but the location
▶ Proof: Minimizing w.r.t. $\mathbf{v}$
1
Define the objective function $f(\mathbf{v})$
$\displaystyle f(\mathbf{v}) = \sum_{i=1}^L \|\mathbf{x}_i - \mathbf{v}\|^2$
↓
2
Take the gradient and set to zero
$\displaystyle \nabla f(\mathbf{v}) = -2\sum_{i=1}^L (\mathbf{x}_i - \mathbf{v}) = 0$
↓
3
Solve for $\mathbf{v}$
$\displaystyle \sum_{i=1}^L \mathbf{x}_i = L \mathbf{v} \implies \mathbf{v} = \frac{1}{L}\sum_{i=1}^L \mathbf{x}_i$
↓
4
The Hessian $2L\mathbf{I}$ is positive definite $\implies$ global minimum
$\displaystyle \boldsymbol{\mu}^* = \frac{1}{L}\sum_{i=1}^L \mathbf{x}_i$
▶ Intuition
Picture choosing a meeting spot for a group of friends scattered across a city, where the "cost" of the meetup is the sum of everyone's squared travel distance. You wouldn't pick city hall or a random friend's house — you'd pick the location that balances everyone's position, which turns out to be their average location. That balancing point, and nothing else, minimizes the total squared travel.
▶ Numerical Example
Setup: Three 1D points: $\mathbf{x} = [2, 4, 6]$. Find the minimizer.
Candidate $\mathbf{v}$$f(\mathbf{v}) = \sum (x_i - v)^2$
$v = 3$$(2-3)^2 + (4-3)^2 + (6-3)^2 = 1 + 1 + 9 = 11$
$v = 4$ (mean)$(2-4)^2 + (4-4)^2 + (6-4)^2 = 4 + 0 + 4 = 8$
$v = 5$$(2-5)^2 + (4-5)^2 + (6-5)^2 = 9 + 1 + 1 = 11$
$\displaystyle \mu^* = (2 + 4 + 6)/3 = 4$ with minimum $f(4) = 8$
02

Proof of Convergence, Step by Step

At iteration $T$, the algorithm's state is a pair: cluster assignments $Z_T$ and cluster means $\boldsymbol{\mu}_T$. One full iteration updates them in strict alternation.

Equation (2)
K-Means Objective
$$\mathcal{J}(Z, \boldsymbol{\mu}) = \sum_{i=1}^{n} \|\mathbf{x}_i - \boldsymbol{\mu}_{z_i}\|^2$$
Total squared distance from every point to its assigned cluster mean
▶ Lloyd's Algorithm Iteration
Each iteration has two halves: (A) Reassign — each point moves to the nearest mean. (B) Update — recompute each mean as the centroid of its newly assigned points.
▶ State Variables
SymbolMeaningRole
$Z_T$ Assignment vector at iteration $T$ $[z_1, z_2, \dots, z_n]$
$\boldsymbol{\mu}_T$ Set of $K$ means at iteration $T$ $\{\boldsymbol{\mu}_1, \dots, \boldsymbol{\mu}_K\}$
$\mathcal{J}(Z, \boldsymbol{\mu})$ Objective value for state $(Z, \boldsymbol{\mu})$ $\mathcal{J} = 42.5$
Equation (3)
Monotonic Decrease Chain
$$\mathcal{J}(Z_{T+1}, \boldsymbol{\mu}_{T+1}) \;\leq\; \mathcal{J}(Z_{T+1}, \boldsymbol{\mu}_T) \;\leq\; \mathcal{J}(Z_T, \boldsymbol{\mu}_T)$$
Reassigning can only help (right inequality); re-centering with the true mean can only help further (left inequality)
▶ Why the Objective Can Only Decrease
A
Reassignment: Each point moves to the closest mean. Distance either stays the same or shrinks.
$\mathcal{J}(Z_{T+1}, \boldsymbol{\mu}_T) \leq \mathcal{J}(Z_T, \boldsymbol{\mu}_T)$
↓
B
Mean update: By the Fundamental Fact, the mean minimizes squared distance for whatever set it represents.
$\mathcal{J}(Z_{T+1}, \boldsymbol{\mu}_{T+1}) \leq \mathcal{J}(Z_{T+1}, \boldsymbol{\mu}_T)$
↓
→
Chain them: $\mathcal{J}$ never increases across an iteration
$\mathcal{J}_{T+1} \leq \mathcal{J}_T$
▶ Monotonic Descent of the Objective
Each bar is the total squared distance after one full iteration. It keeps falling — sometimes in big drops, sometimes barely at all — until an iteration changes nothing and the bar flattens out (green).
▶ Numerical Example: Objective Across Iterations

▶ Intuition
It's like water finding its level: every point flowing to a lower spot only ever lowers the total, never raises it. The system might settle quickly or trickle for a while, but it never sloshes back uphill on its own.
03

Why It Must Stop, Not Just Slow Down

A monotonically decreasing sequence isn't automatically a terminating one. K-Means avoids that trap for a structural reason: there are only finitely many possible partitions.

Equation (4)
Finite State Argument
$$\#\text{partitions} = K^{\,n} < \infty \quad \land \quad \mathcal{J} \text{ strictly decreases on every change}$$
The algorithm cannot revisit a state (strict decrease $\implies$ no cycles), and there are only finitely many states

Finite State Space

Every reassignment step lands on one of a finite (if huge) number of possible partitions.

Strict Decrease

Whenever an iteration changes the assignment, the objective drops by a nonzero amount.

No Repeats

Because $\mathcal{J}$ strictly decreases, the algorithm can never revisit the same partition twice.

→
Key insight

Convergence doesn't come from clever engineering — it's a direct consequence of pairing a strictly decreasing quantity with a finite space of possible states. That combination is what rules out infinite looping.

!
Common Mistake

✖ $\mathcal{J}$ decreasing alone does not guarantee convergence — it could decrease asymptotically forever. The finite state space is what closes the argument.

✖ "K-Means converged" is not the same as "K-Means found the best clustering." A single run reaching a stable fixed point can still be sitting in a mediocre local minimum.

04

Local Minima and Practice

The proof only guarantees that the objective stops decreasing. It says nothing about which fixed point the algorithm lands on.

▶ Objective Landscape — Two Valleys
Two different initializations can roll the algorithm into two different valleys. Both are fixed points, but only one is the global optimum.

Initialization Matters

Different starting means can land in different valleys. Standard practice: run K-Means multiple times from different random starts, keep the best.

Fast in Practice

Worst-case iterations can be large, but on real datasets the objective typically flattens after a modest number of iterations.

Multiple Restarts

The fix for local minima is more restarts, not more iterations of the same run. Each run independently converges to some local minimum.

!
Watch out

"K-Means converged" is not the same claim as "K-Means found the best clustering." A single run reaching a stable fixed point can still be sitting in a mediocre local minimum — the fix is multiple restarts, not more iterations of the same run.

05

Key Takeaways

Mean Minimizes

The mean uniquely minimizes total squared distance to a fixed set — the geometric fact the entire proof leans on.

Monotonic Decrease

Both halves of a K-Means iteration — reassigning points and recomputing means — can only decrease the objective, never increase it.

Finite + Strict = Halt

Finitely many partitions plus a strictly decreasing objective means the algorithm cannot loop forever — it must reach a fixed point.
