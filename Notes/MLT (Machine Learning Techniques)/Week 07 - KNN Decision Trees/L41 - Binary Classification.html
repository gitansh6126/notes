00

The problem regression can't answer

A doctor doesn't want a number between "definitely benign" and "definitely malignant" — they want a decision. An inbox doesn't want a "spamminess score" — it wants the email sorted into the right folder. Classification exists because a huge slice of real decisions are discrete, not continuous.

Formally: binary classification is supervised learning where we map input features \(\textcolor{#5B9BD5}{x} \in \mathbb{R}^d\) to one of exactly two discrete labels \(\textcolor{#5B9BD5}{y}\), usually written as \(\{0,1\}\) or \(\{-1,1\}\). The choice of labeling scheme is just bookkeeping — mathematically interchangeable depending on what's convenient for a given algorithm.

✺ Think of it like...

A bouncer at a club door. They don't hand out a "coolness score" from 0 to 100 — they make one call: in or not in. Every input, no matter how borderline, collapses to one of two outcomes. That collapsing-to-a-decision is the entire game of classification.

What happens if we don't solve it properly? We're left forcing round pegs into square holes — using tools built for continuous prediction (regression) on problems that are fundamentally about drawing a line between two groups. As we'll see, that mismatch causes real, measurable damage.

01

A bit of history

Least-squares regression is centuries old — Gauss and Legendre were fitting continuous curves to astronomical data in the 1800s. When machine learning inherited that toolkit, the first instinct for "classify this" problems was: just reuse regression and threshold the output.

That instinct is reasonable but, as we'll prove later in this lecture, quietly broken. It set up exactly the tension this lecture resolves: regression was built to make continuous values as close as possible to a target. Classification only cares whether you land on the correct side of a boundary — magnitude is irrelevant. That mismatch in what's being optimized is the historical motivation for treating classification as its own discipline, starting from the most literal metric imaginable: just count the mistakes.

02

Engineering thinking → mathematical translation

Before writing a single equation, decide what a good scoring rule should feel like.

🛠 Engineering requirement

A mistake is a mistake — full stop. A loan application rejected by a hair should count exactly the same as one rejected by a mile. We don't want the "how wrong" to matter, only the "wrong or right."

🔢 Mathematical translation

Turn "right or wrong" into a number using a switch: compare the prediction \(\textcolor{#9B6BD9}{h}(\textcolor{#5B9BD5}{x_i})\) to the true label \(\textcolor{#5B9BD5}{y_i}\), and output exactly \(1\) if they disagree, \(0\) if they agree. No partial credit, no penalty for degree of wrongness.

03

Building the 0–1 loss, piece by piece

We never hand you the formula cold. Watch it get assembled, one justified step at a time.

1
Compare one prediction
\( \textcolor{#9B6BD9}{h}(\textcolor{#5B9BD5}{x_i}) \overset{?}{=} \textcolor{#5B9BD5}{y_i} \)
Start with the single most basic question: did the model get this one example right?
2
Turn it into a hard switch
\( \textcolor{#9B6BD9}{\mathbb{I}}(\textcolor{#9B6BD9}{h}(\textcolor{#5B9BD5}{x_i}) \textcolor{#E2574B}{\neq} \textcolor{#5B9BD5}{y_i}) \)
Wrap the comparison in the indicator function — it collapses "right/wrong" into exactly \(1\) or \(0\), nothing in between.
3
Add up every mistake
\( \textcolor{#E2574B}{\sum_{i=1}^{n}} \textcolor{#9B6BD9}{\mathbb{I}}(\textcolor{#9B6BD9}{h}(\textcolor{#5B9BD5}{x_i}) \textcolor{#E2574B}{\neq} \textcolor{#5B9BD5}{y_i}) \)
Sum the switch across all \(\textcolor{#4CAF7D}{n}\) training points — this is a raw mistake count, not yet a rate.
4
Normalize into a rate
\( \textcolor{#4CAF7D}{\dfrac{1}{n}}\textcolor{#E2574B}{\sum_{i=1}^{n}} \textcolor{#9B6BD9}{\mathbb{I}}(\textcolor{#9B6BD9}{h}(\textcolor{#5B9BD5}{x_i}) \textcolor{#E2574B}{\neq} \textcolor{#5B9BD5}{y_i}) \)
Divide by \(\textcolor{#4CAF7D}{n}\) so the result is a fraction between 0 and 1 — comparable across datasets of any size.
The 0–1 Loss Function
\[ \textcolor{#9B6BD9}{\text{Loss}}(\textcolor{#9B6BD9}{h}) \;=\; \textcolor{#4CAF7D}{\frac{1}{n}} \; \textcolor{#E2574B}{\sum_{i=1}^{n}} \; \textcolor{#9B6BD9}{\mathbb{I}}\!\left(\textcolor{#9B6BD9}{h}(\textcolor{#5B9BD5}{x_i}) \; \textcolor{#E2574B}{\neq} \; \textcolor{#5B9BD5}{y_i}\right) \]
Eq. 1.1
x, y — feature vector & true label n — number of training points h, 𝕀 — hypothesis & indicator function Σ, ≠ — sum over data, mismatch test
Conceptual pipeline: compare → flag mismatch → count → normalize → error rate
🔢 Worked example

Say \(n=5\) predictions: \(\hat y = [1,-1,1,1,-1]\) vs true \(y=[1,-1,-1,1,1]\). Points 3 and 5 disagree → 2 mistakes. \(\text{Loss}=\tfrac{2}{5}=0.4\) — the model is wrong 40% of the time.

✗ Common mistakes

Confusing loss with accuracy (they're opposites: \(\text{accuracy}=1-\text{Loss}\)); forgetting the \(1/n\) and reporting a raw count instead of a rate; assuming this loss is smooth enough for plain gradient descent.

05

Linear classifiers — drawing the line

The 0–1 loss tells us how to grade a hypothesis. It says nothing about what shape that hypothesis can take. The simplest useful choice: restrict the search to linear hypotheses.

The Linear Classifier Hypothesis
\[ \textcolor{#9B6BD9}{h}(\textcolor{#5B9BD5}{x}) \;=\; \textcolor{#9B6BD9}{\text{sign}}\!\left(\textcolor{#E0A639}{w}^{\mathsf T} \textcolor{#5B9BD5}{x}\right) \]
Eq. 1.2
x — feature vector (a data point) w — learned weight vector (a parameter) sign, h — the sign function & the hypothesis
  • The model learns one weight vector \(w\).
  • If \(\textcolor{#E0A639}{w}^{\mathsf T} \textcolor{#5B9BD5}{x} > 0\) → predict the positive class.
  • If \(\textcolor{#E0A639}{w}^{\mathsf T} \textcolor{#5B9BD5}{x} < 0\) → predict the negative class.
  • The set of points where \(\textcolor{#E0A639}{w}^{\mathsf T} \textcolor{#5B9BD5}{x} = 0\) is the decision boundary — a hyperplane (just a line, in 2D).
Conceptual pipeline: features → weighted sum → sign → class label
🔢 Worked example

Let \(\textcolor{#E0A639}{w}=[2,-1]\) and \(\textcolor{#5B9BD5}{x}=[3,4]\). Then \(w^{\mathsf T}x = (2\!\cdot\!3)+(-1\!\cdot\!4) = 2\). Since \(2>0\), \(h(x)=\text{sign}(2)=+1\) — this point is predicted to be in the positive class.

✗ Common mistakes

Forgetting a bias term \(b\) (so the boundary is forced through the origin); assuming \(w\) must be a unit vector — it doesn't, only its direction matters for the sign; treating the size of \(w^{\mathsf T}x\) as a confidence score without extra justification.

✺ Think of it like...

A property fence. \(w\) decides the fence's angle and position. Everything on one side belongs to the neighbor on the left; everything on the other side belongs to the neighbor on the right. The fence itself — \(w^{\mathsf T}x=0\) — is neutral territory, belonging to neither.

06

Why the 0–1 loss is secretly brutal

It's the most honest loss function — but honesty comes at a steep computational price.

📉 Non-differentiable

Picture the loss as you nudge one point across the boundary: it sits flat at 0, then jumps straight to 1 the instant it crosses — a staircase, not a slope.

Flat → vertical jump → flat. Gradient descent has nothing to grab onto: the slope is 0 almost everywhere and undefined at the jump.

🧩 NP-hard

Finding the hyperplane that minimizes the exact mistake count means, in the worst case, checking combinatorially many ways to arrange the boundary among the points. There's no known efficient (polynomial-time) algorithm that solves this exactly for general datasets.

⚠ So what do we actually do?

We don't minimize the 0–1 loss directly. Instead — foreshadowing later lectures — we minimize a smooth surrogate loss that behaves similarly but is differentiable, and is what gradient descent can actually work with.

07

From math to executable steps

Annotated, runnable-shape pseudocode for prediction and for computing the loss.

# predict() applies the linear hypothesis h(x) = sign(w^T x) def predict(x, w): score = dot(w, x) # this is w^T x if score > 0: return +1 # right of the boundary else: return -1 # left of the boundary # zero_one_loss() implements Loss(h) = (1/n) * sum( I(h(x_i) != y_i) ) def zero_one_loss(X, y, w): n = len(X) errors = 0 for i in range(n): if predict(X[i], w) != y[i]: # the indicator I(.) errors += 1 return errors / n # the 1/n normalizer

09

Why not just reuse linear regression?

This is the trap the whole lecture is building toward. It seems reasonable: fit a regression line, threshold it. Here's precisely why that's unprincipled.

✺ Think of it like...

A teacher who only needs to know "pass or fail," but is secretly grading on the exact score. One student aces the exam with 100% — far beyond the 40% pass mark. The teacher, obsessed with matching exact scores, drags the whole passing curve upward to "explain" that 100%, even though the pass/fail outcome hadn't changed at all. Everyone near the original boundary is now misclassified, purely because of a student who was already, unambiguously, a pass.

What's happening mathematically: linear regression minimizes squared distance between prediction and target. It is sensitive to magnitude. If a point sits very far from the boundary, regression tilts the whole line to reduce that large squared error — even though that point's classification was already correct and needed zero help.

Squared-error loss cares how far you are from a number. 0–1 loss cares only which side of a line you're on. Those are different questions, and optimizing for one does not reliably answer the other.

✓ 0–1 loss / linear classifier

An extreme, obviously-correct point contributes exactly 0 to the loss — same as a point that barely crossed the line correctly. The boundary doesn't move because of it.

✗ Linear regression

That same extreme point creates a huge squared error, so the fitted line rotates to accommodate it — potentially misclassifying points that were previously correct.

12

Always · Sometimes · Never

Most exam traps live in the gap between these three columns.

✓ Always true

  • 0–1 loss is always between 0 and 1 — it's a rate, not a raw count.
  • The decision boundary \(w^{\mathsf T}x=0\) is a hyperplane.
  • 0–1 loss is always piecewise-flat, with jumps only at misclassification points.

≈ Sometimes true

  • Labels are \(\{0,1\}\) or \(\{-1,1\}\) — the choice depends on the algorithm's convenience, not a fixed rule.
  • A boundary that perfectly separates the training data sometimes fails to generalize to new data.
  • Rounding a regression output sometimes happens to match the 0–1-optimal boundary — by luck, not by design.

✗ Never true

  • 0–1 loss is never differentiable at the decision boundary — never feed it straight into gradient descent.
  • Exact 0–1 loss minimization is never tractable for general datasets (it's NP-hard).
  • Linear regression's squared-error objective is never equivalent to counting misclassifications.
13

Examiner trap library

Written the way an exam would phrase them — including the tempting wrong answer.

"Since accuracy = 1 − 0–1 loss, can't we just gradient-descend on accuracy directly?"
Trap: No — accuracy is just a linear transform of the same step function. It inherits the exact same non-differentiability problem.
An MCQ option claims: "The 0–1 loss is convex."
Trap: False. It isn't even continuous, so convexity doesn't apply the way test-takers assume when they see a "loss function."
"We can just use linear regression for classification by rounding the output."
Trap: Technically runnable, but unprincipled — sensitive to outlier magnitude, as shown in Section 09. Exams often ask you to justify why this fails, not just state that it does.
Reverse phrasing: "Which loss function directly counts the number of errors?"
Trap: Students overthink and reach for squared loss. The answer is the plain 0–1 loss — it's defined to do exactly that, nothing fancier.
⚡ Fast shortcut

See "count the mistakes" or "accuracy" anywhere in a question about loss functions → jump straight to \(\text{Loss}(h)=\frac1n\sum \mathbb{I}(h(x_i)\neq y_i)\). Don't overthink it.
