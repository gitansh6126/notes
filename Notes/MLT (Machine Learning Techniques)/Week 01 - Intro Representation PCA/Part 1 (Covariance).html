01
Covariance Matrix Calculation
the d×d map of how every feature pair moves together
▶
Week 1 › PCA Numericals › Covariance Matrix
Core Intuition
Think of the covariance matrix as a "friendship map" between features. The diagonal tells you how much each feature varies on its own (its variance); the off-diagonal entries tell you whether pairs of features rise and fall together (positive), move oppositely (negative), or are unrelated (~0).
Formula Explanation
C = (1/n) X XT   (X already mean-centered)
Xd × n centered data matrix — each column is one data point
Cd × d covariance matrix — symmetric
Cᵢᵢvariance of feature i (diagonal)
Cᵢⱼcovariance between feature i and j (off-diagonal)
Step-by-Step Algorithm
1
Compute mean vector μ (one mean per feature)
→
2
Center data: subtract μ from every column
→
3
Multiply centered X by its transpose
→
4
Divide every entry by n (or n−1 for sample covariance)
Beginner Example
Beginner

Feature A: [1, 3, 5]   Feature B: [2, 4, 6]. Find C.

μ_A = 3, μ_B = 4

Centered: A' = [−2,0,2], B' = [−2,0,2]

Var(A) = (4+0+4)/3 = 2.67  ·  Var(B) = 2.67  ·  Cov(A,B) = (4+0+4)/3 = 2.67

C = [[2.67, 2.67], [2.67, 2.67]]
Medium Example
Medium

3 features, 4 points: A=[2,4,6,8], B=[1,3,2,4], C=[5,5,7,7]. Find the 3×3 covariance matrix.

μ_A=5, μ_B=2.5, μ_C=6

Center each row, then compute all 6 unique entries (3 variances + 3 covariances) using the centered rows

Var(A)=6.67, Var(B)=1.25, Var(C)=1.0, Cov(A,B)=2.5, Cov(A,C)=1.33, Cov(B,C)=0.5

C is symmetric — fill diagonal + upper triangle, mirror the rest
PYQ-Level Example
PYQ Style

A dataset has 2 features with covariance matrix C = [[4, −2], [−2, 9]]. Interpret the sign and magnitude of the off-diagonal entry, and state whether the features are more or less variable than each other.

Off-diagonal = −2 → the two features move in opposite directions on average

Var(feature 2) = 9 > Var(feature 1) = 4 → feature 2 is more spread out

Negative, moderate association; feature 2 more variable
Challenge Question
Challenge

A 3×3 covariance matrix is symmetric with diagonal [5, 8, 3]. You're told Cov(x₁,x₂) = 2 and the trace of the matrix is 16. Two entries are missing. Can you find them, or is there insufficient information?

Show reasoning
Trace = sum of diagonal = 5+8+3 = 16 ✓ (consistent, but trace only constrains the diagonal). Cov(x₁,x₃) and Cov(x₂,x₃) are NOT determined by the trace or by Cov(x₁,x₂) — they are independent pieces of information. Insufficient information to find them; this is a common trap testing whether you understand which quantities are actually linked.
Common Mistakes
!
  • Forgetting to mean-center the data before multiplying XXᵀ
  • Dividing by n when the question asks for sample covariance (n−1), or vice versa
  • Confusing XXᵀ (d×d) with XᵀX (n×n) — wrong matrix shape entirely
  • Sign errors when multiplying negative centered values
Exam Shortcuts
✓
  • C is always symmetric — compute only the upper triangle and mirror it
  • Diagonal entries are just variances — reuse your Topic 3 workflow directly
  • If the question says "data is already centered," skip straight to step 3
Time-Saving Tricks
→
  • Use C = (1/n)ΣxᵢxᵢT − μμᵀ to avoid writing out centered values by hand
  • Tabulate Σx, Σy, Σxy for every pair once — reuse across all matrix entries
Active Recall
Why must a covariance matrix always be symmetric?
Because Cov(x,y) = Cov(y,x) by definition of the product of centered values — order doesn't matter in multiplication.
What does a diagonal entry of C represent?
The variance of that single feature — how spread out it is on its own.
What must be true about the data before computing C = (1/n)XXᵀ?
X must already be mean-centered — each row/feature must have mean zero.
PYQ Frequency

★★★★★  — appears in nearly every term's PCA numerical section, often as the opening question.

02
Mean Centering
shifting the data cloud so it's centered at the origin
▶
Week 1 › PCA Numericals › Mean Centering
Core Intuition
Imagine re-parking a cluster of cars so their average position sits exactly on a lamp post marked zero — the cars' positions relative to each other don't change, only where the whole cluster sits. That's mean centering: it removes location, keeps shape.
Formula Explanation
x'ᵢ = xᵢ − μ,   μ = (1/n) Σᵢ xᵢ
xᵢoriginal i-th data point (or value)
μmean, computed separately per feature
x'ᵢcentered value — mean of all x'ᵢ is always 0
Step-by-Step Algorithm
1
Identify each feature (row) separately
→
2
Compute mean of that feature across all n points
→
3
Subtract the mean from every value in that feature
→
4
Check: centered values must sum to 0
Beginner Example
Beginner

Center: [4, 6, 8, 10, 12]

μ = (4+6+8+10+12)/5 = 8

Centered = [−4, −2, 0, 2, 4]
Medium Example
Medium

Center a 2-feature, 4-point matrix: A=[3,5,7,9], B=[10,20,15,25]

μ_A = 6, μ_B = 17.5

A' = [−3,−1,1,3]  ·  B' = [−7.5, 2.5, −2.5, 7.5]
PYQ-Level Example
PYQ Style

After centering a feature, a student gets [−2, −1, 0, 1, 3]. Is this centering correct?

Sum = −2−1+0+1+3 = 1 ≠ 0

Incorrect — a properly centered feature must sum to exactly 0
Challenge Question
Challenge

You're given centered values [−3, 1, 2] and told the original mean was 10. Reconstruct the original data — and prove algebraically why mean-centered data always sums to zero.

Show reasoning
Original = centered + μ = [7, 11, 12]. Proof: Σ(xᵢ−μ) = Σxᵢ − nμ = Σxᵢ − n·(Σxᵢ/n) = Σxᵢ − Σxᵢ = 0 for any dataset.
Common Mistakes
!
  • Centering along the wrong axis (subtracting a row mean from a column, or vice versa)
  • Reusing one global mean across all features instead of computing one mean per feature
  • Confusing mean centering with full standardization (also dividing by σ)
Exam Shortcuts
✓
  • Sum-to-zero is a free correctness check — always verify it before moving on
  • Compute μ once and reuse it for both centering and the variance/covariance steps that usually follow
Time-Saving Tricks
→
  • For symmetric-looking data, eyeball the midpoint as μ instead of summing manually
Active Recall
What does mean centering NOT change about the data?
The shape/spread of the data — only its location shifts.
What's a quick way to check if centering was done correctly?
The centered values must sum to zero.
PYQ Frequency

★★★★☆  — rarely asked alone, but appears as a required sub-step inside almost every covariance/PCA question.

03
Variance Calculation
how spread out a single feature is
▶
Week 1 › PCA Numericals › Variance
Core Intuition
Variance answers: "on average, how far do points wander from the center?" Squaring the distances means far-away points count extra — a few outliers can inflate variance a lot more than many small deviations.
Formula Explanation
σ² = (1/n) Σᵢ (xᵢ − μ)²
ndivide by n for population variance, n−1 for sample variance
μmean of the feature
alt formσ² = E[x²] − (E[x])² — often faster with integers
Step-by-Step Algorithm
1
Compute mean μ
→
2
Find each deviation (xᵢ − μ)
→
3
Square every deviation
→
4
Average the squared deviations (÷n or ÷n−1)
Beginner Example
Beginner

Find variance of [2, 4, 6] (population)

μ = 4  ·  deviations = [−2,0,2]  ·  squared = [4,0,4]

σ² = 8/3 = 2.67
Medium Example
Medium

Find the sample variance (n−1) of [5, 9, 12, 14]

μ = 10  ·  deviations = [−5,−1,2,4]  ·  squared = [25,1,4,16]  ·  sum = 46

σ² = 46/3 = 15.33 (note: n−1, not n)
PYQ-Level Example
PYQ Style

A 3-feature dataset's second row is [6, 10, 8, 12]. As part of building the covariance matrix, find Var(feature 2) using the shortcut identity.

E[x] = 9, E[x²] = (36+100+64+144)/4 = 86

σ² = 86 − 81 = 5
Challenge Question
Challenge

A dataset [a, 6, 10, 14] has population variance 14 and mean 10. Find the missing value a.

Show reasoning
Mean=10 → a+6+10+14=40 → a=10. Check variance: deviations=[0,−4,0,4], squared=[0,16,0,16], sum=32, σ²=32/4=8 ≠14. So a=10 fails — must solve simultaneously: (a+30)/4=10 → a=10 is forced by the mean, meaning the variance constraint is then automatically fixed by the data, not a free choice — this shows the two constraints (mean and variance) can conflict, and the question is testing whether you check consistency rather than solving blindly.
Common Mistakes
!
  • Forgetting to square deviations (positive and negative deviations then wrongly cancel to ~0)
  • Using n instead of n−1 (or vice versa) against the question's explicit instruction
  • Reporting standard deviation (√σ²) when variance was asked for
Exam Shortcuts
✓
  • Use σ² = E[x²] − (E[x])² when numbers are small integers — fewer subtraction steps
  • If deviations are already symmetric around 0, squared terms mirror — halve the work
Time-Saving Tricks
→
  • Build a quick table of x, x², running sums before applying any formula — avoids recomputation
Active Recall
Why do we square the deviations instead of just averaging them?
Because raw deviations always sum to zero — squaring prevents positive and negative deviations from cancelling out.
When should you divide by n−1 instead of n?
When computing sample variance (estimating population variance from a sample) rather than the exact population variance.
PYQ Frequency

★★★★☆  — a constant building block, tested directly and as part of covariance-matrix questions.

04
Covariance Calculation
how two variables move together
▶
Week 1 › PCA Numericals › Pairwise Covariance
Core Intuition
If two people's moods always swing the same way — happy together, sad together — they have positive covariance. If one's up when the other's down, that's negative covariance. No pattern at all means covariance near zero.
Formula Explanation
cov(x,y) = (1/n) Σᵢ (xᵢ−μx)(yᵢ−μy)
μx, μymeans of x and y respectively
alt formcov(x,y) = E[xy] − E[x]E[y]
symmetrycov(x,y) = cov(y,x) always
Step-by-Step Algorithm
1
Compute μx and μy
→
2
Pair up deviations (xᵢ−μx)(yᵢ−μy)
→
3
Sum the products
→
4
Divide by n (or n−1)
Beginner Example
Beginner

x=[1,2,3,4], y=[2,4,6,8]. Find cov(x,y)

μx=2.5, μy=5  ·  deviations x'=[−1.5,−0.5,0.5,1.5], y'=[−3,−1,1,3]

products = [4.5, 0.5, 0.5, 4.5], sum=10

cov(x,y) = 10/4 = 2.5
Medium Example
Medium

x=[3,7,9,5,6], y=[8,4,3,9,6]. Find cov(x,y) using the alt formula

E[x]=6, E[y]=6, Σxy = 24+28+27+45+36 = 160, E[xy]=32

cov(x,y) = 32 − 36 = −4 (negative relationship)
PYQ-Level Example
PYQ Style

Two features have cov(x,y) = 12, Var(x)=16, Var(y)=9. Classify the strength of their linear relationship without computing correlation directly (that's Topic 7) — just describe direction and rough intensity.

Sign is positive → they move together. Magnitude (12) is close to √(Var(x)·Var(y)) = √144 = 12 → very strong association

Strong positive relationship (covariance close to its own theoretical maximum)
Challenge Question
Challenge

Prove that cov(x,x) = Var(x), using the definitions from Topics 3 and 4.

Show reasoning
cov(x,x) = (1/n)Σ(xᵢ−μx)(xᵢ−μx) = (1/n)Σ(xᵢ−μx)² = Var(x). The diagonal of any covariance matrix is literally each feature's covariance with itself — which is exactly its variance.
Common Mistakes
!
  • Mixing up which mean subtracts from which variable
  • Sign slips when multiplying two negative deviations
  • Forgetting cov(x,y) = cov(y,x), redoing work unnecessarily
Exam Shortcuts
✓
  • The alt formula E[xy]−E[x]E[y] avoids writing out every deviation pair
  • cov(x,x) = Var(x) — never recompute a diagonal entry from scratch
Time-Saving Tricks
→
  • Tabulate x, y, xy, x², y² in one pass — reuse the same table for variance and covariance
Active Recall
What does a covariance of 0 mean (and NOT mean)?
It means no linear relationship — but it does NOT rule out a non-linear relationship between the variables.
How does cov(x,x) relate to variance?
They're identical — variance is just a variable's covariance with itself.
PYQ Frequency

★★★★☆  — frequently tested standalone and as the off-diagonal engine of covariance-matrix questions.

05
Matrix Construction
organizing raw data before any formula can be applied
▶
Week 1 › PCA Numericals › Matrix Construction
Core Intuition
Before you can search a filing cabinet, you first have to alphabetize the files. Likewise, before any PCA formula works, you have to decide — consistently — which axis is "features" and which is "data points." Get the orientation wrong and every downstream formula silently breaks.
Formula Explanation
X ∈ ℝd×n    — column i is data point xᵢ ∈ ℝᵈ
dnumber of features (rows in this course's convention)
nnumber of data points (columns)
Xᵀn × d — the "flipped" convention some textbooks use instead
Step-by-Step Algorithm
1
List what's being measured (features) vs. how many were measured (samples)
→
2
Decide orientation (this course: features = rows, samples = columns)
→
3
Fill in values consistently, one column per sample
→
4
Sanity-check shape against whatever formula comes next
Beginner Example
Beginner

3 samples, 2 features (height, weight): (150,50), (160,55), (170,60). Build X.

X = [[150,160,170],[50,55,60]]  (2×3, d=2, n=3)
Medium Example
Medium

A table has 4 rows (students) and 3 columns (marks in 3 subjects). Construct X in this course's convention, and also give Xᵀ.

Features = subjects (3), samples = students (4) → X should be 3×4, meaning the raw table (4×3) must be transposed

X = tableT (3×4); Xᵀ = original table (4×3)
PYQ-Level Example
PYQ Style

You compute C = XXᵀ and get a 5×5 matrix. If your raw table had 5 columns and 20 rows, was X built correctly for this course's convention?

C = XXᵀ is d×d. Getting 5×5 means d=5, so X must be 5×n. If the raw table was 20 rows × 5 columns, X = tableT (5×20) — correct only if that transpose was actually applied

Correct only if X = (raw table)ᵀ was used, not the raw table directly
Challenge Question
Challenge

An unlabeled matrix is 6×50. You're told the covariance matrix computed from it should describe relationships between 6 sensors. Which orientation is X, and how do you know?

Show reasoning
Since C must be d×d = 6×6 (one row/column per sensor), and C=(1/n)XXᵀ requires X to be d×n = 6×50, the given 6×50 matrix is already in the correct orientation (features=6 rows, samples=50 columns) — no transpose needed.
Common Mistakes
!
  • Using the raw table's natural orientation without checking which convention the formula expects
  • Switching orientation midway through a multi-part question
  • Forgetting to transpose when the raw table lists samples as rows (very common in spreadsheets)
Exam Shortcuts
✓
  • Check the expected output shape (e.g. C must be d×d) and work backward to confirm X's orientation
Time-Saving Tricks
→
  • Label rows and columns explicitly the moment you write X down — prevents mid-solution confusion
Active Recall
In this course's convention, what do the rows of X represent? The columns?
Rows = features (dimensions), columns = individual data points/samples.
How can you verify X's orientation without re-reading the problem?
Check that a downstream formula's output shape (like C being d×d) matches — work backward from the expected result.
PYQ Frequency

★★★☆☆  — rarely the whole question, but a silent prerequisite that causes wrong answers elsewhere if skipped.

06
Standardization vs Normalization
two different ways to rescale features — don't mix them up
▶
Week 1 › PCA Numericals › Standardization vs Normalization
Core Intuition
Standardization is like grading on a curve — it tells you how many standard deviations above or below average you are, regardless of the original units. Normalization is like compressing everyone's height into a 0–1 "shortest to tallest" scale — it only cares about relative position between the min and max.
Formula Explanation
standardize: z = (x − μ) / σ   |   normalize: x' = (x − min) / (max − min)
AspectStandardizationNormalization
Resulting rangeUnbounded (typically ~[−3, 3])Exactly [0, 1]
Resulting mean / spreadMean = 0, Std = 1Min = 0, Max = 1
Sensitive to outliers?Less — outliers pull σ, dampening effectVery — a single outlier compresses everything else
Preferred before PCA?Yes — PCA assumes comparable variance scalesRarely used before PCA
NeedsMean and standard deviationMinimum and maximum
Step-by-Step Algorithm
1
Standardization: find μ and σ
→
2
Apply z=(x−μ)/σ to every value
1
Normalization: find min and max
→
2
Apply x'=(x−min)/(max−min) to every value
Beginner Example
Beginner

Standardize [2, 4, 6, 8] (μ=5, σ=2.236)

z ≈ [−1.34, −0.45, 0.45, 1.34]
Medium Example
Medium

For the same data [2,4,6,8], compute normalization too, and compare

min=2, max=8, range=6

Normalized = [0, 0.33, 0.67, 1]  vs  Standardized ≈ [−1.34,−0.45,0.45,1.34]
PYQ-Level Example
PYQ Style

Two features are measured in centimeters and kilometers respectively. Which technique should you apply before PCA, and why?

PCA is scale-sensitive — a feature with naturally larger numeric range will dominate variance purely due to units

Standardization — it removes the unit/scale dependence so both features contribute fairly
Challenge Question
Challenge

Data = [1, 2, 3, 4, 100] (one extreme outlier). Compute both standardized and normalized versions of the value "4", and explain why they look so different.

Show reasoning
μ≈22, σ≈39.2 → z(4) ≈ (4−22)/39.2 ≈ −0.46 (barely distinguishable from the other small values). min=1,max=100 → normalized(4) = 3/99 ≈ 0.03 (crushed near 0). The outlier inflates the normalization range far more than it inflates σ, showing why normalization is fragile to outliers.
Common Mistakes
!
  • Using the two terms interchangeably — they are NOT the same operation
  • Dividing by variance instead of standard deviation when standardizing
  • Applying normalization before PCA out of habit from other ML pipelines
Exam Shortcuts
✓
  • Memory anchor: "Standardize → Std = 1" and "Normalize → [0,1] raNge"
  • If units differ across features and PCA is mentioned → standardization, almost always
Time-Saving Tricks
→
  • Compute μ, σ, min, and max in one pass over the data if a question might ask for either
Active Recall
Which technique guarantees output strictly between 0 and 1?
Normalization (min-max scaling).
Why is standardization generally preferred before PCA?
PCA finds directions of maximum variance — if features have different units/scales, the ones with naturally larger numbers dominate unfairly unless standardized.
PYQ Frequency

★★★★★  — a favorite conceptual + numeric combo question, tested almost every term.

07
Correlation Matrix vs Covariance Matrix
the same relationships, with and without the units removed
▶
Week 1 › PCA Numericals › Correlation vs Covariance Matrix
Core Intuition
Covariance tells you the direction of a relationship but its size depends on units — covariance in kilometers looks huge next to covariance in centimeters, even for identical relationships. Correlation strips the units away, always landing in [−1, 1], so you can compare relationship strength fairly across any pair of variables.
Formula Explanation
corr(x,y) = cov(x,y) / (σx · σy)
Rcorrelation matrix — same shape as C, always has 1's on the diagonal
σx, σystandard deviations of x and y
rangeevery entry of R lies in [−1, 1]
AspectCovariance MatrixCorrelation Matrix
Diagonal valuesVariances (can be any positive number)Always exactly 1
Range of entriesUnbounded, depends on unitsAlways [−1, 1]
Affected by scale/units?Yes — changes if you rescale a featureNo — scale-invariant
What it needsRaw (centered) dataCovariance matrix + each std deviation
Best forFeeding directly into PCA's eigen-stepComparing strength of relationships across differently-scaled variables
Step-by-Step Algorithm
1
Compute covariance matrix C
→
2
Compute std deviation σ for each feature (√ of diagonal)
→
3
Divide entry Cᵢⱼ by σᵢ·σⱼ
→
4
Confirm diagonal is exactly 1
Beginner Example
Beginner

cov(x,y)=6, Var(x)=4, Var(y)=9. Find corr(x,y)

σx=2, σy=3

corr(x,y) = 6/(2×3) = 1.0 (perfect positive relationship)
Medium Example
Medium

C = [[4, 2, −1], [2, 9, 3], [−1, 3, 1]]. Find the full correlation matrix R.

σ = [2, 3, 1]

R₁₂ = 2/(2×3)=0.33, R₁₃=−1/(2×1)=−0.5, R₂₃=3/(3×1)=1.0

R = [[1, 0.33, −0.5], [0.33, 1, 1.0], [−0.5, 1.0, 1]]
PYQ-Level Example
PYQ Style

Given only R = [[1, 0.8],[0.8, 1]] and Var(x)=25, Var(y)=16, reconstruct cov(x,y).

σx=5, σy=4

cov(x,y) = corr × σx × σy = 0.8 × 5 × 4 = 16
Challenge Question
Challenge

Can two datasets have identical correlation matrices but completely different covariance matrices? Justify with an example.

Show reasoning
Yes. Correlation is scale-invariant, so simply rescaling every value of one feature (e.g., converting cm to km) changes every covariance entry involving that feature, but leaves the correlation matrix completely unchanged, since the σ's in the denominator rescale by exactly the same factor.
Common Mistakes
!
  • Dividing by variance instead of standard deviation when converting covariance to correlation
  • Forgetting the correlation matrix diagonal must be exactly 1 — a fast self-check
  • Treating a large covariance value as automatically "strong" without checking the corresponding correlation
Exam Shortcuts
✓
  • Diagonal of R is always 1 — never compute it, just write it in
  • R is symmetric just like C — only compute the upper triangle
Time-Saving Tricks
→
  • If only R and the variances are given (no raw data), you can still fully reconstruct C — no need to ask for original data
Active Recall
Why is every diagonal entry of a correlation matrix exactly 1?
Because corr(x,x) = cov(x,x)/(σx·σx) = Var(x)/Var(x) = 1.
What single quantity do you need to convert an entire covariance matrix into a correlation matrix?
Just the standard deviation of each feature (the square roots of the diagonal entries).
PYQ Frequency

★★★★★  — a high-frequency question, often paired directly with Topic 6 in the same exam question.
