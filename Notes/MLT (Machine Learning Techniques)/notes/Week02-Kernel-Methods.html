Kernel Methods & the Kernel Trick

Kernel methods enable linear algorithms to operate in a high-dimensional feature space without explicitly computing coordinates — by replacing dot products with kernel functions.

§1 Motivation §2 Kernel Definition §3 Popular Kernels §4 Kernel Trick §5 Kernel Matrix
01

Why Kernels?

Many real-world datasets are not linearly separable in their original space. The kernel trick lets us implicitly map data to a higher-dimensional space where separation becomes linear — without ever computing the mapping.

▶ The Core Idea
The kernel $K(\mathbf{x}_i, \mathbf{x}_j) = \phi(\mathbf{x}_i)^\mathsf{T} \phi(\mathbf{x}_j)$ lets us work in $\R^m$ while computing only in $\R^d$.
⚠
Why explicit mapping fails

For a polynomial of degree $p$ in $d$ dimensions, the explicit feature space dimension is $O(d^p)$. For $d = 100, p = 4$, that's roughly $4.5$ million dimensions — impossible to compute directly.

▶ Intuition
Imagine you have a crumpled paper ball with red dots on one side and blue on the other. In 2D, you can't draw a line to separate them. But if you unfold the paper (map to 3D), a simple plane separates them perfectly. The kernel trick does this unfolding implicitly.
!
Common Mistake

✖ Thinking kernels are a specific algorithm: Kernels are a technique that can be applied to any algorithm expressed only in terms of dot products (SVMs, PCA, regression, etc.).

02

What is a Kernel?

A kernel is a symmetric, positive semi-definite function that computes a dot product in some (possibly infinite-dimensional) feature space.

Equation (1)
Kernel Definition
$$ K(\mathbf{x}_i, \mathbf{x}_j) = \phi(\mathbf{x}_i)^\mathsf{T} \phi(\mathbf{x}_j) $$
A valid kernel corresponds to an inner product in some feature space $\mathcal{H}$
▶ Read the Equation in English
“A kernel $K$ applied to two points $\mathbf{x}_i$ and $\mathbf{x}_j$ equals the dot product of their images $\phi(\mathbf{x}_i)$ and $\phi(\mathbf{x}_j)$ in the feature space $\mathcal{H}$ — without ever explicitly computing $\phi$.”
▶ Symbol Legend
SymbolMeaningExample
$K(\mathbf{x}_i, \mathbf{x}_j)$ Kernel function — similarity measure $K(\mathbf{x}_i, \mathbf{x}_j) \in \R$
$\phi(\mathbf{x})$ Feature map to $\R^m$ or $\mathcal{H}$ Maps $d$-dim to $m$-dim
$\mathcal{H}$ Reproducing Kernel Hilbert Space (RKHS) Possibly infinite-dimensional
▶ Intuition
Think of a kernel as a smart similarity measure. Instead of "how close are these points in original space?", it answers "how close would they be after a clever transformation?". The kernel function directly computes this transformed similarity.
▶ Mercer's Theorem — When is a Function a Valid Kernel?
1
$K$ must be symmetric: $K(\mathbf{x}_i, \mathbf{x}_j) = K(\mathbf{x}_j, \mathbf{x}_i)$
$\checkmark$
↓
2
For any set of points, the kernel matrix $\mathbf{K}_{ij} = K(\mathbf{x}_i, \mathbf{x}_j)$ must be positive semi-definite
$\mathbf{K} \succeq 0$

If both hold, $\exists\ \phi$ such that $K(\mathbf{x}_i, \mathbf{x}_j) = \phi(\mathbf{x}_i)^\mathsf{T} \phi(\mathbf{x}_j)$

03

Common Kernel Functions

Three kernels dominate practice: polynomial for finite feature spaces, RBF/Gaussian for infinite-dimensional feature spaces, and sigmoid for neural-network-like behavior.

Equation (2)
Polynomial Kernel
$$ K(\mathbf{x}_i, \mathbf{x}_j) = (\mathbf{x}_i^\mathsf{T} \mathbf{x}_j + c)^p $$
$p$ = degree, $c$ = constant term. Maps to $O(d^p)$ features. When $c=0$, it's the homogeneous polynomial kernel.
Equation (3)
RBF (Gaussian) Kernel
$$ K(\mathbf{x}_i, \mathbf{x}_j) = \exp\!\left( -\frac{\|\mathbf{x}_i - \mathbf{x}_j\|^2}{2\sigma^2} \right) $$
$\sigma$ controls the width. Corresponds to an infinite-dimensional feature space. Also written as $K = \exp(-\gamma \| \mathbf{x}_i - \mathbf{x}_j\|^2)$ with $\gamma = 1/(2\sigma^2)$.
▶ Read the Equations in English
“The polynomial kernel raises the dot product (plus constant) to power $p$, creating interactions up to degree $p$. The RBF kernel measures similarity as an exponentially decaying function of squared distance — points close together have $K \approx 1$, points far apart have $K \approx 0$.”
▶ Kernel Parameters
ParameterControlsLow valueHigh value
$p$ (degree) Complexity of polynomial interactions $p=1$ (linear) $p=5$ (highly non-linear)
$\sigma$ / $\gamma$ Width of RBF influence Large $\sigma$ (smooth) Small $\sigma$ (wiggly)
$c$ Bias / constant term in polynomial $c=0$ (homogeneous) $c=1$ (includes bias)
▶ Numerical Example
Setup: Two points $\mathbf{x}_1 = [1, 2]$, $\mathbf{x}_2 = [3, 4]$. Compute polynomial kernel with $p=2, c=0$.

▶ RBF Example
Setup: Same points, $\sigma = 2$. Compute $\|\mathbf{x}_1 - \mathbf{x}_2\|^2 = (1-3)^2 + (2-4)^2 = 8$.

i
Interpreting the RBF value

$K \approx 0.368$ means the points are moderately far apart relative to $\sigma = 2$. If $\sigma$ were smaller, the similarity would drop faster — the kernel becomes more local.

▶ Kernel Selection Guide
Linear ($p=1$) Use when data is already high-dimensional or approximately linearly separable.
Polynomial ($p=2,3$) Use when you expect feature interactions of bounded degree.
RBF/Gaussian Default choice — universal approximator, handles most non-linear patterns.
Sigmoid Use for neural-network-like behavior; less common in practice.
04

The Kernel Trick

The kernel trick replaces every dot product in an algorithm with a kernel function. This lets the algorithm work in the high-dimensional feature space at the computational cost of the original space.

Equation (4)
The Kernel Trick
$$ \langle \phi(\mathbf{x}_i), \phi(\mathbf{x}_j) \rangle \;\longrightarrow\; K(\mathbf{x}_i, \mathbf{x}_j) $$
Replace all inner products with kernel evaluations — the algorithm never sees $\phi(\mathbf{x})$
▶ Read the Substitution in English
“Wherever an algorithm uses a dot product between two feature vectors $\phi(\mathbf{x}_i)$ and $\phi(\mathbf{x}_j)$, replace it with the kernel $K(\mathbf{x}_i, \mathbf{x}_j)$ — a single scalar computation in the original space.”
▶ Dual SVM as an Example
1
Original SVM dual uses dot products: $\max \sum \alpha_i - \frac12 \sum\sum \alpha_i \alpha_j y_i y_j \langle \mathbf{x}_i, \mathbf{x}_j \rangle$
$\langle \mathbf{x}_i, \mathbf{x}_j \rangle$
↓ Replace with kernel
2
Kernelized SVM dual: $\max \sum \alpha_i - \frac12 \sum\sum \alpha_i \alpha_j y_i y_j K(\mathbf{x}_i, \mathbf{x}_j)$
$K(\mathbf{x}_i, \mathbf{x}_j)$
↓ Decision function
3
$f(\mathbf{x}) = \sum_i \alpha_i y_i K(\mathbf{x}_i, \mathbf{x}) + b$ — still only needs kernel evaluations
$K(\mathbf{x}_i, \mathbf{x})$
✓
Why it's a "trick"

The computational cost of the kernelized algorithm depends on $n$ and $d$ (original space), not on $m$ (feature space). For RBF, $m$ is infinite — yet $K$ costs $O(d)$ to compute.

i
What the trick means

Any algorithm that can be re-written solely in terms of dot products can be kernelized. This includes SVMs, PCA (kernel PCA), ridge regression, perceptron, and more — enabling non-linear versions with minimal code change.

!
Common Mistakes

✖ Using non-PSD kernels: If $K$ is not positive semi-definite, the optimization problem becomes non-convex and may not converge.

✖ Applying kernel trick to primal algorithms: The algorithm must be expressible entirely in dot products. Primal gradient-based methods typically cannot be kernelized.

05

The Kernel Matrix (Gram Matrix)

The kernel matrix $\mathbf{K}$ (also called the Gram matrix) collects all pairwise kernel evaluations. Its properties determine whether a kernel is valid and how well-conditioned the learning problem will be.

Equation (5)
Kernel Matrix
$$ \mathbf{K}_{ij} = K(\mathbf{x}_i, \mathbf{x}_j) \quad \text{for } i,j = 1,\dots,n $$
An $n \times n$ symmetric matrix. For a valid kernel, $\mathbf{K}$ must be PSD: $\mathbf{v}^\mathsf{T} \mathbf{K} \mathbf{v} \ge 0\ \forall \mathbf{v} \in \R^n$.
▶ Read the Definition in English
“The kernel matrix $\mathbf{K}$ stores the similarity between every pair of training points. Entry $\mathbf{K}_{ij}$ is the kernel value for $(\mathbf{x}_i, \mathbf{x}_j)$. For Mercer kernels, $\mathbf{K}$ is positive semi-definite — no negative eigenvalues.”
▶ Properties of $\mathbf{K}$
PropertyMeaningImplication
Symmetric $\mathbf{K}_{ij} = \mathbf{K}_{ji}$ Follows from kernel symmetry
PSD All eigenvalues $\ge 0$ Convex optimization guaranteed
$n \times n$ Size depends on samples, not features $O(n^2)$ memory, $O(n^3)$ for inversion
▶ Kernel Matrix Example

i
Computational Note

Kernel methods scale quadratically in $n$ (computing $\mathbf{K}$ is $O(n^2 d)$). For large datasets ($n > 10^5$), approximations like the Nyström method or random Fourier features are used.

!
Common Mistakes

✖ Confusing kernel matrix with covariance matrix: The kernel matrix is $n \times n$ (sample similarities), while the covariance matrix is $d \times d$ (feature similarities).

✖ Assuming all kernel matrices are invertible: PSD guarantees eigenvalues $\ge 0$, but some may be $0$ — the matrix can be singular.
