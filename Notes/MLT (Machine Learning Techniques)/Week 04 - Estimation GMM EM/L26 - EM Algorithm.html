The Expectation-Maximization Algorithm

Last time we found that two easy sub-problems — reweighting responsibilities, and re-estimating parameters — could stand in for one hard one. Today we wire them into a loop, give it a stopping rule, and confront its one real weakness: where you start determines where you end up.

↩
Recap from Lecture 25 We invented responsibilities λik and used Jensen's Inequality to build a lower bound on the log-likelihood. The M-step gave closed-form weighted mean/variance/π; the E-step gave λ via Bayes' Theorem. Today those two steps become one named algorithm.
01

Naming the Loop: EM

The Expectation-Maximization algorithm is nothing more than E-step, M-step, repeat — run until the parameters stop moving in any meaningful way. What makes it worth naming is the guarantee underneath it: because each M-step maximizes the exact bound the E-step just built, the true log-likelihood never decreases from one iteration to the next.

Pseudocode — EM for a Gaussian Mixture Model
initialize θ₀ = (μ, σ², π)          // e.g. from K-Means — see §3
t ← 0

repeat:
    // E-step: responsibilities given current θ
    for each point i, cluster k:
        λ_ik ← P(z_i = k | x_i, θ_t)      // Bayes' rule

    // M-step: parameters given fresh λ
    for each cluster k:
        μ_k  ← weighted mean(λ·k, x)
        σ_k² ← weighted variance(λ·k, x, μ_k)
        π_k  ← weighted share(λ·k)

    θ_(t+1) ← (μ, σ², π)
    t ← t + 1

until |log L(θ_t) − log L(θ_t₋₁)| < ε

return θ_t
💡
The convergence guarantee, in one line Every E-step rebuilds a lower bound that touches the true log-likelihood exactly at the current θ. Every M-step climbs that bound to its peak. Climbing a platform that touches the real mountain and never pokes above it can only take you to equal or higher ground on the real mountain too — hence log L never drops.

What "convergence" actually checks

In practice you don't wait for parameters to stop moving by exactly zero — floating point noise never quite settles. Instead you fix a small tolerance ε (say 10⁻⁶) and stop once successive log-likelihood values differ by less than ε, or after a maximum iteration count as a safety net.

02

EM vs. K-Means: Same Job, Different Commitment

K-Means (Lloyd's Algorithm) looks like a special case of this loop — assign points, recompute centers, repeat — but it makes a much stronger, much earlier commitment than EM does.

K-Means — hard assignment
A single boundary (Voronoi edge) slices the space. The point right on the line is forced fully into one bucket — no middle ground.
EM — soft assignment
No hard line — density blends smoothly. The point in the middle is genuinely ambiguous, and its color (a blended λ) says so honestly.

K-Means is EM's limiting case Shrink every Gaussian's variance toward zero and force λik to round to 0 or 1 (whichever cluster is closest), and the EM updates collapse exactly into K-Means's assign-then-average loop. K-Means isn't a different algorithm so much as EM with all its statistical nuance switched off.
03

The Catch: Only a Local Maximum

The convergence guarantee from §1 promises the log-likelihood never decreases. It does not promise EM finds the best possible parameters. Like gradient ascent on any bumpy surface, EM can walk uphill straight into a foothill and stop there, satisfied, while a taller peak sits untouched elsewhere in the landscape.

⚠️
Why this happens Each M-step only ever climbs the current Jensen bound — it has no way to see over the ridge to a better bound elsewhere. Once θ settles into a point where the bound's peak coincides with a local maximum of the true log-likelihood, EM has no mechanism to escape. Where you initialize μ, σ², and π decides which foothill you're standing on.

04

A Practical Fix: Let K-Means Go First

Since EM's outcome depends so heavily on its starting point, a common and effective strategy is to not start blind. Run the cheap, fast K-Means algorithm first, then hand its results to EM as a warm start.

1 · Run K-Means

Fast, hard clustering gives rough cluster centers with almost no computation.

2 · Convert to θ₀

Use each K-Means cluster's points to compute an initial mean, variance, and mixing weight per cluster.

3 · Hand off to EM

EM refines these into soft, statistically grounded estimates — starting from a sensible neighborhood, not a coin flip.

💡
Why this helps K-Means is cheap enough to re-run from several random starts and keep the best one by its own objective, giving EM a head start that's already roughly in the right neighborhood — turning "which foothill did we land on" from a coin flip into a much better bet.
05

Beyond GMMs

Nothing about the E/M split is specific to Gaussians. Any model with a hidden variable that, once revealed, makes the likelihood factor nicely — Hidden Markov Models, mixture-of-experts, missing-data regression, topic models — can be fit with the same two-step recipe: guess the hidden structure's distribution, then maximize as if that guess were true, and repeat.

06

Where This Leaves Us

TL;DR

  • EM = E-step + M-step, looped until log-likelihood stops improving by more than ε.
  • Its guarantee is monotonic improvement, not global optimality — it converges to a local maximum, not necessarily the best one.
  • Versus K-Means: EM trades a hard, geometric assignment for a soft, probabilistic one that also models variance and shape.
  • Because initialization decides which local maximum you land on, a common fix is to seed EM with K-Means output.
  • The E/M recipe generalizes far past GMMs — anywhere a latent variable would make the likelihood easy if only you knew it.
