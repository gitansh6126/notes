Gaussian Mixture Models:
one bell curve isn't enough

Real data rarely clusters around a single center. GMMs model it as coming from several Gaussians at once — each observation secretly belonging to one of them, a hidden label we never get to see.

§1 Why one Gaussian fails §2 Generative story §3 Latent variables §4 Parameters to estimate
01

Why a single Gaussian fails

When data has multiple modes — several dense clusters rather than one blob — a single Gaussian struggles. Even the Maximum Likelihood mean (the sample mean) lands in the empty space between clusters, assigning low density to the points that actually matter.

// single gaussian vs. multimodal data

The dashed curve is the best single Gaussian fit — yet it places almost no density directly over either real cluster.

!
The core problem

A unimodal model has exactly one "center of mass." When the true data has several centers, that single center satisfies none of them well — the model is structurally too simple.

02

The generative story

A GMM explains each data point with a two-step story. Think of it like a vending machine with K compartments: first a die decides which compartment, then that compartment's own dispenser produces the item.

1

Pick a component

Roll a K-sided die. Face l comes up with probability π_l. The result is the latent indicator z_i for point i.

Σ πₗ = 1,   0 ≤ πₗ ≤ 1
2

Sample from that Gaussian

Given z_i, draw x_i from the Gaussian that belongs to that component — its own mean and variance.

x_i ~ N(μ_{z_i}, σ²_{z_i})

Notice

You never see the die roll — only the final point on the number line. Recovering which component z_i a point likely came from is exactly the inference problem GMMs are built to solve.

03

Latent variables

A GMM is a latent variable model: the data we observe, x₁, …, xₙ, depends on variables we never observe, z₁, …, zₙ — the hidden component label behind each point.

Latent · unobserved
z_i ∈ {1 … K}
determines
Observed
x_i ∈ ℝ

We only ever see x_i. The component label z_i is the piece of the generative story that's hidden from us — inferring it (or averaging over it) is the central challenge of fitting a GMM.

✓
Structured clustering

Unlike K-Means' hard cluster assignments, the latent-variable view gives clustering a full probabilistic backbone: each point has a distribution over which component it likely came from.

04

Parameters to estimate

Fully specifying a GMM means estimating three families of parameters from data — for a K-component mixture, roughly 3K numbers in total.

ParameterSymbolCountRole
Mixing coefficients π = {π₁ … π_K} K  (K−1 free) How likely each component is to be picked
Means μ_k K Center of each component's Gaussian
Variances σ²_k K Spread of each component's Gaussian
Total — 3K  (3K−1 free) The π's sum-to-one constraint removes one degree of freedom
gmm_generative_story.py — sampling one point, made explicit
# pi:    length-K array, sums to 1
# mu:    length-K array of component means
# sigma2: length-K array of component variances

def sample_one_point(pi, mu, sigma2):
    # step 1 — roll the K-sided die
    z_i = random_choice(K, p=pi)

    # step 2 — sample from that component's Gaussian
    x_i = gaussian_sample(mean=mu[z_i], var=sigma2[z_i])

    return x_i  # z_i itself is never returned — it's latent
i
Still Maximum Likelihood — just harder

Estimating 3K parameters at once is far more involved than fitting one Gaussian, since the latent z_i are never observed directly. But the guiding principle is unchanged: find the parameters that make the observed data most probable.
