Gaussian Mixture Models:
one bell curve isn't enough

Real data rarely clusters around a single center. GMMs model it as coming from several Gaussians at once — each observation secretly belonging to one of them, a hidden label we never get to see.

§1 Why One Gaussian Fails §2 Generative Story §3 Marginal Density §4 Latent Variables §5 Parameters
01

Why a single Gaussian fails

When data has multiple modes — several dense clusters rather than one blob — a single Gaussian struggles. Even the Maximum Likelihood mean (the sample mean) lands in the empty space between clusters, assigning low density to the points that actually matter.

▶ Single Gaussian vs. Multimodal Data
The dashed curve is the best single Gaussian fit — yet it places almost no density directly over either real cluster.
!
The core problem

A unimodal model has exactly one "center of mass." When the true data has several centers, that single center satisfies none of them well — the model is structurally too simple.

02

The generative story

A GMM explains each data point with a two-step story. Think of it like a vending machine with $K$ compartments: first a die decides which compartment, then that compartment's own dispenser produces the item.

$z_i \in \{1, \dots, K\}$

Latent Component

Which component generated point $i$. Hidden from us.

$\mathbf{x}_i \in \R^d$

Observed Data

The actual point we see, sampled from the chosen component.

$\boldsymbol{\pi} = \{\pi_1, \dots, \pi_K\}$

Mixing Weights

Probability of each component: $\sum \pi_k = 1$, $\pi_k \ge 0$.

Equation (1)
Generative Process
$$z_i \sim \text{Categorical}(\pi_1, \dots, \pi_K) \qquad \mathbf{x}_i \mid z_i = k \sim \N(\boldsymbol{\mu}_k, \sigma_k^2)$$
First pick a component $k$ with probability $\pi_k$, then sample from that component's Gaussian
▶ Read the Equation in English
“Each data point $\mathbf{x}_i$ is generated by (1) rolling a $K$-sided die to pick a component $z_i$, where face $k$ has probability $\pi_k$, and then (2) sampling $\mathbf{x}_i$ from a Gaussian distribution with mean $\boldsymbol{\mu}_k$ and variance $\sigma_k^2$. The die roll $z_i$ is never observed.”
▶ Symbol Legend
SymbolMeaningExample
$z_i$ Latent variable — which component generated point $i$ $z_i = 2$ means component 2 generated this point
$\pi_k$ Mixing probability of component $k$ $\pi_1 = 0.5$, $\pi_2 = 0.3$, $\pi_3 = 0.2$
$\boldsymbol{\mu}_k$ Mean of component $k$'s Gaussian $\boldsymbol{\mu}_1 = 170$ cm
$\sigma_k^2$ Variance of component $k$'s Gaussian $\sigma_1^2 = 25$
▶ Step-by-Step Generative Process
1
Pick a component $k$ with probability $\pi_k$
$z_i \sim \text{Cat}(\boldsymbol{\pi})$
↓
2
Given $z_i = k$, sample $\mathbf{x}_i$ from that Gaussian
$\mathbf{x}_i \sim \N(\boldsymbol{\mu}_k, \sigma_k^2)$
↓
3
The observed data point is $\mathbf{x}_i$ — $z_i$ is hidden
$\text{Observed: } \mathbf{x}_i \quad \text{Latent: } z_i$
▶ Generative Flow
The component label $z_i$ is the piece of the story that's hidden from us — we only ever see $\mathbf{x}_i$.
03

The marginal density

Since we never observe $z_i$, we must marginalize (sum) over all possible component assignments. The result is a weighted sum of Gaussians — the defining equation of GMMs.

Equation (2)
Marginal Density of a GMM
$$p(\mathbf{x}_i \mid \boldsymbol{\theta}) = \sum_{k=1}^{K} \pi_k \; \N(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \sigma_k^2)$$
The probability of one point is the weighted sum across every Gaussian component
▶ Read the Equation in English
“The probability of observing $\mathbf{x}_i$ under the GMM is the sum over all $K$ components of the mixing weight $\pi_k$ times the Gaussian density of $\mathbf{x}_i$ under component $k$. Each component contributes proportionally to its probability $\pi_k$.”
▶ Intuition
Imagine you hear a sound from behind a wall. There are three speakers ($K = 3$). Each speaker plays a different note, but you can't see which speaker is active. The sound you hear is a weighted blend of all three — some speakers are louder (higher $\pi$) than others.

The marginal density does exactly that: instead of picking one Gaussian, it considers every possible component, weights each by its probability $\pi_k$, and adds them together.
▶ Deriving the Marginal Density
1
Start with the joint: $p(\mathbf{x}_i, z_i = k \mid \boldsymbol{\theta})$
$= \pi_k \cdot \N(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \sigma_k^2)$
↓ Sum over all possible $k$ values (marginalize)
2
Apply the sum rule of probability
$\displaystyle p(\mathbf{x}_i \mid \boldsymbol{\theta}) = \sum_{k=1}^K p(\mathbf{x}_i, z_i = k \mid \boldsymbol{\theta})$
↓ Substitute the joint
3
The GMM marginal density
$\displaystyle p(\mathbf{x}_i \mid \boldsymbol{\theta}) = \sum_{k=1}^K \pi_k \N(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \sigma_k^2)$
▶ Numerical Example ($K = 2$, 1-dimensional)
Setup: GMM with $\pi = [0.6, 0.4]$, $\boldsymbol{\mu} = [170, 165]$, $\boldsymbol{\sigma}^2 = [25, 36]$. Compute $p(\mathbf{x} = 172)$.
$k$$\pi_k$$\mu_k$$\sigma_k^2$$\N(172 \mid \mu_k, \sigma_k^2)$$\pi_k \cdot \N_k$
1 $0.6$ $170$ $25$ $0.0737$ $0.0442$
2 $0.4$ $165$ $36$ $0.0337$ $0.0135$
$p(172 \mid \boldsymbol{\theta}) = 0.0442 + 0.0135 = \mathbf{0.0577}$
04

Latent variables

A GMM is a latent variable model: the data we observe, $\mathbf{x}_1, \dots, \mathbf{x}_N$, depends on variables we never observe, $z_1, \dots, z_N$ — the hidden component label behind each point.

▶ Latent Variable Structure
The latent $z_i$ mediates between the parameters $\boldsymbol{\theta}$ and the observed data $\mathbf{x}_i$. We never see $z_i$, but we must reason about it.
i
Structured clustering

Unlike $K$-Means' hard cluster assignments, the latent-variable view gives clustering a full probabilistic backbone: each point has a distribution over which component it likely came from.

05

Parameters to estimate

Fully specifying a GMM means estimating three families of parameters from data — for a $K$-component mixture, roughly $3K$ numbers in total.

$\boldsymbol{\pi}$

Mixing Coefficients

$K$ numbers ($K-1$ free). $\sum \pi_k = 1$, $\pi_k \ge 0$.

$\boldsymbol{\mu}_k$

Means

$K$ vectors. Center of each component's Gaussian.

$\sigma_k^2$

Variances

$K$ scalars. Spread of each component's Gaussian.

Equation (3)
Complete Set of GMM Parameters
$$\boldsymbol{\theta} = \{ \pi_1, \dots, \pi_K, \boldsymbol{\mu}_1, \dots, \boldsymbol{\mu}_K, \sigma_1^2, \dots, \sigma_K^2 \}$$
$3K$ total parameters (one constraint: $\sum \pi_k = 1$ reduces degrees of freedom by 1)
gmm_generative_story.py — sampling one point
# pi:    length-K array, sums to 1
# mu:    length-K array of component means
# sigma2: length-K array of component variances

def sample_one_point(pi, mu, sigma2):
    # step 1 — roll the K-sided die
    z_i = random_choice(K, p=pi)

    # step 2 — sample from that component's Gaussian
    x_i = gaussian_sample(mean=mu[z_i], var=sigma2[z_i])

    return x_i  # z_i itself is never returned — it's latent
i
Still Maximum Likelihood — just harder

Estimating $3K$ parameters at once is far more involved than fitting one Gaussian, since the latent $z_i$ are never observed directly. But the guiding principle is unchanged: find the parameters that make the observed data most probable.
