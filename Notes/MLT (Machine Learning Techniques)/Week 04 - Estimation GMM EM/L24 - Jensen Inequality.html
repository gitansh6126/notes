Convexity, concavity &
Jensen's inequality

Last time, the log-likelihood of a GMM got stuck behind a sum trapped inside a logarithm. The fix starts with a simple geometric fact about curves and the straight lines drawn between their points.

§1 Convex Functions §2 Concave Functions §3 Jensen's Inequality §4 Special Cases §5 The GMM Bottleneck
01

Convex functions

Pick any two points $a$ and $b$ on a curve, and draw the straight chord between them. Where that chord sits, relative to the curve, is the entire definition.

Equation (1)
Definition of a Convex Function
$$f(\lambda a + (1 - \lambda) b) \leq \lambda f(a) + (1 - \lambda) f(b) \quad \forall \lambda \in [0, 1]$$
The chord lies on or above the curve
▶ Read the Equation in English
“For a convex function $f$, the function value at any weighted average of $a$ and $b$ is less than or equal to the weighted average of the function values. The function at the interpolation point is never above the straight line connecting $f(a)$ and $f(b)$.”
▶ Symbol Legend
SymbolMeaningExample
$a, b$ Any two points in the domain of $f$ $a = 0$, $b = 4$
$\lambda$ Interpolation weight between 0 and 1 $\lambda = 0.5$ gives midpoint
$f(\lambda a + (1-\lambda) b)$ Function value at the interpolated point On the curve
$\lambda f(a) + (1-\lambda) f(b)$ Interpolated function value (on the chord) On the straight line
▶ Intuition
A convex function is shaped like a bowl ($f(x) = x^2$). If you pick two points on the bowl and stretch a rubber band between them, the band (the chord) lies above the bowl's surface everywhere. The function "sags" below its chords.
▶ Convex Function: $f(x) = x^2$
For a convex function, the chord lies above the curve. The gold dot (on chord) is above the teal dot (on curve).
02

Concave functions

A concave function is the mirror image: the chord lies on or below the curve. Like a dome.

Equation (2)
Definition of a Concave Function
$$f(\lambda a + (1 - \lambda) b) \geq \lambda f(a) + (1 - \lambda) f(b) \quad \forall \lambda \in [0, 1]$$
The chord lies on or below the curve
▶ Read the Equation in English
“For a concave function $f$, the function value at any weighted average of $a$ and $b$ is greater than or equal to the weighted average of the function values. The function at the interpolation point is never below the straight line connecting $f(a)$ and $f(b)$.”
▶ Intuition
A concave function is shaped like a dome ($f(x) = \log x$). The chord lies below the curve.

Critically for GMMs: the logarithm function $\log(x)$ is concave! This means $ \log(\lambda a + (1-\lambda) b) \geq \lambda \log a + (1-\lambda) \log b$ — the weighted average of logs is less than or equal to the log of the weighted average.
▶ Concave Function: $f(x) = \log(x)$
For a concave function like $\log(x)$, the chord lies below the curve. The curve (gold) is above the chord (teal).
Equation (3)
Why Log Is Concave
$$\frac{d^2}{dx^2} \log x = -\frac{1}{x^2} < 0 \quad\Longrightarrow\quad \text{$\log$ is concave}$$
A function is concave if its second derivative is everywhere non-positive
03

Jensen's inequality

Convexity and concavity were defined for two points. Jensen's inequality is the same idea stretched to any number of points $a_1, \dots, a_K$, weighted by $\lambda_1, \dots, \lambda_K$ that sum to 1.

Equation (4)
Jensen's Inequality — General Form
$$\text{If $f$ is concave:} \qquad f\!\left( \sum_{i=1}^{K} \lambda_i a_i \right) \geq \sum_{i=1}^{K} \lambda_i f(a_i)$$ $$\text{If $f$ is convex:} \qquad f\!\left( \sum_{i=1}^{K} \lambda_i a_i \right) \leq \sum_{i=1}^{K} \lambda_i f(a_i)$$ $$\text{where}\quad \sum_{i=1}^{K} \lambda_i = 1, \quad \lambda_i \geq 0$$
The same chord-vs-curve idea, now for $K$ points at once with weights summing to 1
▶ Read the Equation in English
“For a concave function, the function value at the weighted average of the inputs is at least the weighted average of the function values at each input. For a convex function, the opposite holds.”
▶ Symbol Legend
SymbolMeaningExample
$a_i$ Input points $a_1 = 2, a_2 = 5, a_3 = 8$
$\lambda_i$ Weights that sum to 1 (can be interpreted as probabilities) $\lambda_1 = 0.5, \lambda_2 = 0.3, \lambda_3 = 0.2$
$\sum \lambda_i a_i$ Weighted average — lies in convex hull of $\{a_i\}$ $0.5 \cdot 2 + 0.3 \cdot 5 + 0.2 \cdot 8 = 4.1$
▶ Intuition
$\sum_i \lambda_i a_i$ is just a weighted average — geometrically, it's guaranteed to land inside the convex hull of $a_1, \dots, a_K$. Jensen's inequality compares the function's value there to the weighted average of the function's values at each point individually.

For a concave function like $\log$, the function of the average is larger than the average of the function values.
▶ Two-Point vs. $K$-Point Forms
Convex (two-point) $f(\lambda a + (1-\lambda)b) \leq \lambda f(a) + (1-\lambda)f(b)$
Convex (Jensen) $f(\sum \lambda_i a_i) \leq \sum \lambda_i f(a_i)$
Concave (two-point) $f(\lambda a + (1-\lambda)b) \geq \lambda f(a) + (1-\lambda)f(b)$
Concave (Jensen) $f(\sum \lambda_i a_i) \geq \sum \lambda_i f(a_i)$
04

Special cases

Two edge cases round out the picture: what happens when the inequality collapses to equality, and what happens when curvature isn't consistent at all.

$\text{Linear}$

Equality holds

If $f$ is linear ($f(x) = mx + c$), the chord and curve coincide exactly: $f(\lambda a + (1-\lambda)b) = \lambda f(a) + (1-\lambda) f(b)$ for all $\lambda, a, b$.

$\text{Affine}$

Both convex and concave

An affine function satisfies the definitions of both convexity and concavity simultaneously. It is the only class of functions that does so.

$\text{Oscillating}$

Neither — curvature flips

An oscillating curve (like $\sin x$) crosses its own chords, so no global convexity or concavity holds — Jensen's inequality cannot be applied globally.

Equation (5)
Linear Function — Equality Case
$$\text{If } f(x) = mx + c, \text{ then } f(\lambda a + (1-\lambda)b) = \lambda f(a) + (1-\lambda) f(b)$$
Linear functions are both convex and concave — Jensen's inequality becomes equality
i
Why equality matters

When we apply Jensen's inequality to bound the GMM log-likelihood, we get an inequality — not equality. The gap between the two sides is exactly what the EM algorithm tries to close iteratively.

05

Back to the GMM bottleneck

The log function is concave. That single fact is the key that unlocks the log-sum problem from the previous lecture.

Equation (6)
Applying Jensen to the GMM Log-Likelihood
$$\log\!\left( \sum_{k=1}^{K} \pi_k \, \N(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \sigma_k^2) \right) \geq \sum_{k=1}^{K} \pi_k \, \log\!\left( \N(\mathbf{x}_i \mid \boldsymbol{\mu}_k, \sigma_k^2) \right)$$
Because $\log$ is concave and $\pi_k$ are valid weights ($\sum \pi_k = 1$, $\pi_k \ge 0$)
▶ Read the Equation in English
“The log of the weighted sum of Gaussian densities is at least the weighted sum of the logs of each Gaussian density. This swaps the order: the sum moves outside the log, where we can differentiate it term-by-term — at the cost of only having a lower bound.”
▶ From Intractable to Tractable
1
Start with the intractable GMM log-likelihood
$\displaystyle \ell(\boldsymbol{\theta}) = \sum_i \log\!\bigl( \sum_k \pi_k \N_k \bigr)$
↓ Apply Jensen — $\log$ is concave, $\pi_k$ is a valid weight
2
A tractable lower bound emerges
$\displaystyle \ell(\boldsymbol{\theta}) \geq \sum_i \sum_k \pi_k \log \N_k$
↓ Now each component's parameters are separable
3
Maximize the bound instead of the true log-likelihood
$\displaystyle \text{EM algorithm}$

✔
Why a lower bound is enough

We don't get equality — but we get a tractable lower bound on the log-likelihood that we can differentiate cleanly. Maximizing that bound, instead of the true log-likelihood directly, is the core idea behind the Expectation-Maximization (EM) algorithm.

!
Common Mistakes

✖ Forgetting the direction: For concave $f$, Jensen says $f(\text{average}) \ge \text{average of } f$. Since $\log$ is concave, $\log(\sum \pi_k \N_k) \ge \sum \pi_k \log \N_k$. The bound is a lower bound.

✖ Assuming the bound is tight: Jensen gives an inequality, not equality. The gap between the two sides depends on how "spread out" the $\N_k$ values are. The EM algorithm works by iteratively tightening this bound.
