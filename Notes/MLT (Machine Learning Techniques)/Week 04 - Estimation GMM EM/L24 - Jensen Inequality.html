01

Convex vs. concave

Pick any two points a and b on a curve, and draw the straight chord between them. Where that chord sits, relative to the curve, is the entire definition.

Convex

The chord lies on or above the curve. Like a bowl: interpolating between two points never underestimates how low the function actually dips.

Concave

The chord lies on or below the curve. Like a dome: interpolating between two points never overestimates the peak.

convex
f(λa + (1−λ)b) ≤ λf(a) + (1−λ)f(b)
concave
f(λa + (1−λ)b) ≥ λf(a) + (1−λ)f(b)
for any λ ∈ [0, 1]

02

Special cases

Two edge cases round out the picture: what happens when the inequality collapses to equality, and what happens when curvature isn't consistent at all.

Linear — both at once
Chord and curve coincide: f(λa+(1−λ)b) = λf(a)+(1−λ)f(b)
Neither — curvature flips
An oscillating curve crosses its own chords, so no global inequality holds
03

Jensen's inequality

Convexity and concavity were defined for two points. Jensen's inequality is the same idea stretched to any number of points a₁, …, a_K, weighted by λ₁, …, λ_K that sum to 1.

jensen's inequality — concave f
f( Σᵢ λᵢ aᵢ ) ≥ Σᵢ λᵢ f(aᵢ)
Σ λᵢ = 1, λᵢ ≥ 0 — the same "chord below the curve" idea, for K points at once

Convex hull, in one line

Σᵢ λᵢ aᵢ is just a weighted average — geometrically, it's guaranteed to land inside the convex hull of a₁, …, a_K. Jensen's inequality compares the function's value there to the weighted average of the function's values at each point individually.

04

Back to the GMM bottleneck

The log function is concave. That single fact is the key that unlocks the log-sum problem from last lecture.

why_log_is_concave_matters.txt
// last lecture, we got stuck here:
log( Σₖ πₖ · 𝒩(x_i | μₖ, σₖ²) )

// log is concave, and πₖ are weights that sum to 1 —
// exactly the setup Jensen's inequality needs:
log( Σₖ πₖ · 𝒩ₖ ) ≥ Σₖ πₖ · log( 𝒩ₖ )

// the sum has moved OUTSIDE the log
✓
Why a lower bound is enough

We don't get equality — but we get a tractable lower bound on the log-likelihood that we can differentiate cleanly. Maximizing that bound, instead of the true log-likelihood directly, is the core idea behind the algorithm we cover next.
