Maximum Likelihood
Estimation

Given the evidence in front of you, which explanation makes it least surprising? That question, formalized by R.A. Fisher, is the whole idea.

Fisher, R.A. Parameter Estimation Core Method
Overview

The central idea

Maximum Likelihood Estimation (MLE) is a method for deriving estimators for the parameters of a probability distribution, based only on data you've already observed. Instead of guessing a parameter and checking if it "feels right," MLE asks a sharper question:

Out of every possible parameter value, which one makes the data I actually saw the most probable?
🔍
Detective analogy A detective doesn't ask "which suspect is nice?" — they ask "which suspect's story best explains the evidence on the table?" MLE runs the same logic on numbers: the data is the evidence, the parameter is the suspect, and the likelihood function scores how well each suspect explains what happened.
1

Key Concepts

Concept A

Likelihood Function, L

A function of the parameters, with the data held fixed. It tells you the probability (or probability density, for continuous data) of the data you observed, evaluated under a given parameter guess.

L(θ; x1,…,xn) = P(data | θ)
Concept B

The MLE, θ̂

The single parameter value that maximizes the likelihood function — the "best suspect" given the evidence.

θ̂ML = arg maxθ L(θ; x1,…,xn)
Concept C

Log-Likelihood

Logarithms are monotonically increasing, so the θ that maximizes log L(θ) is exactly the θ that maximizes L(θ). This matters because log turns a product of terms into a sum — far easier to differentiate.

log L(θ) = Σ log P(xi | θ)
// product → sum, same optimal θ
Concept D

The Probabilistic Model

Before you can compute anything, you must choose the "box" — the assumed distribution the data came from (Bernoulli, Gaussian, Laplacian…). This choice isn't cosmetic: different boxes produce genuinely different estimators from identical data.

Same data, different model → different answer. The model is an assumption, not a fact handed to you.

MLE lands exactly where intuition would: the sample mean — here, 5⁄8.

p̂ML = (1/n) Σ xi // fraction of 1s

Real-life readYou don't need calculus to sense this is right — if 5 of 8 flips came up heads, "p ≈ 0.625" is the least surprising story. MLE just proves it formally.
Exhibit C — Continuous Data

Heights (Gaussian Model)

Data is real-valued. A single exact height has probability zero, so likelihood is built from the probability density function (PDF) instead of raw probability.

Green dots = observed heights. The curve's mean μ̂ that best explains their spread is the MLE.

Assuming known variance, the MLE for the unknown mean μ is — again — the sample mean.

μ̂ML = (1/n) Σ xi
Swap the "box": assume a Laplacian distribution instead of Gaussian, and the MLE shifts to the sample median. Same data, different model, different estimator.

Methodological Summary

The Procedure

Define the model

Choose the "box" — the probability distribution you believe generated the data (Bernoulli, Gaussian, Laplacian, etc.).

Build the likelihood

Multiply the individual probabilities or densities across all data points, assuming independence.

L(θ) = ∏i=1n P(xi | θ)

Take the log

Apply the natural logarithm to convert the product into a sum — same optimal θ, much friendlier algebra.

Differentiate and solve

Take the derivative with respect to θ, set it to zero, and solve. That solution is θ̂ML.

d/dθ [log L(θ)] = 0 → solve for θ
Closing the Case

Why MLE holds up

MLE is intuitive in the simple cases — coin flips give you the sample mean, and that matches gut instinct. But its real strength shows up when instinct runs out: for any distribution where you can write down a probability density or mass function, MLE hands you a systematic, mathematically derived estimator. No guesswork, no case-by-case reinvention — just evidence, a model, and calculus doing the rest.

⚖️
The verdictChoose your model carefully — it's the lens the entire estimate is seen through — then let the likelihood function point to the parameter that makes your data the least surprising outcome possible.
