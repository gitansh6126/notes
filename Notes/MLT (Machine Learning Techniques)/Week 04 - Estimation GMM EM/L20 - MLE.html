Maximum Likelihood Estimation

Given the evidence in front of you, which explanation makes it least surprising? That question, formalized by R.A. Fisher, is the whole idea.

§1 Core Idea §2 Likelihood Function §3 Bernoulli MLE §4 Gaussian MLE §5 Methodology
01

The central idea

Maximum Likelihood Estimation (MLE) is a method for deriving estimators for the parameters of a probability distribution, based only on data you've already observed. Instead of guessing a parameter and checking if it "feels right," MLE asks a sharper question.

?
The core question

Out of every possible parameter value, which one makes the data I actually saw the most probable?

Equation (1)
Maximum Likelihood Estimator
$$\hat{\boldsymbol{\theta}}_{\text{ML}} = \arg\max_{\boldsymbol{\theta}} \mathcal{L}(\boldsymbol{\theta}; \mathbf{x}_1, \dots, \mathbf{x}_N)$$
The MLE is the parameter value that maximizes the likelihood function
▶ Read the Equation in English
“The maximum likelihood estimate $\hat{\boldsymbol{\theta}}_{\text{ML}}$ is the value of $\boldsymbol{\theta}$ that makes the likelihood function $\mathcal{L}$ as large as possible — it is the parameter value under which the observed data is the least surprising.”
▶ Symbol Legend
SymbolMeaningExample
$\mathcal{L}(\boldsymbol{\theta})$ Likelihood function — probability of data given parameters $\mathcal{L}(p) = p^{n_h}(1-p)^{n_t}$
$\hat{\boldsymbol{\theta}}_{\text{ML}}$ The MLE — the parameter that maximizes $\mathcal{L}$ $\hat{p} = n_h / N$
$\arg\max$ "The argument that maximizes" — the input giving the highest output $\arg\max_x (-x^2) = 0$
▶ Intuition
A detective doesn't ask "which suspect is nice?" — they ask "which suspect's story best explains the evidence on the table?" MLE runs the same logic on numbers: the data is the evidence, the parameter is the suspect, and the likelihood function scores how well each suspect explains what happened.

The MLE is simply the "best suspect" — the one whose story makes the observed evidence least surprising.
▶ Likelihood Landscape
The likelihood curve — slide $\boldsymbol{\theta}$ along the horizontal axis. At every point, the curve's height is how well that $\boldsymbol{\theta}$ explains the data. MLE walks to the peak.
02

The likelihood function

The likelihood function is the workhorse of MLE. It is a function of the parameters, with the data held fixed. For i.i.d. data, it factorizes into a product of per-point probabilities.

Equation (2)
Likelihood Function (IID)
$$\mathcal{L}(\boldsymbol{\theta}) = \prod_{i=1}^{N} P(\mathbf{x}_i \mid \boldsymbol{\theta})$$
The joint probability of all data, viewed as a function of $\boldsymbol{\theta}$
▶ Read the Equation in English
“The likelihood of $\boldsymbol{\theta}$ given the data is the product over all $N$ data points of each point's probability (or probability density) evaluated under that $\boldsymbol{\theta}$.”
▶ Product → Sum (Log-Likelihood)
1
Start with the likelihood — a product over $N$ points
$\displaystyle \mathcal{L}(\boldsymbol{\theta}) = \prod_{i=1}^N P(\mathbf{x}_i \mid \boldsymbol{\theta})$
↓ Take $\log$ — monotonic, preserves argmax
2
Apply $\log(\prod a_i) = \sum \log a_i$
$\displaystyle \ell(\boldsymbol{\theta}) = \sum_{i=1}^N \log P(\mathbf{x}_i \mid \boldsymbol{\theta})$
↓ Now differentiate and set to zero
3
Solve $\displaystyle \frac{\partial \ell(\boldsymbol{\theta})}{\partial \boldsymbol{\theta}} = 0$ for $\hat{\boldsymbol{\theta}}_{\text{ML}}$
$\displaystyle \hat{\boldsymbol{\theta}}_{\text{ML}}$
✔
Why log?

Logarithms are monotonically increasing, so the $\boldsymbol{\theta}$ that maximizes $\log \mathcal{L}(\boldsymbol{\theta})$ is exactly the $\boldsymbol{\theta}$ that maximizes $\mathcal{L}(\boldsymbol{\theta})$. The log turns a product of terms into a sum — far easier to differentiate.

03

Bernoulli (Coin Flip) MLE

The simplest setting: binary data $\{0, 1\}$ from a Bernoulli distribution with unknown parameter $p$. The MLE turns out to be exactly the sample mean.

Equation (3)
Bernoulli Likelihood
$$\mathcal{L}(p) = \prod_{i=1}^{N} p^{x_i} (1-p)^{1-x_i} = p^{n_h} (1-p)^{N - n_h}$$
Where $n_h = \sum x_i$ is the number of heads (1s)
▶ Read the Equation in English
“The likelihood of $p$ is $p$ raised to the number of heads times $(1-p)$ raised to the number of tails. Each flip contributes a factor: $p$ if heads, $1-p$ if tails.”
▶ Deriving the Bernoulli MLE
1
Write the log-likelihood
$\ell(p) = n_h \log p + (N - n_h) \log(1-p)$
↓ Differentiate with respect to $p$
2
Set derivative to zero
$\displaystyle \frac{d\ell}{dp} = \frac{n_h}{p} - \frac{N - n_h}{1-p} = 0$
↓ Solve for $p$
3
The MLE is the sample mean
$\displaystyle \hat{p}_{\text{ML}} = \frac{n_h}{N} = \frac{1}{N} \sum_{i=1}^N x_i$
▶ Numerical Example
Setup: 8 coin flips: H, H, T, H, T, H, H, T (5 heads, 3 tails).
StepExpressionValue
$\ell(p)$ $5 \log p + 3 \log(1-p)$ —
Derivative $\frac{5}{p} - \frac{3}{1-p} = 0$ —
Solve $5(1-p) = 3p$ $5 - 5p = 3p$
$\hat{p}_{\text{ML}}$ $5 / 8$ $0.625$
$\displaystyle \hat{p}_{\text{ML}} = \frac{5}{8} = 0.625$
▶ Intuition
You don't need calculus to sense this is right — if 5 of 8 flips came up heads, "$p \approx 0.625$" is the least surprising story. MLE just proves it formally.

The MLE lands exactly where intuition would: the sample mean. For Bernoulli data, this is always the fraction of 1s. And this is no coincidence — for many distributions, the MLE of the location parameter is the sample mean.
i
What this means

The MLE $\hat{p} = 0.625$ is the value of $p$ that maximizes the probability of seeing exactly this sequence. Any other value of $p$ — say $0.5$ or $0.8$ — would make this specific sequence less probable. The MLE is the parameter that best "fits" the data in the likelihood sense.

!
Common Mistakes

✖ Confusing likelihood with probability: $\mathcal{L}(p)$ is not the probability that $p$ is correct — it's the probability of the data given $p$. These are different quantities.

✖ Assuming MLE is always unbiased: The MLE can be biased for small samples. For example, the MLE of variance $\hat{\sigma}^2 = \frac{1}{N}\sum (x_i - \bar{x})^2$ is biased (it underestimates on average for small $N$).

04

Gaussian (Normal) MLE

For continuous data, we use the probability density function (PDF). For a Gaussian with unknown mean $\mu$ and known variance $\sigma^2$, the MLE of $\mu$ is again the sample mean.

Equation (4)
Gaussian Log-Likelihood
$$\ell(\mu) = -\frac{N}{2} \log(2\pi\sigma^2) - \frac{1}{2\sigma^2} \sum_{i=1}^{N} (x_i - \mu)^2$$
Maximizing $\ell(\mu)$ is equivalent to minimizing the sum of squared residuals
▶ Read the Equation in English
“The log-likelihood of $\mu$ under a Gaussian model is a constant (depending only on $\sigma^2$) minus the sum of squared deviations from $\mu$, scaled by $1/2\sigma^2$. Maximizing this is the same as minimizing the sum of squares — which is achieved by the sample mean.”
▶ Deriving the Gaussian MLE for $\mu$
1
Start with the log-likelihood
$\displaystyle \ell(\mu) = C - \frac{1}{2\sigma^2} \sum_{i=1}^N (x_i - \mu)^2$
↓ Differentiate with respect to $\mu$
2
Set derivative to zero
$\displaystyle \frac{d\ell}{d\mu} = \frac{1}{\sigma^2} \sum_{i=1}^N (x_i - \mu) = 0$
↓ Solve for $\mu$
3
The MLE is the sample mean
$\displaystyle \hat{\mu}_{\text{ML}} = \frac{1}{N} \sum_{i=1}^N x_i$
!
Model choice matters

Swap the "box": assume a Laplacian distribution instead of Gaussian, and the MLE shifts to the sample median. Same data, different model, different estimator. The model is an assumption, not a fact handed to you.

▶ Numerical Example

▶ Model Comparison: Same Data, Different MLE
Bernoulli Binary data $\{0, 1\}$ — MLE is sample mean (fraction of 1s)
Gaussian Continuous data — MLE of $\mu$ is sample mean
Laplacian Heavy-tailed data — MLE of location is sample median
Poisson Count data — MLE of $\lambda$ is sample mean
05

Methodological summary

MLE follows a consistent recipe regardless of the distribution. The steps are always the same — only the algebraic details change.

▶ The MLE Recipe
1
Define the model — choose the probability distribution
$P(\mathbf{x} \mid \boldsymbol{\theta})$
↓
2
Build the likelihood — multiply over all points
$\displaystyle \mathcal{L}(\boldsymbol{\theta}) = \prod_i P(\mathbf{x}_i \mid \boldsymbol{\theta})$
↓
3
Take the log — product becomes sum, same optimal $\boldsymbol{\theta}$
$\displaystyle \ell(\boldsymbol{\theta}) = \sum_i \log P(\mathbf{x}_i \mid \boldsymbol{\theta})$
↓
4
Differentiate and solve — set gradient to zero
$\displaystyle \frac{\partial \ell}{\partial \boldsymbol{\theta}} = 0 \;\longrightarrow\; \hat{\boldsymbol{\theta}}_{\text{ML}}$
Equation (5)
Score Equation
$$\frac{\partial}{\partial \boldsymbol{\theta}} \sum_{i=1}^{N} \log P(\mathbf{x}_i \mid \boldsymbol{\theta}) = 0$$
The score equation — setting the derivative of the log-likelihood to zero
→
Closing the case

MLE is intuitive in simple cases — coin flips give you the sample mean, matching gut instinct. But its real strength shows up when instinct runs out: for any distribution where you can write down a probability density or mass function, MLE hands you a systematic, mathematically derived estimator. No guesswork, no case-by-case reinvention — just evidence, a model, and calculus doing the rest.
