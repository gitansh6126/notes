0Squared Error Felt Arbitrary — Was It?

Every earlier lecture used squared error without justifying it statistically

Lecture 28 chose squared error because it's non-negative and differentiable. That's a good engineering reason, but it leaves an honest gap: is squared error the "correct" thing to minimize, or just a convenient one? This lecture answers that by asking a different question first — not "what should we minimize?" but "what process could have generated this data?" — and showing the squared-error answer falls out on its own.

RMS lens If you assume every RMS batch's actual revenue deviates from your model's prediction by pure random noise scattered symmetrically around zero (some batches over-perform, some under-perform, no systematic bias), this lecture shows that assumption alone — nothing else — justifies fitting by squared error.
Stage 3 → 5 · Mathematical Translation, Anatomy

3Assuming a Data-Generating Process

Instead of just fitting a line, assume there's a true underlying process that produced the labels:

The generative model
yi = wTxi + ϵ
Eq. 33.1
w — the true, fixed (but unknown) parameter vector we want to estimate
ϵ — random noise, assumed 𝒩(0, σ²) — Gaussian, mean zero, known variance σ²
Reading the assumption Mean-zero noise means the true relationship really is linear on average — errors are symmetric scatter, not systematic bias in one direction. This is a real, checkable modeling assumption, not a mathematical convenience.

Because ϵ is Gaussian and just shifts the mean, y itself — conditioned on x and w — is also Gaussian:

Conditional distribution of a single label
P(yi | xi; w) ∼ 𝒩(wTxi, σ2)
Eq. 33.2

🖼Likelihood, Pictured

Reading the diagram The bell curve is centered exactly at the model's prediction wᵀxᵢ. The observed label yᵢ sits somewhere under that curve — usually near the peak, sometimes in the tails. The height of the curve at yᵢ is literally P(yᵢ|xᵢ;w) — the likelihood contributed by this one data point. A model w that puts high curves under most of your observed points has high total likelihood.
Stage 6 · Derivation

6From Likelihood to Squared Error

Write the likelihood over all n IID points

L(w) = ∏ᵢ₌₁ⁿ P(yᵢ|xᵢ;w)

IID lets the joint probability of the whole dataset factor into a plain product of each point's individual probability.

Substitute the Gaussian density

L(w) = ∏ᵢ (1/√(2πσ²)) · exp(−(yᵢ − wᵀxᵢ)² / 2σ²)

Just plugging Eq. 33.2's Gaussian PDF in for each P(yᵢ|xᵢ;w).

Take the log — product becomes a sum

log L(w) = Σᵢ [ −½log(2πσ²) − (yᵢ − wᵀxᵢ)² / 2σ² ]

Log is monotonic, so maximizing log L(w) is exactly equivalent to maximizing L(w) — but sums are far easier to differentiate than products.

Drop everything that doesn't depend on w

argmax_w log L(w) = argmax_w [ −Σᵢ (yᵢ − wᵀxᵢ)² / 2σ² ]

The −½log(2πσ²) term is a constant with respect to w (σ² is assumed known) — it shifts log L(w) up or down but can't change which w maximizes it, so it's dropped.

Flip the sign: maximizing a negative = minimizing the positive

= argmin_w Σᵢ (yᵢ − wᵀxᵢ)²    = argmin_w Σᵢ (wᵀxᵢ − yᵢ)²

1/2σ² is a positive constant multiplier, so it doesn't change the argmin either. What's left is exactly the sum-of-squared-errors objective from Lecture 28 — reached with zero mention of "differentiability" as a justification.

The punchline Maximizing likelihood under Gaussian noise is mathematically identical to minimizing squared error. Every closed-form, geometric, and kernel-based technique from Lectures 29–32 was — without saying so explicitly — always solving this exact MLE problem.

15Every Loss Function Hides a Noise Assumption

The general recipe, stated by this lecture

Nothing in the derivation above was specific to the Gaussian — swap in a different noise distribution, and MLE will hand you a different loss function automatically.

Stage 9 · Edge Cases

9What This Assumes That Could Be Wrong

Noise isn't actually Gaussian

If real errors have heavier tails than a Gaussian (occasional huge outliers), squared error over-penalizes those outliers relative to what MLE under the true distribution would want — this is precisely why L1/Laplacian assumptions exist as an alternative.

σ² isn't actually known

The derivation treats σ² as a known constant so it can be dropped from the argmax. If σ² must also be estimated, it becomes a second parameter under MLE — it doesn't change the resulting w*, but it does change what else the model can tell you (e.g. prediction intervals).

Stage 18 · Examiner Trap Library

18Common Exam Traps

⚠ Trap 1 — forgetting the sign flip between argmax and argmin

Maximizing the log-likelihood requires maximizing a negative sum of squares — which is the same as minimizing the (positive) sum of squares. Writing "MLE ⟹ maximize Σ(yᵢ−wᵀxᵢ)²" instead of minimize is a sign error that flips the entire optimization direction.

⚠ Trap 2 — treating likelihood as a probability

L(w) = P(data | w) is a function of w for fixed observed data — not a probability distribution over w, and it doesn't need to integrate to 1 over w. Calling L(w) "the probability of w" is a common but incorrect phrasing.

⚠ Trap 3 — assuming squared error is the "correct" loss in general

Squared error is only the MLE-justified loss under a Gaussian noise assumption. If a question describes noise with heavy tails or asymmetry, blindly defending squared error as "the standard, correct choice" ignores that its justification is conditional on that specific assumption.

Stage 20 · Industry Implementation

20Verifying MLE = Least Squares in Code

import numpy as np
from scipy.optimize import minimize

# Negative log-likelihood under Gaussian noise (dropping the constant term)
def neg_log_likelihood(w, X, y, sigma2=1.0):
    residuals = X @ w - y
    return np.sum(residuals**2) / (2 * sigma2)   # minimizing this == maximizing log L(w)

result = minimize(neg_log_likelihood, x0=np.zeros(X.shape[1]), args=(X, y))
w_mle = result.x

# Sanity check: this should match the closed-form Normal Equation solution
w_closed_form = np.linalg.inv(X.T @ X) @ X.T @ y
# w_mle ≈ w_closed_form, confirming MLE under Gaussian noise = least squares

Stage 22 · One-Page Brain

22Everything on One Page

Core Intuition

  • Assume a generative process: yᵢ = wᵀxᵢ + ϵ, ϵ ~ 𝒩(0, σ²)
  • MLE picks the w that makes the observed data most probable
  • Under Gaussian noise, that's provably the same w as least squares
  • Different noise assumption → different "natural" loss function

Key Formulas

  • yᵢ = wᵀxᵢ + ϵ,   P(yᵢ|xᵢ;w) ~ 𝒩(wᵀxᵢ, σ²)
  • L(w) = ∏ᵢ P(yᵢ|xᵢ;w)
  • argmax_w log L(w) = argmin_w Σᵢ(wᵀxᵢ − yᵢ)²
  • Gaussian → L2, Laplacian → L1
