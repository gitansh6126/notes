Lecture 38 · Probabilistic Regression

Bayesian Modeling for Linear Regression

Stop treating W as one fixed unknown number to estimate. Treat it as a random variable with its own belief distribution — then watch Ridge Regression fall out as a special case, for free.

Bayes' Theorem Conjugate Priors MAP Estimation Gaussian Prior Regularization

Cognitive Pattern

what kind of thinking is this?

🎲 Probabilistic Inference

We reason about W as a distribution, not a point — belief gets updated by evidence, exactly like updating odds after new information.

⚙️ Optimization

Finding the single best W (the MAP estimate) is still a maximization/minimization problem underneath the probability.

🔷 Transformation

A probability statement (posterior) is algebraically transformed into a familiar-looking loss function with a penalty term.

0

The Problem

Why does this problem exist?

Ordinary (frequentist) regression treats W as some fixed, unknown but exact true number, and asks "what single value best explains the data I saw?" That view has no natural place to put outside knowledge — e.g. "I expect weights to be small and centered near zero unless the data strongly says otherwise." It also gives no answer for why ridge regression's λ penalty is the "correct" thing to add — it just works empirically.

⚠️
What happens if we don't solve it? We're stuck justifying regularization as a numerical trick (Lecture 35) rather than a principled modeling choice — and we have no framework for expressing genuine prior knowledge, or for reasoning about uncertainty in W itself rather than just a single fitted value.
🎯
Real-life analogy. A detective doesn't start an investigation with a blank mind — they start with priors ("most break-ins are not conducted by strangers with no connection to the house") and update those beliefs as evidence comes in. The final verdict is a posterior belief — a blend of prior suspicion and new evidence, not evidence alone. Bayesian regression treats the weights the same way.
1

History

Bayes' theorem (Rev. Thomas Bayes, published 1763) originally addressed simple conditional probability puzzles. Its use for parameter estimation in statistical models — treating unknown parameters as random variables with a prior distribution — grew into the broader field of Bayesian statistics through the 20th century, and was resisted for decades by the dominant frequentist school, which considered "probability of a parameter" philosophically suspect (parameters are fixed, not random, in that view). Bayesian linear regression became a standard tool once conjugate priors were identified — priors chosen specifically so the posterior update stays in closed form, avoiding intractable integrals.

Engineering Thinking

before any math

Property we want

A way to combine "what the data says" with "what we already believe" into one coherent estimate — and to keep the math tractable while doing it.

Constraint

If prior and posterior belong to different distribution families, every update requires new (often intractable) integration. We need the family to stay closed under updating.

💡
Engineering instinct: pick a prior shape that "plays nicely" algebraically with the likelihood shape you already have (Gaussian likelihood → try a Gaussian prior) so that multiplying them together stays inside the same family — this is exactly what "conjugate" means.

③

Formula Anatomy

X, Y— observed data (features, targets)
W— weight vector, now a random variable
γ²— prior variance: how tightly we believe W is near 0
N(μ,Σ)— Gaussian (Normal) distribution
P(W|Data)— posterior: belief about W after seeing data
1/γ²— plays the exact role of λ from Lecture 35
🔗
Key anatomical insight: a tight prior (small γ²) means "I strongly believe W is near zero" → large 1/γ² → strong regularization. A wide, uninformative prior (γ² → ∞) means "I have no opinion" → 1/γ² → 0 → MAP reduces exactly to plain Maximum Likelihood.
6

Derivation — From Posterior to Penalized Loss

1

Posterior ∝ Likelihood × Prior

P(W|Data) ∝ P(Data|W) · P(W)

Drop the W-independent normalizer P(Data); maximizing the right side is equivalent to maximizing the posterior.

2

Take the log

log P(W|Data) ∝ log P(Data|W) + log P(W)

Log turns a product into a sum, and is monotonic — the same W maximizes both the posterior and its log.

3

Expand each Gaussian log-density

log P(Data|W) = −½Σ(yᵢ−Wᵀxᵢ)² + const

The log of a Gaussian density is a negative squared-error term plus a constant that doesn't involve W.

4

Expand the prior's log-density

log P(W) = −(1/2γ²)‖W‖² + const

Same trick applied to the Gaussian prior on W itself.

5

Flip sign → minimize instead of maximize

min_W ½Σ(yᵢ−Wᵀxᵢ)² + (1/2γ²)‖W‖²

Maximizing a sum of negative terms is the same as minimizing the sum of their positive counterparts — this is exactly the ridge-regression objective.

⚖️
Notice: this derivation never mentions eigenvalues or trace at all — yet it lands on the exact same closed-form estimator as Lecture 35's linear-algebra argument, just arrived at from a completely different (probabilistic) direction, with λ = 1/γ².

Algorithm

# pseudocode: Bayesian (MAP) linear regression fit
def fit_map_ridge(X, Y, gamma_sq):
    lam = 1 / gamma_sq                 # prior variance -> penalty strength
    A = X @ X.T + lam * np.eye(d)
    W_map = np.linalg.inv(A) @ X @ Y
    return W_map

Optimization

ApproachWhat you getCost
MAP (point estimate)Single best W — same cost as ridge regressionCheap — one matrix inverse
Full Bayesian posteriorEntire distribution over W (mean + covariance)More expensive — need posterior covariance too, but still closed-form for Gaussian conjugate case
Posterior predictiveDistribution over new y, not just a point predictionRequires integrating over all plausible W — closed form here, intractable for non-conjugate models
⚡
Because the Gaussian prior is conjugate to the Gaussian likelihood, computing the full posterior — not just its mode (MAP) — is just as cheap: it's another Gaussian with a closed-form mean and covariance. This is the entire payoff of choosing a conjugate prior.
9

Edge Cases & Limitations

γ² → ∞

Uninformative prior. Penalty term vanishes. MAP = plain ML estimator exactly.

γ² → 0

Overwhelmingly strong prior. W is forced toward 0 regardless of data — the data is essentially ignored.

Non-conjugate priors

Swap the Gaussian prior for, say, a Laplace prior (→ Lasso) and the posterior is no longer Gaussian — no closed form; needs numerical/sampling methods (MCMC, variational inference).

MAP ≠ full Bayesian

MAP throws away all the uncertainty information the posterior carries — it reports only the single most likely W, not how confident we should be in it.

Problem Families

MLE

No prior (or a flat/uninformative one). Pure data-driven point estimate.

MAP (this lecture)

Prior + likelihood → single best point (posterior mode). Gaussian prior → Ridge.

Full Bayesian Inference

Keep the entire posterior distribution — report uncertainty, not just a point.

🧩
Swap the prior family and you get a whole map of related estimators: Gaussian prior → Ridge, Laplace prior → Lasso, non-informative/flat prior → plain MLE.
16

Invariant Library ⭐

Always True

Gaussian likelihood × Gaussian prior = Gaussian posterior (conjugacy). MAP = MLE when the prior is uninformative (flat).

Sometimes True

"MAP always outperforms MLE" — only when the prior actually reflects reality; a badly-chosen prior can hurt.

Never True

The posterior is never computed by ignoring the prior — dropping it silently turns Bayesian inference back into plain MLE, not a "better" Bayesian answer.

17

Expert Thinking

🧠
An expert sees any "+ λ‖W‖²" penalty and immediately reads it as "someone put a Gaussian prior on the weights" — and any "+λ‖W‖₁" penalty as "someone put a Laplace prior on the weights." The regularizer is the prior in disguise; recognizing this instantly tells you what assumption is baked into the model.
18

Examiner Trap Library ⭐

🪤
Trap 1: Writing the prior as depending on the data — it doesn't. P(W) is fixed before any data is observed; only the posterior depends on data.
🪤
Trap 2: Forgetting to drop P(Data) as a constant when deriving MAP — it doesn't affect the argmax over W, but students often try to "compute" it unnecessarily.
🪤
Trap 3: Mixing up which variance is which — γ² is the prior's variance (belief about W), not the noise variance of the likelihood (belief about Y given W).
🪤
Trap 4: Assuming MAP gives you uncertainty estimates — it doesn't. MAP is a single point (the peak of the posterior); you need the full posterior covariance for uncertainty.
19

Exam Shortcuts

  • Told "Gaussian prior with variance γ²" → immediately write λ = 1/γ² and reuse the ridge-regression closed form.
  • Asked to derive MAP → don't re-derive from scratch: log(likelihood·prior), expand two Gaussian log-densities, drop constants, done.
  • Told "Laplace prior" → skip straight to "this is Lasso," no closed form exists.
  • γ² → ∞ on an exam question → answer is always "reduces to MLE."

Industry Implementation

# scikit-learn: Bayesian Ridge Regression
from sklearn.linear_model import BayesianRidge

model = BayesianRidge()   # learns prior precision from data automatically
model.fit(X_train, y_train)

y_pred, y_std = model.predict(X_test, return_std=True)
# y_std gives per-prediction uncertainty -- impossible to get from plain Ridge

print("posterior mean weights:", model.coef_)
print("implied lambda (alpha/lambda_):", model.lambda_)
🛠️
For full posterior sampling beyond Gaussian conjugacy (e.g. non-conjugate priors, hierarchical models), tools like PyMC or Stan use MCMC/variational inference instead of closed forms.

Core Intuition

  • Treat W as a random variable with a prior belief. Combine that belief with the likelihood of the observed data via Bayes' theorem to get a posterior. The mode of that posterior (MAP) turns out to be exactly Ridge Regression when the prior is Gaussian.
