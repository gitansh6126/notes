Cross-Validation for Minimizing Mean Squared Error

How the trace of a matrix, its eigenvalues, and a single scalar knob λ combine to fix a structural weakness of Maximum Likelihood regression — and how K-fold cross-validation tells you exactly which λ to pick.

Linear Algebra Regularization Ridge Regression Bias–Variance Model Selection

Cognitive Pattern

what kind of thinking is this?

🔷 Linear Algebra

Eigenvalues of XXT control how "invertible" the problem is. Small eigenvalue → matrix is nearly singular in that direction.

⚙️ Optimization

We are not deriving a new loss — we are tuning a hyperparameter λ to minimize an out-of-sample error surface.

🎲 Probabilistic Tradeoff

Adding λ trades a little bias for a large reduction in variance — the central bias–variance tradeoff, seen through a linear-algebra lens.

0

The Problem

Why does this problem exist?

The Maximum Likelihood estimator WML = (XXT)-1XY is unbiased — on average, across infinitely many datasets, it gets the right answer. But unbiased does not mean low error. If the columns of your data matrix X are highly correlated, or you have limited data relative to features, XXT becomes nearly singular. Inverting a nearly-singular matrix is numerically explosive: tiny noise in the data gets massively amplified into the estimated weights.

⚠️
What happens if we don't solve it? The model still fits the training data perfectly (or near-perfectly) but the learned weights W swing wildly for small changes in the data — classic overfitting / high-variance behavior. Predictions on new data become unreliable even though training error looks great.

🏗️
Real-life analogy. Imagine balancing a table using measurements from two rulers that almost always agree (nearly the same measurement, i.e. nearly parallel data directions). A tiny disagreement between them forces you to make an enormous adjustment to keep the table level — because you're relying on the difference between two near-identical numbers, which is a very unstable quantity. That instability is exactly what a small eigenvalue does to XXT.
1

History

This exact fix — adding λI before inverting — is the classical Tikhonov regularization idea (1940s–60s, ill-posed inverse problems), which entered statistics as Ridge Regression (Hoerl & Kennard, 1970). The motivating failure was always the same: ordinary least squares breaks down whenever the design matrix is close to singular (collinear features, more features than samples). Ridge simply stabilizes the inversion. What this lecture adds is the mechanism — trace and eigenvalues — for why it works, and the practical machinery — cross-validation — for choosing the knob.

Engineering Thinking

before any math

Property we want

A tiny eigenvalue in XXT should not be allowed to blow up the inverse.

Constraint

We can't change the data. Any fix must act on the estimator, not the dataset — and must reduce to the old estimator when the fix is "turned off."

💡
Engineering instinct: if the danger is "dividing by something close to zero," the cheapest fix is to stop it from ever getting that close to zero — add a small positive number to it before dividing.

Derivation — Why Regularization Lowers MSE

1

Start from the trace identity

MSE(WML) ∝ Σᵢ 1/λᵢ

Every small eigenvalue contributes one dangerously large term to this sum.

2

Add λI before inverting

(XXᵀ + λI)

Adding a scalar multiple of the identity shifts every eigenvalue up by exactly λ, without changing the eigenvectors.

3

New eigenvalues of the inverse

1/(λᵢ + λ)

Since λᵢ + λ > λᵢ for any λ > 0, each reciprocal term is strictly smaller than before.

4

Sum shrinks → MSE shrinks

Σᵢ 1/(λᵢ+λ) < Σᵢ 1/λᵢ

Because every one of the n terms in the sum got strictly smaller, the total — and therefore the MSE bound — goes down. There exists a λ > 0 for which Wnew beats WML.

⚖️
The catch: λ doesn't come for free. Too large a λ over-shrinks the weights toward zero, introducing bias — you trade variance for bias. The best λ is the one that minimizes total error, not just the variance term. That's exactly the job cross-validation does.
7

Algorithm — Finding the Best λ

Validation-Set Approach

Split data once: 80% train / 20% validate. For each candidate λ, fit on train, measure error on validation, keep the λ with lowest validation error.

Weakness

One random split can be lucky or unlucky — the chosen λ depends heavily on which 20% happened to land in the validation set.

K-Fold Cross-Validation (K = 5 shown)

Split data into K equal folds. Each fold takes a turn as the validation set while the rest train the model — then average the K errors.

🍰
Analogy. Think of K-fold like taste-testing a cake recipe with 5 different judges, where each judge in turn tastes a slice they didn't help bake, and everyone else's slices were used to bake it. Averaging 5 independent opinions is far more trustworthy than asking just one judge.
# pseudocode
for lambda in candidate_lambdas:
    errors = []
    for k in range(K):
        train_folds = all_folds except fold[k]
        W = fit_ridge(train_folds, lambda)
        err = validation_error(W, fold[k])
        errors.append(err)
    avg_error[lambda] = mean(errors)

best_lambda = argmin(avg_error)
🔁
Leave-One-Out CV (LOOCV): the extreme case K = n (one data point per fold). Uses almost all data for training each time (low bias in the error estimate) but requires n full model fits per λ — expensive when n is large.

In practice, K = 5 or K = 10 is the standard sweet spot — enough folds to average out split-luck, without paying the full O(n) refits of LOOCV. For linear/ridge regression specifically, LOOCV error even has a closed-form shortcut (no refitting needed) — worth knowing exists, even if not derived here.
9

Edge Cases & Limitations

λ = 0

Wnew collapses exactly to WML — no regularization at all.

λ → ∞

Wnew → 0 — the model ignores the data entirely and predicts a constant. Massive bias.

Very small K

K=2 or 3 gives a noisy, high-bias estimate of error — each fold trains on very little data.

K too close to n

LOOCV error estimates across folds become highly correlated with each other, so the variance of the CV estimate itself can be high, despite each fit having low bias.

🚫
Assumption check: XXᵀ must actually be symmetric positive semi-definite for this eigenvalue argument to hold (true by construction for XXᵀ). Also — never let any information from the validation fold leak into training (e.g. don't standardize using statistics computed on the full dataset before splitting).

Pattern Recognition

when to reach for this
  • Features are highly correlated (multicollinearity) → XXᵀ close to singular.
  • More features than samples (p ≥ n) → XXᵀ is literally singular.
  • ML weights are wildly large or unstable across small data changes.
  • Need to choose any hyperparameter (λ, K in k-NN, depth in trees, C in SVM) → cross-validation is the general-purpose answer.

Problem Families

Ridge (L2)

This lecture. Penalty = λ‖W‖². Shrinks weights smoothly, keeps all features.

Lasso (L1)

Penalty = λ‖W‖₁. Shrinks some weights to exactly zero → feature selection.

Elastic Net

Weighted mix of L1 + L2 — best of both, two hyperparameters to tune via CV.

Invariant Library ⭐

Always True

trace(A) = Σ eigenvalues, for any square A. XXᵀ + λI is always invertible for λ>0, even if XXᵀ is singular.

Sometimes True

"Higher λ always improves test error" — only up to a point; too high and bias dominates.

Never True

λ can never be negative (would push some eigenvalue toward 0 or flip sign, re-creating instability or destroying positive-definiteness).

17

Expert Thinking

🧠
An expert doesn't compute eigenvalues by hand to "see" instability — they glance at correlated features or p≥n and immediately think "ill-conditioned, reach for ridge, tune λ by CV." The trace/eigenvalue argument is the justification you derive once; after that it compresses into a one-line reflex.
18

Examiner Trap Library ⭐

🪤
Trap 1: "Larger λ always reduces MSE" — false. It reduces the variance term but increases bias; MSE is U-shaped in λ, not monotonic.
🪤
Trap 2: Confusing trace with determinant. trace = Σλᵢ (sum); determinant = Πλᵢ (product). Only the trace argument is used here.
🪤
Trap 3: Thinking K-fold trains K independent final models to deploy — it doesn't. K-fold is only for selecting λ; the final model is refit once on all the data using the chosen λ*.
🪤
Trap 4: Assuming LOOCV is "always best" because it uses the most training data per fold — its error estimate can have higher variance across different datasets than 5- or 10-fold CV, despite lower bias.
19

Exam Shortcuts

  • Asked for "sum of eigenvalues" → it's the trace; no need to actually factor the matrix.
  • Asked to compare MSE of WML vs Wnew → just compare Σ1/λᵢ vs Σ1/(λᵢ+λ) term-by-term; the second is always smaller for λ>0.
  • K-fold total model fits (excluding final refit) for G values of λ = G × K.
  • LOOCV is just K-fold with K = n — don't treat it as a separate formula to memorize.

Industry Implementation

# scikit-learn: ridge regression + K-fold CV to pick lambda (alpha)
from sklearn.linear_model import Ridge
from sklearn.model_selection import GridSearchCV, KFold

alphas = {'alpha': [0.001, 0.01, 0.1, 1, 10, 100]}  # candidate lambdas
cv = KFold(n_splits=5, shuffle=True, random_state=0)

search = GridSearchCV(Ridge(), alphas, cv=cv,
                      scoring='neg_mean_squared_error')
search.fit(X_train, y_train)

print("best lambda:", search.best_params_['alpha'])
best_model = search.best_estimator_   # refit on all training data w/ best λ
🛠️
sklearn calls λ alpha. RidgeCV is an even faster built-in shortcut that uses the closed-form efficient LOOCV trick for ridge specifically, avoiding a full grid search loop.
