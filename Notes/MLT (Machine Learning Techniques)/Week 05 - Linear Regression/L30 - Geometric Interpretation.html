0Why Look for a Geometric Picture at All?

Algebra gave us an answer, not an intuition

Lecture 29 handed us w* = (XXᵀ)⁻¹Xy through pure calculus — expand, differentiate, set to zero. That's rigorous, but it doesn't tell you why this is the right answer in any way you can picture. This lecture answers that: linear regression is nothing more exotic than finding the closest point on a flat surface to a point that isn't on it.

RMS lens Picture every possible combination of your input features as points confined to a flat sheet floating in a bigger space. The actual outcomes you observed (y) almost never land exactly on that sheet — there's always some noise or unmodeled factor. Regression finds the point on the sheet that's as close as possible to what actually happened.
Stage 17 · Expert Thinking — the single hardest thing to internalize about this lecture

⚠You Just Switched Vector Spaces

Read this before anything else below In Lecture 29, we searched for w living in ℝᴰ parameter space — one axis per feature. In this lecture, the vectors we draw and project — the feature vectors and y — live in ℝᴺ data-point space — one axis per training example. These are two entirely different spaces that happen to share the same underlying matrix X. Confusing which space a picture is drawn in is the single most common way to misread this lecture.

Stage 3 → 5 · Mathematical Translation, Anatomy

3The Concrete Example: N = 3, D = 2

Two features, three data points

Suppose we have D = 2 features (say, height and weight) measured across N = 3 data points. Each feature — read across all 3 data points — becomes one vector in ℝ³:

Feature vectors and the label vector, all in ℝᴺ
f1, f2 ∈ ℝ3     y ∈ ℝ3
Eq. 30.1
f1, f2 — the two feature vectors (rows of X), each with one entry per data point
y — the label vector, one entry per data point, same space as the features
ℝ3 — data-point space here, since N = 3

Because f₁ and f₂ are two vectors in ℝ³, they span a 2-dimensional subspace — a plane passing through the origin. Any prediction the model can possibly make is confined to this plane, since every prediction is some linear combination of f₁ and f₂.

Any prediction is a point in this plane
XTw = w1f1 + w2f2
Eq. 30.2
The key realization Xᵀw is just a weighted sum of the feature vectors — that's exactly the definition of "a point in span{f₁, f₂}." No matter what w you choose, Xᵀw can never leave that plane. The only freedom you have is where on the plane you land.

🖼The Picture: Projecting y Onto the Plane

Reading the diagram y (gold) sits off the plane entirely — it's not achievable by any linear combination of f₁ and f₂. The blue point Xᵀw* is the closest point on the plane to y — found by dropping a perpendicular from y straight down onto the plane. The red dashed vector is the leftover error, and it's drawn at a right angle to the plane on purpose: that's the defining property of an orthogonal projection.
Stage 6 · Derivation — recovering the Normal Equation without calculus

6From "Closest Point" to the Normal Equation

State the orthogonality condition

(y − Xᵀw*) ⊥ f_i    for every feature vector f_i, i = 1, …, D

This is just the geometric definition of "closest point on a plane": the leftover error must be perpendicular to every direction within the plane, not just one.

Write orthogonality as a dot product equal to zero

f_i · (y − Xᵀw*) = 0    for every i

Two vectors are orthogonal exactly when their dot product is zero — this restates step 1 in a form we can compute with.

Stack all D conditions into one matrix equation

X(y − Xᵀw*) = 0   ⟹   Xy − XXᵀw* = 0

Each row of X is one f_i, so stacking every f_i · (…) = 0 condition into a single matrix-vector product gives exactly this — X's rows against the error vector.

Rearrange — this is the Normal Equation from Lecture 29

XXᵀw* = Xy

Identical to Lecture 29's result, but reached purely through "the error must be perpendicular to the subspace" — no gradients, no calculus.

Why this matters Two completely different routes — calculus (Lecture 29) and geometry (this lecture) — land on the exact same equation. That's not a coincidence: minimizing squared distance is orthogonal projection. This is one instance of a much more general fact in linear algebra.
Stage 9 · Edge Cases

9When the Picture Gets More Interesting

y already lies in the plane

If y happens to already be some exact linear combination of f₁ and f₂, the projection is y itself — zero error, a perfect linear fit. This is the geometric picture of the "achievable zero training error" case from Lecture 28, now visible directly.

f₁ and f₂ are not independent

If the feature vectors are linearly dependent (parallel, or one is a multiple of the other), they don't span a full 2D plane — only a line. The subspace collapses, and this is exactly the geometric picture of XXᵀ losing rank from Lecture 29.

The general statement The example uses N = 3, D = 2 purely so it can be drawn. Nothing about the argument depends on those numbers — the same projection logic holds for any N-dimensional data-point space and any D-dimensional feature subspace, exactly as long as D ≤ N and the feature vectors are linearly independent.

Stage 18 · Examiner Trap Library

18Common Exam Traps

⚠ Trap 1 — mixing up which space the picture lives in

A question might draw a "2D plane in 3D space" and ask you to identify axes. The axes here are data points (N of them), not features (D of them) — the opposite of the parameter-space picture from Lecture 29. Labeling axes as features is the single most common mistake with this diagram.

⚠ Trap 2 — thinking the error vector is perpendicular to y, not to the subspace

The orthogonality is between the error vector and every vector in the plane (equivalently, every feature vector) — not between the error and y. y itself is generally not perpendicular to anything in particular.

⚠ Trap 3 — assuming the projection is always unique in every degenerate case

The closest point on the subspace is always unique (that part never fails). What can fail to be unique is w* — if the feature vectors are linearly dependent, multiple different w vectors can produce that same closest point.

Stage 22 · One-Page Brain

22Everything on One Page

Core Intuition

  • Feature vectors (rows of X) span a subspace in ℝᴺ
  • Any prediction Xᵀw is confined to that subspace
  • y usually isn't in it — regression finds the closest point that is
  • That closest point is the orthogonal projection of y

Key Formulas

  • Xᵀw = w₁f₁ + … + w_Df_D ∈ span{f₁,…,f_D}
  • Orthogonality: f_i · (y − Xᵀw*) = 0 ∀i
  • Stacked: X(y − Xᵀw*) = 0
  • ⟹ XXᵀw* = Xy (same Normal Equation as Lecture 29)

5 Traps

  • Dependent features → collapsed subspace (matches Lecture 29's rank issue)
