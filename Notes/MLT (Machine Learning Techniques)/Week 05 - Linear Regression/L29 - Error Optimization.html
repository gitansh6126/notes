0Why Bother With Matrix Notation?

The per-example sum is correct but clumsy

Lecture 28 left us minimizing ŌłæßĄó (wßĄĆxßĄó ŌłÆ yßĄó)┬▓ ŌĆö a sum over n individual training examples. That's correct, but it hides the structure of the problem. Every example contributes an identical-shaped term, which is exactly the signal that the whole sum should collapse into one matrix expression instead of n separate ones.

The payoff: matrix form turns an n-term sum into a single algebraic object we can differentiate once, and ŌĆö if a certain matrix is invertible ŌĆö solve exactly, with no iteration required.

RMS lens This is the difference between writing 500 separate spreadsheet formulas (one per lead) and writing one matrix formula that processes all 500 at once. Same math, but only one of these actually scales.

9What If XXßĄĆ Isn't Invertible?

When invertibility fails

XXßĄĆ is a d ├Ś d matrix, and it fails to be invertible exactly when it doesn't have full rank ŌĆö equivalently, when it has a non-trivial null space (some nonzero vector v with XXßĄĆv = 0). In practice this happens when:

SituationWhy it breaks invertibility
More features than examples (d > n)X itself can't have rank higher than n, so XXßĄĆ (d ├Ś d) can't reach full rank d
Redundant / linearly dependent featuresTwo perfectly correlated features carry the same information ŌĆö XXßĄĆ can't distinguish their individual contributions
The fix ŌĆö MooreŌĆōPenrose pseudo-inverse
w* = (XXT)+Xy
Eq. 29.2
What the pseudo-inverse actually buys you When XXßĄĆ is singular, the Normal Equation has either no exact solution or infinitely many. The pseudo-inverse (XXßĄĆ)Ōü║ is defined for every matrix, singular or not, and specifically returns the least-norm solution ŌĆö among all w that satisfy the Normal Equation equally well, it picks the one with the smallest ŌĆ¢wŌĆ¢┬▓. It's a mathematically principled tie-breaker, not a numerical hack.
RMS lens d > n shows up whenever you track more features than you have historical examples ŌĆö e.g. 40 tracked ad-campaign attributes but only 25 past RMS batches to learn from. The Normal Equation alone can't pick a unique w in that regime; the pseudo-inverse is what makes the problem solvable at all.

Stage 18 ┬Ę Examiner Trap Library

18Common Exam Traps

ŌÜĀ Trap 1 ŌĆö XXßĄĆ vs XßĄĆX confusion

Under this lecture's convention (X is d ├Ś n), XXßĄĆ is d ├Ś d ŌĆö the matrix that must be inverted. XßĄĆX would be n ├Ś n, a completely different object. Swapping them is one of the most common silent errors, especially since many textbooks use the opposite convention where X is n ├Ś d and it's XßĄĆX that gets inverted.

ŌÜĀ Trap 2 ŌĆö assuming a unique solution always exists

"Solve the Normal Equation" only yields one clean w* when XXßĄĆ is invertible. A question that sets up d > n and still asks for "the" solution via direct inversion is testing whether you catch that the inverse doesn't exist there.

ŌÜĀ Trap 3 ŌĆö treating the pseudo-inverse as "an approximation"

The pseudo-inverse isn't a fuzzy or approximate stand-in for the true inverse ŌĆö it's an exact, well-defined operation (MooreŌĆōPenrose) that coincides exactly with the regular inverse whenever one exists, and gives a specific, well-defined least-norm solution when it doesn't.

20Solving This in Practice

# NOTE: numpy/sklearn convention is rows-as-examples (opposite of this lecture's X)
# So here, X_np.shape = (n, d) -- the transpose of this lecture's X
import numpy as np

# Direct Normal Equation (only safe if XXßĄĆ / XßĄĆX is invertible)
w = np.linalg.inv(X_np.T @ X_np) @ X_np.T @ y

# Robust version: uses the pseudo-inverse automatically,
# works even when the direct inverse doesn't exist
w_robust, *_ = np.linalg.lstsq(X_np, y, rcond=None)

# sklearn's LinearRegression uses an SVD-based least-squares solver
# under the hood -- effectively the pseudo-inverse approach, not a raw np.linalg.inv
Practical note Real implementations almost never call np.linalg.inv directly on XXßĄĆ ŌĆö it's numerically unstable when the matrix is close to singular. lstsq / SVD-based solvers are preferred precisely because they degrade gracefully into the pseudo-inverse behavior instead of blowing up.
