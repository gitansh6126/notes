0Where a Straight Line Runs Out of Road

The assumption baked into every earlier lecture

Every regression model since Lecture 28 has assumed y = WTX — a straight-line (or flat-hyperplane) relationship. If the true relationship is curved — quadratic, or involves a cross-term like height × weight — no choice of W can capture it. The model will show high error on both the training set and new data, no matter how it's tuned.

RMS lens If RMS batch revenue depends on enrollment and price in a way that isn't additive — say, revenue peaks at a moderate price and drops off at both extremes — a straight line through that data can't represent a peak at all. You need curvature, not just a better-fitted line.

The obvious fix, and why it's expensive

The direct fix is to manually engineer non-linear features — add x², or x₁·x₂ as new columns — then run ordinary linear regression on the expanded set. This works, but the expanded feature space can explode: a modest polynomial expansion over d original features already produces many new columns, and richer mappings (infinite-dimensional ones, as we'll see) make this approach impossible to compute directly.

The reframe this lecture makes What if we never had to build the expanded features explicitly at all — and instead worked only with dot products between data points, computed directly in the original space? That's the kernel trick.
Stage 6 · Derivation, Step 1

1W* Doesn't Need the Whole of ℝᴰ

The key observation

Split any candidate W into two parts: a component inside span{x₁,…,x_n} (the data), and a component perpendicular to that span. The perpendicular component contributes nothing to any prediction — because predictions are dot products WTxᵢ, and a vector perpendicular to every xᵢ dots to zero with all of them.

So without losing any predictive power, we can restrict the search to W vectors that live entirely within the span of the training data.

The dual representation of W*
W* = XΑ*
Eq. 32.1
X — the d × n data matrix, columns are training points (same convention as Lectures 29–30)
Α* — an n × 1 coefficient vector; one coefficient per training point, replacing the d weights
Why this is the pivotal move We've swapped searching over W ∈ ℝᴰ for searching over Α ∈ ℝᴺ. That doesn't sound like progress on its own — but it means every appearance of the data from here on happens only through dot products between data points, never through the raw features directly. That's exactly the opening the kernel trick needs.
Stage 6 · Derivation, Step 2

2Substituting In: the Kernel Matrix Appears

Start from the Normal Equation (Lecture 29)

XXᵀW* = Xy

The same equation derived by calculus in Lecture 29 and by orthogonal projection in Lecture 30.

Substitute W* = XΑ*

XXᵀ(XΑ*) = Xy  ⟹  X(XᵀXΑ*) = Xy

Just regrouping the parentheses around XᵀX — nothing new yet, but the shape XᵀX is about to matter a lot.

Take the sufficient condition inside the parentheses

XᵀXΑ* = y

If the term inside the parentheses equals y, the outer equation automatically holds. This sidesteps needing X to be invertible or square — a solution to this smaller equation is enough.

Name XᵀX the Kernel Matrix

K = XᵀX,   K_ij = x_i · x_j = Γ(x_i, x_j)

K is n × n — every entry is just a dot product (or, generalized, a similarity function Γ) between two training points. Note: this is XᵀX, not the XXᵀ from Lecture 29 — a different matrix, different shape.

Solve for Α*

Α* = K−1y

An n × n system instead of a d × d one — this is what makes non-linear feature spaces (even infinite-dimensional ones) tractable: K's size depends only on how much data you have, never on how rich the feature mapping is.

Stage 7 · The Algorithm's Core Trick

✚The Kernel Trick, Explained

Why K_ij can be "Γ(x_i, x_j)" instead of a plain dot product

Suppose you wanted to fit a quadratic relationship. The "honest" approach: map each xᵢ into an expanded feature vector φ(xᵢ) that includes x², cross-terms, etc., then run ordinary linear regression on the φ(xᵢ)'s. Every place K_ij appears, you'd compute φ(xᵢ)·φ(xⱼ).

The trick: for many useful mappings φ, the dot product φ(xᵢ)·φ(xⱼ) can be computed directly from xᵢ and xⱼ, without ever constructing φ(xᵢ) explicitly. This direct-computation shortcut is the kernel function Γ(xᵢ, xⱼ), and it's what "the kernel trick" refers to.

Kernel Γ(xᵢ, xⱼ)Implicitly captures
(xᵢ · xⱼ)²Quadratic relationships, including cross-terms — without ever building the expanded quadratic feature vector
(xᵢ · xⱼ)³Cubic relationships, same idea one degree higher
exp(−‖xᵢ − xⱼ‖² / 2σ²)  (Gaussian / RBF)An implicitly infinite-dimensional feature space — impossible to construct explicitly, yet still just an n × n K matrix to invert
Why this is genuinely efficient, not just elegant Whatever the feature space's true dimensionality — even infinite, as with the Gaussian kernel — K stays exactly n × n. The kernel trick's entire value is decoupling the richness of the model from the cost of computing it.
Stage 4 → 5 · Formula Construction, Anatomy

4Predicting on a New Point

Prediction for x_test
Prediction = ∑i=1n Αi* · K(xi, xtest)
Eq. 32.2
K(x_i, x_test) — similarity between the new point and the i-th training point
Αi* — the learned importance/weight of training point i
The intuition in plain language A new point's prediction is a weighted vote from every training point, where the vote strength is how similar the new point is to that training point (via K), and the weight Α_i* is how important that training point turned out to be during fitting. No explicit feature mapping ever needs to be computed, at training or prediction time.
Stage 9 · Edge Cases

9Where This Can Break

K not invertible

Same failure mode as XXᵀ in Lecture 29 — duplicate or near-identical training points make K singular. The fix mirrors that lecture too: use a pseudo-inverse, or add a small ridge term (K + λI) to guarantee invertibility.

Kernel matrix scales with N, not D

K is n × n — this method trades away exactly the D-scaling problem, but reintroduces an N-scaling one: huge datasets make K itself huge to invert. This is why kernel methods often pair with approximations for very large N.

Stage 18 · Examiner Trap Library

18Common Exam Traps

⚠ Trap 1 — K = XᵀX vs XXᵀ, again

This is the same d×d vs n×n confusion from Lecture 29, now with new names attached. XXᵀ (d×d) is the primal Normal Equation's matrix; K = XᵀX (n×n) is the kernel matrix used once W is expressed dually as XΑ*. Mixing them up breaks both the dimensions and the entire point of the kernel trick.

⚠ Trap 2 — thinking the kernel function must literally be a dot product

K_ij = xᵢ · xⱼ is the special case where no mapping is applied (the identity, i.e., ordinary linear regression). Γ(xᵢ, xⱼ) generalizes this to any valid kernel function — it only needs to correspond to some dot product φ(xᵢ)·φ(xⱼ) in a possibly different (even infinite-dimensional) space.

⚠ Trap 3 — believing kernel regression avoids computing similarities to all N training points

The kernel trick avoids computing the expanded feature mapping — it does NOT avoid needing all N training points at prediction time. Every prediction still sums over all N terms K(xᵢ, x_test); this is a real cost that scales with N, unlike the plain W*ᵀx_test that ordinary linear regression uses at test time.

Stage 20 · Industry Implementation

20Kernel Regression in Practice

# X_np.shape = (n, d) -- numpy/sklearn convention, rows are examples
from sklearn.kernel_ridge import KernelRidge

# Gaussian / RBF kernel -- implicitly infinite-dimensional feature space
model = KernelRidge(kernel='rbf', gamma=0.5, alpha=1.0)  # alpha adds the ridge term for invertibility
model.fit(X_train, y_train)     # internally solves (K + alpha*I) Alpha = y
prediction = model.predict(X_test)   # sums Alpha_i * K(x_i, x_test) under the hood

# Polynomial kernel -- explicit finite-degree non-linearity
model_poly = KernelRidge(kernel='poly', degree=2)
Practical note Notice KernelRidge, not plain KernelRegression — nearly every real implementation adds a small ridge (λI) term by default, precisely to sidestep the K-invertibility edge case above.

22Everything on One Page

Core Intuition

  • W* only needs to live in span(training data) — the rest of W contributes nothing
  • So write W* = XΑ*, and solve for Α ∈ ℝⁿ instead of W ∈ ℝᴰ
  • Every step now touches data only via dot products → swap in any kernel Γ
  • Kernel trick: compute φ(xᵢ)·φ(xⱼ) directly, without ever building φ(x)

Key Formulas

  • W* = XΑ*
  • K = XᵀX, K_ij = Γ(x_i, x_j)
  • Α* = K⁻¹y
  • Prediction(x_test) = ∑ᵢ Α_i* K(x_i, x_test)
