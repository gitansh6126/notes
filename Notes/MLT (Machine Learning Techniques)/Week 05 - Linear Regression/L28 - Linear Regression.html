0The Problem: What Makes a Function "Good"?

From "predict Y" to "measure how wrong we are"

Lecture 27 defined the regression goal: learn h : ℝd → ℝ. But that goal is unusable without a way to compare two candidate functions and say which one is better. You can't optimize what you can't measure.

So before any algorithm exists, we need a number that goes down when predictions get better and up when they get worse — an error function.

RMS lens If you're predicting RMS batch revenue, "the model works" is meaningless until you define how wrong is acceptable — off by ₹500 vs off by ₹50,000 need to be penalized very differently. The error function is that definition, made precise.

Engineering requirements for an error function

RequirementWhy
Non-negative"Error" shouldn't be able to cancel itself out or go negative
Zero only at a perfect fitGives a clean target: drive the error to its floor
Penalizes larger mistakes moreBeing off by 10 should hurt more than 10× being off by 1
DifferentiableSo we can actually optimize it with calculus, not just guess-and-check
Stage 3 → 5 · Mathematical Translation, Formula Construction, Anatomy

3Building the Error Function

Step 1 — for one training example, measure the gap between prediction and truth:

Per-example error
(h(xi) − yi)2
Eq. 28.1
Why squared, not just absolute difference? Squaring does two jobs at once: it makes the value non-negative automatically (no need for a separate absolute-value rule), and it's smooth and differentiable everywhere — including at zero, where |·| has a sharp corner that breaks calculus-based optimization. This single choice is why it's called least squares, not "least absolute deviations."

Step 2 — sum this over every training example to get the total error of a candidate function:

Total squared error over the dataset
Error(h) = ∑i=1n (h(xi) − yi)2
Eq. 28.2
h — the candidate function being evaluated
xi — the i-th feature vector
yi — the i-th true label
n — total number of training examples
∑ — accumulate the mistake across every example, none skipped
Reading the anatomy Error(h) is non-negative (sum of squares) and hits exactly 0 only when h(xi) = yi for every single training point — a perfect fit on the data we have.
Stage 9 · Edge Cases — the central failure mode of this lecture

9The Trap Hiding Inside "Minimize the Error"

If Error(h) = 0 is achievable, is that the goal?

If h is allowed to be any function at all from ℝᵈ to ℝ, then yes — trivially. Just define h to literally memorize: h(x_i) = y_i for every training point, and anything else for points it hasn't seen. Error on the training set becomes exactly zero.

This is overfitting: perfect performance on data you've already seen, and no reason at all to expect it to work on data you haven't.

Unrestricted h — wiggles through every point, Error = 0 on training data, but wildly unstable between points.
Linear h — doesn't touch every point exactly, small nonzero Error, but the trend is stable and trustworthy on new points.
Root cause, stated precisely Overfitting isn't caused by "too much training" or "bad luck" — it's caused by the search space being too large. When h is free to be any function, the optimizer has no reason to prefer the true underlying pattern over noise; both achieve the same Error = 0.
RMS lens A model that perfectly "predicts" every past batch's revenue by essentially memorizing each historical case is useless the moment a new batch starts — it has learned your history, not the underlying relationship between enrollment, pricing, and revenue.
Stage 2 → 5 · The Fix: Restrict the Search Space

✚Linear Regression as Inductive Bias

The fix isn't a smarter optimizer — it's a smaller search space

Instead of letting h be anything, we impose structure: assume the relationship between input and output is linear. This single assumption is called an inductive bias — a deliberate restriction that trades away some flexibility in exchange for a model that can't simply memorize noise.

Linear hypothesis space, Hlinear
hw(x) = wTx
Eq. 28.3
w ∈ ℝᵈ — the weight vector; the only thing left to learn
x — the input feature vector
T — transpose, turning the dot product w·x into matrix notation
Note from the lecture This is often extended to hw(x) = wTx + b, adding an intercept/bias term b so the line doesn't have to pass through the origin.

Restricting to Hlinear collapses the entire search from "any function" down to a search over a single vector w ∈ ℝᵈ:

The actual optimization problem we now solve
minw ∈ ℝᵈ   ∑i=1n (wTxi − yi)2
Eq. 28.4
Notice the variable being searched over changed from h (an entire function, infinite-dimensional) to w (a single vector in ℝᵈ, finite-dimensional). That collapse is the entire point of introducing a hypothesis space.
Stage 6 · Derivation — where does solving this actually lead?

6Where This Optimization Goes Next

The lecture stops at setting up Eq. 28.4 — but it's worth seeing one step ahead, since it's the natural next move and shows why "linear + squared error" was such a deliberate combination:

Start from the objective

∑ᵢ (wᵀxᵢ − yᵢ)²

This is a smooth, differentiable function of w — a direct consequence of choosing squared error back in Eq. 28.1.

Differentiate with respect to w

∇w ∑ᵢ (wᵀxᵢ − yᵢ)² = 2 ∑ᵢ (wᵀxᵢ − yᵢ) xᵢ

Because the objective is a sum of squared linear terms, its gradient is itself linear in w — this is what makes a closed-form solution possible at all.

Set the gradient to zero

∑ᵢ (wᵀxᵢ − yᵢ) xᵢ = 0

A minimum of a smooth convex function occurs exactly where its gradient vanishes — no need to check second-order conditions separately, since sum-of-squares in a linear model is always convex.

Solve for w (preview of the Normal Equation)

w* = (XᵀX)−1 Xᵀy

Stacking all xᵢ into a matrix X and all yᵢ into a vector y turns the per-example sum into one clean matrix expression — the subject of the next lecture.

Honest scope note Steps 3–4 are a preview, not part of this lecture's stated content — Lecture 28 only sets up Eq. 28.4. It's included here because seeing where the optimization leads makes the squared-error choice in Eq. 28.1 make sense in hindsight.

Stage 16 · Invariant Library

16Always / Sometimes / Never True

Always True

Error(h) = ∑(h(xᵢ) − yᵢ)² is non-negative, and zero only at a perfect fit on the training set.

Restricting the hypothesis space always removes some functions that could have achieved zero training error.

Sometimes True

A linear model achieves zero training error — only when the true relationship really is linear and n ≤ d+1, not in general.

Never True

Zero training error never guarantees good generalization — the two are logically independent, and this lecture's central point.

Stage 18 · Examiner Trap Library

18Common Exam Traps

⚠ Trap 1 — "lower training error" ≠ "better model"

A question may show two models and ask which is "better" using only training Error(h). Without a held-out test set, the model with the lowest training error could simply be the most overfit one.

⚠ Trap 2 — confusing why squared error was chosen

The reason isn't "it's more intuitive" — it's specifically differentiability and the resulting convex, closed-form-solvable optimization. |h(xᵢ) − yᵢ| is also non-negative and zero at a perfect fit, but isn't differentiable at zero.

⚠ Trap 3 — treating w and h as the same kind of object

h is a function (infinite-dimensional search space in general); w is a finite vector in ℝᵈ once you've committed to Hlinear. The whole benefit of linear regression comes from collapsing the former into the latter.

Stage 20 · Industry Implementation

20Where This Shows Up in sklearn

# X.shape = (n, d) — n examples, d features each
# y.shape = (n,)   — n real-valued labels
from sklearn.linear_model import LinearRegression

model = LinearRegression()
model.fit(X_train, y_train)      # solves the min_w problem internally (Eq. 28.4)

# model.coef_  ≈ the learned w
# model.intercept_ ≈ the learned b, if fit_intercept=True (default)

train_error = ((model.predict(X_train) - y_train) ** 2).sum()
test_error  = ((model.predict(X_test)  - y_test)  ** 2).sum()

# the gap between train_error and test_error is your overfitting signal
Looking ahead When train_error is very low but test_error is high — even inside the restricted linear space — that's the motivation for the next tools in the course: regularization (Ridge, Lasso), which shrink w itself rather than restricting the function class further.
