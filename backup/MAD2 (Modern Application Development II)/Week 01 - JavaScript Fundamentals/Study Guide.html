Scope

Deep Dive
Global Scope

Variables declared outside any function or block. Accessible everywhere.

var globalVar = "I'm global";
let globalLet = "Also global";

function foo() {
  console.log(globalVar); // accessible
}
Function Scope

var is function-scoped. Visible inside the function only.

function bar() {
  var secret = "hidden";
  console.log(secret); // "hidden"
}
// console.log(secret); // ReferenceError
Block Scope

let / const are block-scoped ({ }).

if (true) {
  let blockScoped = "inside";
  var notBlockScoped = "leaks";
}
console.log(notBlockScoped); // "leaks"
// console.log(blockScoped); // ReferenceError
Lexical Scope Important

Inner functions have access to variables of outer (parent) functions. Scope is determined by where functions are written, not where they are called.

function outer() {
  let outerVar = "I'm outside";
  function inner() {
    console.log(outerVar); // "I'm outside"
  }
  inner();
}
outer();
PYQ — Nested Function Scope
PYQ ★
var x = 10;
function a() {
  console.log(x);
  function b() {
    var x = 20;
    console.log(x);
  }
  b();
}
a();

Output: 10, 20
▶ inner x shadows outer in b

PYQ ★
let a = 1;
function outer() {
  console.log(a);
  let a = 2;
}
outer();

Output: ReferenceError
▶ TDZ for let in function scope

Hoisting

Deep Dive
Variable Hoisting var
// What you write:
console.log(a);
var a = 5;

// What JS sees at runtime:
var a;          // hoisted (undefined)
console.log(a); // undefined
a = 5;          // init stays

Only the declaration is hoisted, not the initialization.

Function Hoisting Declaration
// Function declarations are hoisted entirely
sayHi(); // "Hello!" — works!
function sayHi() {
  console.log("Hello!");
}

// Function expressions are NOT hoisted
sayBye(); // TypeError — not a function
var sayBye = function() {
  console.log("Bye!");
};
Temporal Dead Zone (TDZ) let / const
// let and const are hoisted but NOT initialized
// They exist in the "TDZ" from start of scope → declaration
{
  // TDZ starts here for 'x'
  // console.log(x); // ReferenceError
  let x = 5;
  // TDZ ends
  console.log(x); // 5
}
💡
Memory Aid: var gets a welcome party (undefined). let/const arrive but stay invisible until their line.
PYQ — Hoisting
PYQ ★
console.log(foo());
function foo() { return 1; }
var foo = function() { return 2; };

Output: 1 ▶ Declaration hoisted above expression, function body wins

The this Keyword

Deep Dive
Four Binding Rules
RulePatternthis value
1. DefaultStandalone function callwindow (browser) / global (Node) / undefined (strict)
2. ImplicitMethod call: obj.method()The object before the dot
3. Explicit.call(), .apply(), .bind()Provided value
4. newnew Constructor()Newly created instance
Default Binding
function show() {
  console.log(this);
}
show(); // window (non-strict)
Implicit Binding
const obj = {
  name: "MAD2",
  greet() { console.log(this.name); }
};
obj.greet(); // "MAD2"
Explicit Binding
function greet() {
  console.log(this.name);
}
const user = { name: "Exam" };
greet.call(user);   // "Exam"
greet.apply(user);  // "Exam"
const bound = greet.bind(user);
bound();            // "Exam"
Arrow Function this (Lexical) Tricky
const obj = {
  name: "MAD2",
  regular: function() { console.log(this.name); },
  arrow: () => { console.log(this.name); }
};
obj.regular(); // "MAD2" (own this)
obj.arrow();   // undefined (inherits from outer scope)
⚠️
Arrow functions do not have their own this. They inherit it from the enclosing (parent) scope — lexically, not dynamically.
Common Traps Table
PatternWhat this refers to
obj.method()obj
const fn = obj.method; fn()window (lost context)
setTimeout(obj.method, 100)window
Arrow function in object methodOuter scope (not the object)
Callback inside methodwindow (unless arrow)
Event handler (element.onclick)The element
PYQ — this Patterns
PYQ ★ Implicit
const obj = {
  x: 42,
  getX() { return this.x; }
};
console.log(obj.getX());

Answer: 42

PYQ ★ Lost Context
const obj = {
  x: 42,
  getX() { return this.x; }
};
const fn = obj.getX;
console.log(fn());

Answer: undefined (in strict) / window.x

PYQ ★ Arrow + setTimeout
const obj = {
  x: 10,
  log() {
    setTimeout(() => {
      console.log(this.x);
    }, 0);
  }
};
obj.log();

Answer: 10 (arrow inherits from log's this)

Closures

Deep Dive
Definition

A closure is a function that "remembers" the variables from its lexical scope even after the outer function has returned.

function outer(x) {
  return function inner(y) {
    return x + y;  // 'x' is from outer scope
  };
}
const add5 = outer(5);
console.log(add5(3)); // 8 — closure "remembers" x=5
Data Privacy
function counter() {
  let count = 0;
  return {
    increment: () => ++count,
    get: () => count
  };
}
const c = counter();
console.log(c.get()); // 0
c.increment();
console.log(c.get()); // 1
Currying
const multiply = a => b => a * b;
const double = multiply(2);
console.log(double(5)); // 10
Loop Problem
// Using var — all log 3
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// Output: 3, 3, 3

// Fix with let — 0, 1, 2
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
PYQ — Closure Output
PYQ ★
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), i * 100);
}

Output: 3 3 3 ▶ All closures share same i

PYQ ★
function createCounter(start) {
  return function() {
    return start++;
  };
}
const cnt = createCounter(5);
console.log(cnt());
console.log(cnt());

Output: 5, 6

Prototype & Inheritance

Deep Dive
Prototype Chain
// Every JS object has a [[Prototype]]
const arr = [1, 2, 3];
console.log(arr.__proto__ === Array.prototype);     // true
console.log(Array.prototype.__proto__ === Object.prototype); // true
console.log(Object.prototype.__proto__);             // null (end of chain)

// Property lookup walks up the chain
arr.hasOwnProperty("length"); // from Array.prototype?
💡
__proto__ is the actual prototype link of an object. prototype is a property on constructor functions (used with new).
Inheritance via Prototypes
function Animal(name) {
  this.name = name;
}
Animal.prototype.speak = function() {
  console.log(this.name + " makes noise");
};

function Dog(name) {
  Animal.call(this, name);
}
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.bark = function() {
  console.log("Woof!");
};

const d = new Dog("Rex");
d.speak(); // "Rex makes noise"
d.bark();  // "Woof!"
ES6 Classes = Syntactic Sugar
class Animal {
  constructor(name) { this.name = name; }
  speak() { console.log(this.name + " makes noise"); }
}
class Dog extends Animal {
  constructor(name) { super(name); }
  bark() { console.log("Woof!"); }
}
const d = new Dog("Rex");
d.speak();
PYQ — Prototype Look-up
PYQ ★
function Foo() {}
Foo.prototype.x = 10;

const a = new Foo();
Foo.prototype = { x: 20 };
const b = new Foo();

console.log(a.x, b.x);

Output: 10 20 ▶ a links to old prototype, b to new

ES6 Classes

Deep Dive
Class Declaration
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  greet() { return `Hi, I'm ${this.name}`; }
  static isPerson(obj) {
    return obj instanceof Person;
  }
  get info() { return `${this.name} (${this.age})`; }
  set info(val) {
    [this.name, this.age] = val.split(" ");
  }
}
Inheritance
class Student extends Person {
  constructor(name, age, rollNo) {
    super(name, age); // must call super first!
    this.rollNo = rollNo;
  }
  greet() {
    return super.greet() + ` and I'm student #${this.rollNo}`;
  }
}

const s = new Student("Alice", 20, 101);
console.log(s.greet()); // "Hi, I'm Alice and I'm student #101"
PYQ — Class Output
PYQ ★
class A {
  constructor() { this.x = 1; }
  method() { console.log(this.x); }
}
class B extends A {
  constructor() {
    super();
    this.x = 2;
  }
  method() { console.log(this.x + 1); }
}
const b = new B();
b.method();

Output: 3

bind() / call() / apply()

Deep Dive
Comparison Table
MethodInvokes?ArgumentsReturns
call(thisArg, arg1, arg2, ...)✓ ImmediatelyListFunction result
apply(thisArg, [argsArray])✓ ImmediatelyArrayFunction result
bind(thisArg, arg1, ...)✗ Returns new functionListBound function
call
function introduce(city) {
  console.log(`I'm ${this.name} from ${city}`);
}
introduce.call({name: "Ram"}, "Chennai");
// "I'm Ram from Chennai"
apply
function sum(a, b, c) { return a + b + c; }
console.log(sum.apply(null, [1, 2, 3]));
// 6 — spread array as args

// Borrowing methods:
const args = [1, 2, 3];
const max = Math.max.apply(null, args);
console.log(max); // 3
bind
function log(prefix, message) {
  console.log(`${prefix}: ${message}`);
}
const logError = log.bind(null, "ERROR");
logError("Something broke");
// "ERROR: Something broke"

// Fix this:
const bound = obj.method.bind(obj);
PYQ — Output Prediction
PYQ ★
const obj = { x: 1 };
function foo() { console.log(this.x); }
foo.call(obj);

Answer: 1

PYQ ★
const obj = { x: 1 };
function foo() { console.log(this.x); }
const bound = foo.bind(obj);
bound.call({ x: 2 });

Answer: 1 ▶ bind cannot be overwritten

PYQ ★
function foo(a, b) { return a + b; }
const bar = foo.bind(null, 2);
console.log(bar(3));

Answer: 5 ▶ Partial application

Arrow Functions

Deep Dive
Syntax Variations
// Basic
const add = (a, b) => a + b;

// Single param — parens optional
const square = x => x * x;

// No params
const now = () => Date.now();

// Block body — explicit return
const sum = (a, b) => {
  const result = a + b;
  return result;
};

// Returning object literal
const getObj = () => ({ key: "value" });

Tricky Questions

10 Must-Know
⚠️ this in nested arrow
const obj = {
  x: 1,
  outer() {
    const inner = () => {
      console.log(this.x);
    };
    inner();
  }
};
obj.outer(); // 1 (arrow inherits from outer)
⚠️ Closure + setTimeout loop
// var i — prints 3,3,3
// let i — prints 0,1,2
for (var i = 0; i < 3; i++) {
  (function(j) {
    setTimeout(() => console.log(j), 100);
  })(i);
} // 0,1,2 (IIFE fix)
⚠️ Function hoisting
foo(); // 1
function foo() { console.log(1); }
var foo = function() { console.log(2); };
⚠️ typeof null
console.log(typeof null);      // "object" (bug)
console.log(typeof undefined); // "undefined"
console.log(null === undefined); // false
⚠️ [] == ![]
// [] == ![]  →  [] == false  →  "" == false  →  0 == 0  → true
console.log([] == ![]); // true
⚠️ Event handler this
btn.addEventListener("click", function() {
  console.log(this); // btn element
});
btn.addEventListener("click", () => {
  console.log(this); // window (lexical)
});
⚠️ Late prototype modification
function Foo() {}
Foo.prototype.x = 10;
const a = new Foo();
Foo.prototype = { x: 20 };
console.log(a.x); // 10 (still old proto)
⚠️ new + arrow
const Fn = () => {};
new Fn(); // TypeError: Fn is not a constructor
⚠️ bind chaining
function foo() { console.log(this.x); }
const a = foo.bind({ x: 1 });
const b = a.bind({ x: 2 });
b(); // 1 (first bind wins)
⚠️ const mutation
const obj = { a: 1 };
obj.a = 2;       // ✓ allowed
obj.b = 3;       // ✓ allowed
// obj = {}; // ✗ TypeError
console.log(obj); // { a: 2, b: 3 }

Common Traps

Student Mistakes

1. Hoisting
❌ Student Thinks: "var a = 5 is moved to top"
✅ Actually: "Only declaration is hoisted, not initialization"

2. this in closure
❌ Student Thinks: "this refers to outer function"
✅ Actually: "this is determined by how function is called"

3. const immutability
❌ Student Thinks: "const obj = {} can't be modified"
✅ Actually: "const prevents reassignment, not mutation"

4. typeof null
❌ Student Thinks: "null is an object"
✅ Actually: "typeof null returns 'object' (legacy bug)"

5. Arrow function this
❌ Student Thinks: "arrow functions have their own this"
✅ Actually: "arrow functions inherit this from parent scope"
