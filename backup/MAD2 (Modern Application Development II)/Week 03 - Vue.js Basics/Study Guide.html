Topic Deep Dives

01 Components

Props

  • Static props: prop-name="value"
  • Dynamic props: :prop-name="dataVar" (v-bind)
  • Validation: type: String, required: true, default: 'fallback', validator: fn
  • One-way data flow: parent → child; child never mutates props
  • Props are read-only in the child

Emit

  • this.$emit('event-name', payload)
  • Parent listens: @event-name="handler"
  • v-model is syntactic sugar for :value + @input (Vue 2) or :modelValue + @update:modelValue (Vue 3)

Parent-Child Communication

  • Parent → Child: props
  • Child → Parent: $emit
  • Alternative — Event Bus: new Vue() as hub (Vue 2) / mitt (Vue 3)
  • Provide/Inject: for deeply nested components; provide() in ancestor, inject in descendant
PYQ: Component communication questions — Explain the one-way data flow pattern in Vue. How does $emit enable child-to-parent communication?

02 Slots

  • Default slot: <slot></slot> — catches everything not named
  • Named slots: <slot name="header"></slot> → <template v-slot:header>
  • Slot props: <slot :item="item"></slot> — child can pass data into the slot
  • Fallback content: <slot>Default content</slot> — shown when no slot content provided
  • Shorthand: v-slot:header → #header
PYQ: Slot rendering questions — What is the output when a named slot is provided but the child defines multiple named slots?

03 Scoped Slots

  • Child passes data to parent via <slot :data="data"></slot>
  • Parent accesses: <template v-slot:default="slotProps">{{ slotProps.data }}</template>
  • Destructuring: v-slot="{ item, index }"
  • Named scoped slots: v-slot:column="props"
  • Use cases: reusable list renderers, table column templates, data-formatting wrappers
PYQ: Scoped slot output — Given a scoped slot passing user.name and user.age, what is the rendered output?

04 Watch

  • Basic: watch: { dataVar(newVal, oldVal) { … } }
  • Deep watch: watch: { obj: { handler() {}, deep: true } } — watches nested properties
  • Immediate: immediate: true — runs handler immediately on creation
  • Multiple sources (Vue 3): watch([refA, refB], ([a, b], [oldA, oldB]) => …)
  • vs Computed: Use watch for side effects (API calls, async); use computed for derived values
PYQ: Watch output questions — Given a deep watcher on user.profile.city, does changing user.profile.city trigger the handler?

05 Computed

  • Definition: properties that are computed reactively based on other reactive data
  • Caching: computed properties are cached based on their reactive dependencies; only re-evaluate when dependencies change
  • Getter only: computed: { fullName() { return this.first + ' ' + this.last } }
  • Getter/Setter: computed: { fullName: { get() {}, set(val) {} } }
  • vs Methods: methods re-run on every render, computed caches; use methods for arguments, computed for derived state
  • vs Watch: computed for derived state (pure), watch for side effects (impure)
  • Dependency tracking: Vue tracks which reactive dependencies the computed uses; when those change, computed invalidates
PYQ: Computed property output — If count is 5 and computed: { double() { return this.count * 2 } }, what is double?

06 Lifecycle Hooks

Execution Order

HookWhen It RunsWhat You Can Do
beforeCreateAfter init, before reactivity—
createdReactivity ready, no DOMFetch data, init non-DOM
beforeMountBefore DOM renderLast-minute changes
mountedDOM attachedAccess DOM, third-party libs
beforeUpdateBefore re-renderLog pre-update state
updatedAfter re-renderDOM updated
beforeDestroyBefore teardownCleanup, remove listeners
destroyedComponent gone—

Key Points

  • API calls in created vs mounted: Both work; created is slightly earlier (no DOM needed for fetch)
  • Cleanup in beforeDestroy: Remove event listeners, clear intervals, unsubscribe from stores
  • keep-alive hooks: activated / deactivated when toggling cached components
PYQ: When does X happen? — In which lifecycle hook is the DOM first available? (Answer: mounted)

07 Router

  • Setup: createRouter (Vue 3) with createWebHistory or createWebHashHistory
  • Dynamic routing: /user/:id → accessed via $route.params.id
  • Nested routes: children array under parent route
  • Navigation guards:
    • beforeEach (global) — auth checks, redirects
    • beforeEnter (per-route)
    • beforeResolve — after guards but before navigation
    • afterEach — post-navigation analytics
    • In-component: beforeRouteEnter, beforeRouteUpdate, beforeRouteLeave
  • Lazy loading: component: () => import('./Comp.vue')
  • $route vs $router: $route is current route (params, query); $router is the instance (push, replace, go)
  • Named routes/views: name: 'user' → router.push({ name: 'user', params: { id: 1 } })
  • Navigation methods: push (adds history), replace (no history), go (back/forward)
PYQ: Router guard output — If beforeEach redirects to /login when no token, what happens when navigating to /dashboard without a token?

09 Reactivity System

FeatureVue 2 (Object.defineProperty)Vue 3 (Proxy)
DetectionGet/set on existing keysAny property get/set
New propertiesNot reactive ($set needed)Automatically reactive
Array indexNot detectedDetected
Array.lengthNot detectedDetected
DeleteNot detected ($delete needed)Detected

Dependency Tracking

  • Each reactive property has a Dep (dependency) instance
  • When a getter runs, the current watcher subscribes to the Dep
  • When a setter fires, the Dep notifies all subscribed watchers
  • Composition API: ref() (primitive values), reactive() (objects)

Tricky Questions ⚠️
⚠️ Reactivity with array index
Vue 2 can't detect arr[0] = newVal. Use Vue.set(arr, 0, newVal) or splice.
⚠️ Async Vuex mutations
Mutating state outside a mutation handler violates Vuex rules; DevTools can't track it.
⚠️ Computed vs Watch
Computed doesn't work for side effects (API calls, async). Use watch for that.
⚠️ Slot scope
Parent cannot access child data in slots unless child uses scoped slots.
⚠️ Navigation guard redirect loop
Guard that redirects unconditionally causes infinite redirect. Check target first.
⚠️ Lifecycle with keep-alive
activated/deactivated run on toggle; mounted runs only once.
⚠️ Props mutation
Never mutate props in child — one-way data flow violation. Use local copy or emit.
⚠️ Vuex module namespace
Without namespaced: true, getters/mutations register globally — collisions possible.
⚠️ Router params change
Component is reused, not remounted. Use watch: { '$route'(to, from) {} }.
⚠️ Event bus memory leak
Forgetting $off in beforeDestroy causes memory leaks and duplicate handlers.

Watch/Computed 5 Watch/Computed Patterns

  1. Computed derived state: fullName() { return this.first + ' ' + this.last }
  2. Computed with getter/setter: fullName: { get() {}, set(v) { … } }
  3. Watch with deep: deep: true for nested object changes
  4. Watch with immediate: immediate: true to run on creation
  5. Watch multiple sources (Vue 3): watch([a, b], ([newA, newB], [oldA, oldB]) => …)

Slots 5 Slot Patterns

  1. Default slot: <slot>fallback</slot>
  2. Named slot: <slot name="header"/> → v-slot:header
  3. Scoped slot: <slot :item="item"/> → v-slot="{ item }"
  4. Named scoped slot: <slot name="row" :data="d"/> → v-slot:row="{ data }"
  5. Slot forwarding: Wrapper component forwards all slots to child
